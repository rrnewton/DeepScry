# JavaScript Testing Infrastructure Summary

## Core Test Scripts (Committed to Git)

These are the **only** JavaScript test files tracked in git and are part of `make validate`:

### 1. `web/test_fancy_tui.js`
**Purpose**: End-to-end test for the fancy TUI (browser-based terminal UI)

**What it tests**:
- WASM module loading
- Deck loading and initialization
- TUI launch and rendering
- Turn execution (runs 5 manual turns)
- Auto-run mode
- Screenshot capture for visual regression testing
- Performance metrics and timing

**Called by**: `make validate` → `make web-test`

**Dependencies**: Playwright (browser automation)

**Output**:
- Screenshots in `web/screenshots/`: `01_setup.png`, `02_tui_launched.png`, `03_turn_XX.png`, `04_after_autorun.png`
- Test results JSON: `web/screenshots/test_results.json`

**Exit code**: 0 on success, 1 on failure

---

### 2. `web/test_human_input.js`
**Purpose**: Test human controller input handling in WASM TUI

**What it tests**:
- Human player can play lands one at a time
- Each land appears on battlefield correctly
- DOM text extraction and game state verification
- Sequential action handling

**Called by**: `make validate` → `make web-test`

**Dependencies**: Playwright

**Output**:
- Screenshots in `web/screenshots/`
- Test results JSON: `web/screenshots/human_test_results.json`

**Exit code**: 0 on success, 1 on failure

---

### 3. `web/test_wasm.js`
**Purpose**: Basic WASM module loading and initialization test

**What it tests**:
- WASM module loads without errors
- Basic initialization checks

**Called by**: `make wasm-test` (not part of default validate)

**Dependencies**: Playwright

**Output**: Console logs only

**Exit code**: 0 on success, 1 on failure

## Validation Flow

```
make validate
  ├─ make test           # Rust unit/integration tests
  └─ make web-test       # JavaScript E2E tests
       └─ cd web && node test_fancy_tui.js && node test_human_input.js
```

## Test Requirements

All three test scripts require:
- Node.js with Playwright installed (`npm install` in web/ directory)
- Python 3 (for running local HTTP server on port 8765/8766)
- WASM build artifacts in `web/` directory (generated by `mtg export-wasm`)
- Chromium browser (installed by Playwright)

## Debug Scripts (NOT in Git)

All one-off debugging and test scripts should be placed in the `debug/` directory, which is gitignored:

- `debug/check_*.js` - Font rendering and measurement checks
- `debug/debug_*.js` - Various debugging utilities
- `debug/inspect_*.js` - DOM inspection tools
- `debug/measure_*.js` - Measurement utilities
- `debug/quick_screenshot.js` - Quick screenshot capture
- `debug/*.py` - Python debugging servers
- `debug/*.txt` - Test output logs

See `CLAUDE.md` section "Debugging Scripts and Temporary Files" for details.

## Adding New Tests

### For Core Test Suite (will be in git):
1. Create test in `web/test_*.js`
2. Add to `Makefile` web-test target
3. Ensure it runs in CI
4. Document in this file

### For One-Off Debugging (won't be in git):
1. Create script in `debug/`
2. Run manually as needed
3. No need to add to Makefile
4. Will be automatically gitignored
