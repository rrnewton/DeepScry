# Description

## Damage log shows incorrect life total (double subtraction)

Context:
- Date: 2026-04-03
- Decks: monored_simple.dck mirror
- Mode: zero-vs-zero, seed 7

Steps to Reproduce:
1. Run: target/release/mtg tui decks/monored_simple.dck --p1 zero --p2 zero --seed 7 --p1-draw "Mountain;Lightning Bolt;Lightning Bolt;Grizzly Bears;Mountain;Mountain;Mountain" -v verbose --no-color-logs --stop-on-choice 10
2. Observe Lightning Bolt resolving

Expected Behavior:
- Log: "Lightning Bolt (8) deals 3 damage to Zero2 (life: 17)" — P2 starts at 20, takes 3, ends at 17

Actual Behavior:
- Log: "Lightning Bolt (8) deals 3 damage to Zero2 (life: 14)" — shows 14 instead of 17
- State dump correctly shows Life: 17

Rules Notes:
- This is a cosmetic logging bug, not a gameplay bug. Life totals are tracked correctly.

Root Cause:
- In priority.rs, resolve_spell() (line 69) applies damage BEFORE log_effect_execution() (line ~193) runs
- log_effect_execution() in logging.rs:134 reads current_life from game state (already reduced to 17)
- Then computes life_after = current_life - amount = 17 - 3 = 14 (double subtraction)

Fix:
- Either log BEFORE resolve_spell, or read life BEFORE resolve and pass it to the logger
- Simplest: change logging.rs:135 to just use current_life (which is already post-damage)

Code Locations:
- mtg-engine/src/game/game_loop/priority.rs:69 — resolve_spell called before logging
- mtg-engine/src/game/game_loop/priority.rs:193 — log_effect_execution called after
- mtg-engine/src/game/game_loop/logging.rs:134-135 — double subtraction
