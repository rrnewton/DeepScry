# Description

All Hallow's Eve (Legends, cardsfolder/a/all_hallows_eve.txt) does not correctly self-exile with two scream counters when it resolves.

Script:
  A:SP$ ChangeZone | Origin$ Stack | Destination$ Exile | RememberChanged$ True | SubAbility$ DBPutCounter
  SVar:DBPutCounter:DB$ PutCounter | Defined$ Remembered | CounterType$ SCREAM | CounterNum$ 2 | SubAbility$ DBCleanup

Observed:
  Player 1 casts All Hallow's Eve
  All Hallow's Eve resolves
  All Hallow's Eve (3) puts 2 Scream counter(s) on Unknown (0)
  Graveyard: 3 (was 2), Exile: 0

Two bugs:
1. ChangeZone Origin$ Stack | Destination$ Exile does not redirect: the resolved sorcery still ends up in graveyard. Search for ChangeZone handling with Origin$ Stack in mtg-engine/src/loader/effect_converter.rs and mtg-engine/src/game/actions/.
2. RememberChanged$ True does not register the moved card, so the chained PutCounter SubAbility resolves against an empty Remembered list (manifests as 'puts counters on Unknown (0)').

Reproducer: ./target/release/mtg tui --start-state test_puzzles/all_hallows_eve_resurrection.pzl --p1=fixed --p2=zero --p1-fixed-inputs="cast All Hallow's Eve" --stop-on-choice=20 --json --log-tail=400 --seed 42 --verbosity 3
