# Description

## PER-CARD ASPECT ENUMERATION (2026-06-11_#3227(8f869714), agent beads-enumerate-1994)

PURPOSE: an ACCURATE, SCRIPT-DERIVED checklist of every behavioral aspect each
unique card exhibits — the true surface area to verify, irrespective of current
compatibility. Each `- [ ]` is an UNCHECKED aspect; gameplay-evidence ticking is
a LATER phase (do NOT read these as WORKING). Derived by reading each card's
Forge-style script in cardsfolder/. Known BROKEN/status notes are preserved in
the STATUS NOTES section that follows; the checkboxes here describe what the card
is SUPPOSED to do per its printed text + script.

FORMAT per card:
  ### <Card> — <mana cost> <type line> [decks] (cardsfolder/<file>.txt)
  - [ ] <one aspect per printed ability / mode / keyword / notable interaction>
Legend: (A)=activated, (T)=triggered, (S)=static, (R)=replacement, (K)=keyword,
(cast)=cast-time/targeting, (cost)=cost/additional-cost, (mode)=modal,
(int)=interaction/scaling/oddity.

Scope: 95 unique cards (union of decks/championship/1994/*.dck main+sideboard,
4 finalist decks 01_dolan_wug_stasis / 02_lestree_rg_zoo / 03_symens_zoo /
04_defoucaud_zoo). All 95 resolve to a cardsfolder script. Deck membership in
`[..]` brackets uses the deck number prefixes 01/02/03/04.

### Bugs found (top 3)

**B1 — Chaos Orb never activated (FIXED)**
Root cause: heuristic's has_valuable_destroy_target() only considered tapped
creatures (Royal Assassin logic). Chaos Orb has no 'tapped' requirement in its
description, so targets_tapped=false and the check always failed.
Fix: Added requires_tapped_target: bool to ActivatedAbilityType::Destroy variant.
When false (general destroy, e.g. Chaos Orb), the heuristic evaluates any
opponent permanent with value >= 25. Verified: post-fix game shows Chaos Orb
activating and destroying Argothian Pixies.
Files: heuristic_controller/mod.rs, abilities.rs, creature_eval.rs

**B2 — Ancestral Recall always draws for caster (NOT FIXED)**
Root cause: ApiType::Draw converter in effect_converter.rs ignores ValidTgts$
Player. Always resolves to PlayerId::placeholder() which defaults to the caster.
Decision: functionally acceptable for 99% of use (draw 3 for yourself). Deferring
to a future targeted pass — not a game-breaking crash or silence.

**B3 — Stasis upkeep sacrifice trigger never fired (FIXED)**
Root cause: ApiType::Sacrifice not handled in the phase trigger SVar parser
(card.rs parse_triggers()). The trigger was detected correctly but effects list
was empty, so neither the sacrifice nor the UnlessCost payment ever happened.
Stasis would sit forever, skipping untap steps, without ever costing the
controller the {U} upkeep or self-sacrificing.
Fix: Added new Effect::SacrificeSelf { source: CardId } variant. Phase trigger
parser now emits SacrificeSelf (placeholder) wrapped in UnlessCostWrapper for
DB$ Sacrifice | UnlessCost$ U patterns. check_triggers_for_controller resolves
the placeholder to the actual source CardId at fire time. Also affects Aura Flux
and Arcades Sabboth which share the same SVar pattern.
Verified: debug log shows 'SacrificeSelf: 92 moved to Graveyard' in games where
Stasis is in play and controller cannot pay {U}.
Files: core/effects.rs (new variant), game/actions/mod.rs, game/game_loop/logging.rs,
game/actions/targeting.rs, game/state.rs, loader/card.rs
