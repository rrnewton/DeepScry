== NETWORK DESYNC (robots42 mirror) — root-caused, NOT yet fixed (fix-desync-realdecks pass) ==

The `decks/old_school/03_robots_jesseisbak.dck` seed=42 mirror match still
desyncs P2 (WASM shadow) and is therefore NOT re-added to the
`web/test_network_multideck.js` gate. The desync is a DIFFERENT, broader root
cause than mtg-609 (which was fixed this pass):

ROOT CAUSE: WASM-shadow **in-stack-resolution re-entry**. When a spell's
resolution requires a player choice mid-resolution and returns `NeedInput`,
`resolve_top_spell_with_discard_hook` / `resolve_top_spell_from_stack_interactive`
(game_loop/priority.rs) unwind with the spell STILL on the stack
(`resolve_spell_finalize` not yet reached). On WASM GameLoop re-entry the whole
function re-runs from the FIRST effect — re-executing already-applied effects.

Confirmed for two cards in this deck (instrumented runs, since reverted):
- **Copy Artifact** (`DB$ Clone | Choices$ Artifact.Other`): the Clone target
  choice ran TWICE on the WASM client (CLONE_DEBUG at action 905 then again at
  907) while the server ran it once -> client 3 actions adrift, P2 hash mismatch
  (e.g. choice_seq=183, Turn 13 Main2; choice_seq varies run-to-run because the
  native P1 random controller is unseeded).
- **Balance**: server resolved Balance's symmetric sacrifice/discard (~7 actions)
  while the WASM client was still blocked pre-resolution (client 7 behind,
  Turn 9 Main1, choice_seq=107).

Because the native P1 client is `--controller random` with no seed, the failure
choice_seq drifts (107/172/183/324/447 observed across runs) but always lands on
an in-resolution choice (Clone / Balance / discard-style effect).

FIX DIRECTION (next pass): give spell resolution an effect-resume index so
re-entry skips already-executed effects, mirroring the existing activated-ability
mechanism (`pending_activation_effect_idx` + `pending_activation` in priority.rs
and the `pending_cast` cast-time-targeting resume). This is a broader change to
the resolution hook than the mtg-609 phase-trigger guard and must be validated
across every card that uses choice-bearing in-resolution effects (DiscardCards,
Loot, Scry, Surveil, Clone, Dig, SearchLibrary). Deferred to keep this pass's
mtg-609 fix clean and isolated.

Note: the mtg-609 upkeep/end-step re-entry guard landed this pass does NOT fix
this — it is a distinct stack-resolution path.
