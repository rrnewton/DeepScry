# Description

## Context
- Date: 2026-04-03
- Decks: decks/old_school2/rg_ponza.dck (mirror)
- Seed: 42
- Mode: heuristic-vs-heuristic

## Steps to Reproduce
1. Run: `./target/release/mtg tui decks/old_school2/rg_ponza.dck decks/old_school2/rg_ponza.dck --p1=heuristic --p2=heuristic --seed=42 --verbosity=3`
2. Look at Turn 13 combat damage output

## Expected Behavior
When 2 unblocked Su-Chi (4/4 each) deal damage simultaneously, the log should show consistent values. Either:
- Sequential: "Su-Chi deals 4 → life: 12" then "Su-Chi deals 4 → life: 8"
- Or simultaneous: both show "life: 8" (the final result)

## Actual Behavior
```
Su-Chi (53) deals 4 damage to AI-Heuristic2 (life: 12)
Su-Chi (54) deals 4 damage to AI-Heuristic2 (life: 12)
```

Both show life: 12, but P2's actual life on the next turn display is 8 (correctly computed as 16-4-4=8). The log values are cosmetically incorrect.

Also observed with 3 attackers (Turn 14):
```
Llanowar Elves deals 1 → life: 7
Su-Chi deals 4 → life: 4
Su-Chi deals 4 → life: 4
```
Starting from 8: 8-1-4-4=-1. Game correctly reports "lost the game (life: -1)" but intermediate display is inconsistent (7 is correct for first hit, but 4 should be 3 for second, and -1 for third).

## Rules Notes
- CR 510.2: All combat damage is dealt simultaneously (in normal combat damage step)
- The actual damage computation is CORRECT (final life totals match)
- This is purely a display/logging issue, not a gameplay bug

## Evidence
- Command: see above
- Verified final life totals match expected arithmetic across all turns
