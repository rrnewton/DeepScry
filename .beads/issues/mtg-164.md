# Description

## Description

## Problem

The benchmark infrastructure has conflicting global allocator declarations that prevent proper DHAT heap profiling of gameplay workloads. Currently:

1. **benches/allocator.rs** sets global allocator via features (bench-stats-alloc, bench-mimalloc, bench-jemalloc)
2. **bin/rewind_bench.rs** tries to add DHAT support but conflicts with benchlib's allocator
3. **benches/dhat_profile.rs** works by being standalone (doesn't include benchlib)

**Result**: Cannot run DHAT profiling on rewind_bench's full 1000-game workload. Only captures initialization allocations, not gameplay.

**Critical Gap**: We need to profile the actual gameplay allocations to understand:
- Source of 78-92 GB allocations in parallel benchmarks (vs 3.2 GB sequential)
- GameState::clone() allocation patterns for MCTS
- Per-turn allocation hotspots during gameplay

## Solution Implemented (2025-11-07_#819)

### Phase 1: Allocator Conflict Resolution ✅

**Changes to allocator.rs**:
- Added feature conflict checks for `dhat-heap` vs all bench-* features
- Made global allocator declarations conditional on `not(feature = "dhat-heap")`
- Updated `allocator_name()` to recognize DHAT mode

**Changes to rewind_bench.rs**:
- Made `Region::new(GLOBAL)` and stats_alloc usage conditional on `not(feature = "dhat-heap")`
- Added `print_results_dhat()` function for DHAT profiling mode
- Updated documentation with correct build command: `--no-default-features --features dhat-heap`

**Result**: Clean compilation in both modes without conflicts.

### Phase 2: Profiling & Analysis ✅

Successfully profiled 1000-game workload:

```bash
cargo run --release --no-default-features --features dhat-heap --bin rewind_bench -- -n 1000 --dhat
```

**Metrics**:
- Total allocations: 1,477,549,735 bytes (1.48 GB)
- Allocation calls: 20,404,820 blocks
- Average per game: ~1.48 MB
- Peak footprint: 765,516 bytes
- Performance: 31.20 games/sec, 585 actions/game

**Key Finding**: >60% of allocations (900+ MB) come from `String::to_lowercase()` calls in:
- `get_complex_mana_production` (mana_engine.rs:561)
- `get_valid_targets_for_ability` (actions.rs:494-497)
- `has_mana_ability` (mana_engine.rs:482)

**Documentation**: Complete analysis in `experiment_results/dhat_allocation_analysis_2025-11-07_#819.md`
