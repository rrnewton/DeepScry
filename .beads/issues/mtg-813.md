# Description

THE NEW PRIZE (successor to the action_count-in-hash prize, landed @1070b585): we can run randomized fuzz testing in parallel on the machine for ONE HOUR over the old-school deck set, with ALL debug checks ON, and NOT find a single deck+seed that desyncs. 'Desync is always fatal' (docs/NETWORK_ARCHITECTURE.md) — this prize is met only when an hour of expedition surfaces zero new fatal divergences.

== DEFINITION OF A '1-HOUR BUG-FINDING EXPEDITION' ==
Take a set of decks (the old-school corpus: decks/old_school/*.dck + decks/old_school2/*.dck), play them with RANDOM seeds, ALL debug checks turned on (network-debug, rewind/replay verifier, maximally-strict state hash incl. action_count — now ON by default post-1070b585). Whether to focus local vs network, wasm vs native is a CONFIGURABLE knob — we want ALL of these options available and exercised over the hour:
  - local single-process games (native determinism: same seed -> identical gamelog)
  - network games: native-client vs native-client, native-client vs WASM-browser-client, AND WASM vs WASM (the configs that exist)
  - native-vs-WASM strict equivalence
  - snapshot/resume stress
Pick concurrency that avoids port collisions (today the TOCTOU port picker collides — see mtg-806 :0 fix; needed for clean parallel runs).
