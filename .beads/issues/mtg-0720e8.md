# Description

## Problem

Split cards with names like "Claim // Fame" or "Life // Death" fail to load because the card loader doesn't handle the " // " separator in filenames.

## Error

\`\`\`
Loading card database...
Error: InvalidCardFormat("Card file not found: 'Claim // Fame' (expected path: cardsfolder/c/claim_//_fame.txt)")
\`\`\`

## Root Cause

Split cards have names with " // " between the two halves. The current filename conversion logic doesn't handle this pattern correctly. It's trying to look for \`claim_//_fame.txt\` instead of the actual file.

## Discovery

Found when running \`./scripts/random_decks_tournament.sh\`, which selects 50 random decks from 6,727+ available decks. Many commander and standard decks contain split cards.

## Examples of Affected Cards

- Claim // Fame
- Life // Death
- Fire // Ice
- Boom // Bust
- Hide // Seek
- Many others with the split card format

## Expected Behavior

Split card files are likely stored with a different naming convention, similar to double-faced cards. Need to:
1. Identify the actual filename pattern for split cards in cardsfolder
2. Add special handling in card loader (similar to DFC prefix matching from commit e65e7ae)
3. Support both halves being castable/usable

## Investigation Needed

\`\`\`bash
## Find split card files

find forge-java/forge-gui/res/cardsfolder -name "*life*death*" -o -name "*death*life*"
