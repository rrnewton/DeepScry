# Description

Running 'make validate' regenerates the 11 tracked PNG screenshots under web/screenshots/landing_page_qa/ (produced by the WASM/playwright e2e tests) with non-deterministic bytes, leaving the worktree dirty after every validate run. Three separate agents (code-elegance, Clone, deploy) hit this and had to 'git checkout -- web/screenshots/landing_page_qa/' before committing/rebasing; one (code-elegance) accidentally swept the changed PNGs into a commit via 'git add -A'. FIX options: (a) gitignore web/screenshots/landing_page_qa/*.png and stop tracking them (they are QA artifacts, not source); or (b) make the screenshot rendering deterministic; or (c) have the e2e tests write screenshots to a gitignored dir. Until fixed: never 'git add -A' in a worktree post-validate; discard these PNGs explicitly.
