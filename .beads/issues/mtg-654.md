# Description

RESOLVED (WORKING) 2026-05-30, compat-wave18-triggers.

Fellwar Stone's ApiType::ManaReflected now produces mana (was a no-op).
"{T}: Add one mana of any color that a land an opponent controls could produce."

Implementation (info-independent / deterministic; NO controller-choice layer
crossed — modeled exactly like the existing Produced$ Any path):

1. effect_converter.rs: new ApiType::ManaReflected arm produces an AddMana with
   all five colors set (upper bound), same encoding as Produced$ Any. So the
   static mana-production cache derives ManaProductionKind::AnyColor and Fellwar
   Stone is recognized as a mana source.

2. loader/card.rs: ManaReflected counts as is_mana_ability (CR 605) and sets the
   new ActivatedAbility.produces_reflected_mana flag (core/effects.rs).

3. actions/mod.rs: at tap time (the any-color branch of tap_for_mana_for_cost),
   when produces_reflected_mana is set, the produced color is constrained to the
   REFLECTED SET computed by new helper reflected_mana_colors(player): the union
   of colors that opponents' lands could produce, read from each land's static
   mana_production cache (Fixed -> that color, Choice -> those colors, AnyColor ->
   WUBRG, Colorless -> none). The cost-hint color is honored if in the set, else
   the first reflected color in WUBRG order. This is the same deterministic,
   cost-hint-driven color selection used for AnyColor lands today — no
   ChoiceRequest, no hidden info (color set derives purely from public
   battlefield state). DEFERRED: nothing functionally; the only simplification vs
   a full controller prompt is that color selection is engine-deterministic
   (cost-hint-driven), identical to how all AnyColor sources already behave in
   this engine.
