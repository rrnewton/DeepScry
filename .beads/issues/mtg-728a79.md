# Description

## Problem

Cards with activated abilities that use \`AB$ ChangeZone\` (tutoring, searching library) are not being parsed/loaded correctly. This prevents players from using abilities like fetch lands, evolving wilds, etc.

## Example: Vibrant Cityscape

**Card**: Vibrant Cityscape
**Definition** (forge-java/forge-gui/res/cardsfolder/v/vibrant_cityscape.txt):
\`\`\`
A:AB$ ChangeZone | Cost$ T Sac<1/CARDNAME> | Origin$ Library | Destination$ Battlefield | Tapped$ True | ChangeType$ Land.Basic | ChangeTypeDesc$ basic land | SpellDescription$ Search your library for a basic land card, put it onto the battlefield tapped, then shuffle.
\`\`\`

**Expected**: After playing Vibrant Cityscape, player should be able to tap and sacrifice it to search for a basic land.

**Actual**: The activated ability doesn't appear in available actions.

## Root Cause

The card loader doesn't support:
1. \`AB$ ChangeZone\` activated ability type
2. \`Sac<1/CARDNAME>\` cost syntax (sacrifice this permanent)
3. Parsing library search / tutoring effects

## Scope

This affects ALL cards with similar activated abilities:
- Fetch lands (Evolving Wilds, Terramorphic Expanse, etc.)
- Tutor effects with activation costs
- Any \`AB$ ChangeZone\` ability
