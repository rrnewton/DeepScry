# Description

CLUSTER: UI/infra. Related: mtg-569, mtg-432, mtg-707, mtg-669.

In a 2-player NETWORKED game, the TUI log render showed duplicated/garbled turn-banner lines interleaved with the battlefield box-drawing.

ROOT CAUSE (found 2026-06-02, branch fix-log-rendering) — SHARED with mtg-432: same stale-wrapped-log-cache bug. The network human path (fancy_tui.rs run_network_mode_human_v2) calls rewind_to_turn_start on replay, which truncates the log buffer (logger.truncate_to). The LogWrapCache only invalidated on width/perspective change or append-only growth, never on truncation. After truncate+replay re-grew the log past the cached processed_count, the append-only needs_update path appended the freshly-replayed turn banners ON TOP of the cached pre-rewind banners -> DUPLICATED turn-banner lines; misaligned line_starts -> garbled interleaving with box borders.

FIX (commit 54e84c31): logger log_epoch bumped on every truncate_to/clear_logs; LogWrapCache compares epoch (epoch_stale) and forces a full rebuild on mismatch. See mtg-432 for full detail + regression tests.

NOTE: the name confusion (typed username 'oaeu' vs deck-name 'rogue_rogerbrand' used as a player name) is a SEPARATE concern not addressed here — should be split to its own issue if still reproducing. The duplication/garble is fixed by the epoch invalidation. Pending full validate + push; not self-merged.
