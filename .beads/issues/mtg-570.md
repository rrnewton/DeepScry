# Description

CLUSTER: UI/infra (possibly network). Related: mtg-569.

In a 2-player networked game, the TUI log render shows duplicated/garbled turn-banner lines interleaved with the battlefield box-drawing, e.g.:

>>> Turn 1 - oaeu 20 (01_rogue_rogerbrand 20… │ (repeated many times, interleaved with │ │ box borders)

Two possible causes to investigate:
1. UI: the log pane re-renders/appends the same line on every frame without dedup, or the ratatui/WASM TUI render loop double-prints.
2. Network: the server sends duplicate log/state messages that the client appends naively.

Note the player names are "oaeu" (typed username) and "01_rogue_rogerbrand" (deck name used as a name?) — suggests a name/deck-label confusion worth checking too. Start by diffing what the server sends vs what the client renders.
