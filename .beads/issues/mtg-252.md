# Description

## Network Library Search State Divergence

**Problem:** During library searches (typecycling like Mountaincycling) in network mode, the client's shadow game state diverges from the server because searched cards aren't properly instantiated, causing 'Entity not found' errors when trying to use the cards.

**Root Cause:** The `process_card_reveal` function in client.rs had a catch-all `_ => {}` that only logged `RevealReason::Searched` without instantiating the card in the shadow game.

**Solution:** Added explicit handling for `RevealReason::Searched` to instantiate cards in the shadow game's `game.cards` storage, similar to how `RevealReason::Draw` is handled.

**Files Modified:**
- `mtg-engine/src/network/client.rs` - Added `RevealReason::Searched` handling in `process_card_reveal`
- `mtg-engine/src/network/server.rs` - Added `CardRevealed` with `RevealReason::Searched` for library search results
- `mtg-engine/src/network/protocol.rs` - Added `library_search_result` field to `OpponentChoice`
- `mtg-engine/src/network/local_controller.rs` - Handle library search result from server choice
- `mtg-engine/src/game/game_loop/network_choice.rs` - Added `sync_to_action()` after library search choice

**Status:** FIXED - All 823 tests pass, network_vs_local_equivalence_e2e.sh produces identical gamelogs
