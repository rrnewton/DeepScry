# Description

Test all behavioral aspects of All Hallow's Eve in MTG Forge-rs.

Set: LEG (mtg-386)
Deck: rogue_rogerbrand (mtg-387)
Card script: cardsfolder/a/all_hallows_eve.txt
Test puzzle: test_puzzles/all_hallows_eve_resurrection.pzl

Findings (2026-05-12, compat1):

1. [x] Castable for {2}{B}{B} as Sorcery
2. [BROKEN] On resolution, the card does NOT go to Exile. It ends up in Graveyard instead. Exile zone stays empty (0); Graveyard count goes from 2 to 3 (the All Hallow's Eve being added).
   - Likely cause: ChangeZone Origin$ Stack | Destination$ Exile is not handled — the engine probably defaults to graveyard for resolved sorceries, ignoring the ChangeZone redirect.
3. [BROKEN] 'puts 2 Scream counter(s) on Unknown (0)' — the counters go to a phantom 'Unknown(0)' object, not on the All Hallow's Eve. The Defined$ Remembered list is empty when the PutCounter subability resolves (because RememberChanged$ True did not record the card or because the move-to-exile failed first).
4. [BLOCKED] Trigger never fires: the upkeep trigger requires PresentZone$ Exile + IsPresent$ Card.Self+counters_GE1_SCREAM. Since the card is in graveyard with no counters, the trigger condition never holds.
5. [BLOCKED] No removal of counters per upkeep (because trigger doesn't fire).
6. [BLOCKED] No transition to graveyard from exile + no mass-resurrection.

Reproducer: ./target/release/mtg tui --start-state test_puzzles/all_hallows_eve_resurrection.pzl --p1=fixed --p2=zero --p1-fixed-inputs="cast All Hallow's Eve" --p2-fixed-inputs='' --stop-on-choice=20 --json --log-tail=400 --seed 42 --verbosity 3

Log evidence:
  Player 1 casts All Hallow's Eve (3) (putting on stack)
  All Hallow's Eve (3) resolves
  All Hallow's Eve (3) puts 2 Scream counter(s) on Unknown (0)
  All Hallow's Eve (3) clears remembered cards
  Hand: 0 | Library: 10 | Graveyard: 3 | Exile: 0   <-- WRONG: should be Exile: 1, Graveyard: 2

CARD STATUS: BROKEN — the self-exile-with-counters mechanism is unimplemented or buggy. Two distinct bugs:
  (a) ChangeZone Origin$ Stack | Destination$ Exile not redirecting resolved sorcery to Exile (search loader/effect_converter.rs and game/actions/* for ChangeZone with Origin$ Stack).
  (b) RememberChanged$ True not populating the Remembered list before SubAbility chain runs.

Recommend filing follow-up bug 'all-hallows-eve-self-exile-broken' P3 referencing this compat issue.
