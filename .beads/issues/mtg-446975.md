# Description

Files: agentplay/agent_game.py (new)

Action: Create the main Python script that orchestrates agent-based MTG games:
1. Parse CLI args: deck files, seed, player controllers (agent/heuristic/random), max-choices, max-turns
2. Call start_game.sh to initialize the game and get first snapshot
3. Game loop:
   a. Parse the JSON snapshot to get current game state + choice menu
   b. If active player is agent-controlled: invoke claude -p with game state and choice list, parse numeric choice from output
   c. If active player is heuristic/random: let the engine handle it (pass choice=* or similar)
   d. Call continue_game.sh with the chosen action
   e. Check stop conditions (game over, max choices, max turns)
4. The script orchestrates the alternation: agent decision → engine execution → next choice

Key details:
- start_game.sh creates game dir with initial state, p1/p2_choices.txt, snapshot
- continue_game.sh takes game_dir and choice number, appends to choices, replays full game
- JSON snapshot contains: game state, choice menu, active player, turn number
- claude -p invocation should include: current game state summary, available choices, previous reasoning context
- Use with-proxy for claude invocations if needed

Verify:
- python3 agentplay/agent_game.py --help shows usage
- Script can start a game and make at least one agent choice
- Handles game-over detection gracefully
