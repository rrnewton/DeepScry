# Description

Test all behavioral aspects of Timetwister in MTG Forge-rs (1994 Old School, Troll Disk deck mtg-562).

Card file: cardsfolder/t/timetwister.txt
Card text: {2}{U} Sorcery — Each player shuffles their hand and graveyard into their library, then draws seven cards. (Then put Timetwister into its owner's graveyard.)
Script:
  A:SP$ ChangeZoneAll | ChangeType$ Card | Origin$ Hand,Graveyard | Destination$ Library | Shuffle$ True | UseAllOriginZones$ True | SubAbility$ DBDraw
  SVar:DBDraw:DB$ Draw | NumCards$ 7 | Defined$ Player

Findings (2026-05-29_#2456(e30f4ce1), compat-oldschool-wave5):

1. [x] Parses as {2}{U} Sorcery; the DBDraw sub-ability (each player draws 7)
   resolves correctly.
2. [BROKEN] The shuffle-back of Hand+Graveyard into the library is WRONG.
   ROOT CAUSE (two layers):
   - effect_converter.rs ApiType::ChangeZoneAll calls
     Zone::from_str_lenient("Hand,Graveyard"). That helper only parses a
     SINGLE zone, so the comma-list returns None and the code falls back to
     `Zone::Battlefield`. Timetwister therefore tries to move all BATTLEFIELD
     permanents (ChangeType$ Card matches everything) into the library —
     i.e. it shuffles away each player's board instead of their hand+gy.
   - Effect::ChangeZoneAll carries a single `origin: Zone`, so even a
     correct parse cannot express the two origin zones Hand AND Graveyard.
   - The application handler (actions/mod.rs Effect::ChangeZoneAll) also
     lists Hand/Exile/Library/Stack origins as "not yet supported"
     (returns empty), so Hand would be a no-op even if parsed.
   - The gamelog prints a raw Debug dump of TargetRestriction { ... } and
     "from Battlefield to Library", a sentinel-grade misleading log
     (compatibility_tracking SKILL §2.2).
   Proper fix requires: (a) ChangeZoneAll to support multiple origin zones
   (origins list), (b) converter to parse comma-separated Origin$ +
   UseAllOriginZones$, (c) application support for Hand origin, (d) a clean
   gamelog line. This touches the Effect enum (hashing / network / AI), so
   it is deferred as a focused bug rather than a late-session change.

Reproducer (shows the bug):

```sh
./target/release/mtg tui decks/old_school/06_troll_disk_daniellebrunazzo.dck \
  decks/old_school/06_troll_disk_daniellebrunazzo.dck \
  --p1=heuristic --p2=heuristic --seed 13 --verbosity 3
```

Observed (WRONG) log:

```
  Timetwister (59) moves all matching cards (TargetRestriction { ... }) from Battlefield to Library
  Timetwister (59) causes each player to draw 7 card(s)
```

CARD STATUS: BROKEN — draws 7 correctly, but the hand+graveyard shuffle-back
is mis-routed to the battlefield (wrong zone, would shuffle the board into
the library) and logs a sentinel. Engine gap in ChangeZoneAll multi-origin
support. Shares the bug class with any Origin$ <list> + Hand-origin
ChangeZoneAll (e.g. Wheel-style mass shuffle, Diminishing Returns).
