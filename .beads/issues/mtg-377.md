# Description

## Context

Commit 01d90c49 fixed Clue token's 'Sac<1/CARDNAME>' self-sacrifice activation parsing. The fix is targeted to that one cost-string shape.

## Pattern
Predefined and SVar-defined tokens often reuse parameterized cost macros where CARDNAME is substituted by the bearer:
- Clue: T, Sac<1/CARDNAME>, 2: Draw a card
- Treasure: T, Sac<1/CARDNAME>: Add one mana of any color
- Food: 2, T, Sac<1/CARDNAME>: Gain 3 life
- Blood: 1, T, Sac<1/CARDNAME>, Discard: Draw
- Map: 1, T, Sac<1/CARDNAME>: Scry then explore (sub-class — needs explore primitive)

## Goal
- Audit all token defs in cardsfolder/tokens/ for the parameterized cost shape
- Add regression test that exercises Treasure and Food (Clue already covered)
- Confirm 'CARDNAME' substitution happens at activation, not at parse time

## Files
- mtg-engine/src/loader/card.rs (token loading)
- mtg-engine/src/cost.rs (Sac<N/...> parsing)
- cardsfolder/tokens/
