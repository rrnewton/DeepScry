# Description

Two related lobby UI improvements for web/index.html.

## Task 1: Passcode show/hide eyeball
Both passcode inputs (create-game form + per-row join form) are type=password. Add an inline eyeball toggle (SVG, eye/eye-slash) that flips type between password and text. Friend-share passcodes are not high-security — typo prevention beats hiding. Must be accessible (aria-label, aria-pressed, keyboard-focusable).

## Task 2: Scalable filterable game list
Server-side (mtg-engine/src/network/):
- Extend ClientMessage::ListGames with optional ListGamesQuery { filter, limit, offset }. filter is case-insensitive substring vs game name OR creator name. limit default 20, max 100. offset default 0.
- Extend ServerMessage::GameList with total_count: u32.
- Backwards compat: absent filter+limit → return all games (preserves existing tests).

Client-side (web/index.html):
- Filter inputs (debounced 200ms) above games list.
- Scrollable container with max-height + overflow-y: auto.
- 'Showing X of Y · [< prev] [next >]' pagination footer (page size 20).
- Preserve auto-refresh behavior.

Tests: extend web/test_landing_page_ux.js with toggle + filter + pagination scenarios.

Validation: make validate + make clippy.
