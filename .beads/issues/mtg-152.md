# Description

Implement full spell copying for CopySpellAbility when used as a SubAbility (e.g., Chain Lightning).

## Current Status

The CopySpellAbility effect now has the infrastructure to handle both:
- `Defined$ TriggeredSpellAbility` - for delayed triggers (existing)
- `Defined$ Parent` - for SubAbility chaining (NOT YET IMPLEMENTED)

When `Defined$ Parent` is used (e.g., Chain Lightning), the current implementation:
1. Logs that the copy would happen (INFO level)
2. Does NOT actually create the copy
3. The opponent pays the cost but no copy is put on the stack

## Chain Lightning Behavior

Chain Lightning works as follows:
1. Deals 3 damage to any target
2. Target's controller may pay RR
3. If paid, copy the spell and they may choose new targets
4. The copy can target the original caster, creating a "chain"

## Implementation Requirements

To fully implement this:
1. Track the currently resolving spell in execute_effect context
2. Clone the spell's effects with potentially new targets
3. Create a new spell instance controlled by the specified Controller$
4. Put the copy on the stack
5. Handle target selection (MayChooseTarget$ True allows retargeting)

## Cards Affected

- Chain Lightning (Legends, 4ED)
- Any other card using `DB$ CopySpellAbility | Defined$ Parent`

## Files to Modify

- `mtg-engine/src/game/actions/mod.rs`: execute_effect for CopySpellAbility
- May need context tracking in resolve_spell
