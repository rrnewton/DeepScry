# Description

## Problem

Several game state mutations are NOT logged to the undo system, making it impossible to correctly rewind game state. This breaks tree search and AI development.

## Missing Mutations

### 1. `turn_entered_battlefield` field (Card struct line 251)
   - Set in: `play_land` (actions/mod.rs:35)
   - Set in: `resolve_spell` when entering battlefield (actions/mod.rs:274)
   - **Impact**: Cannot properly undo summoning sickness state

### 2. `attached_to` field (Card struct line 283)
   - Set in: `attach_equipment` (actions/mod.rs:349)
   - Cleared in: `detach_equipment` (actions/mod.rs:391)
   - Cleared in: state-based action when creature leaves battlefield (state.rs:268)
   - **Impact**: Cannot undo Equipment attachment state

### 3. `lands_played_this_turn` field (Player struct line 25)
   - Incremented in: `play_land` via `player.play_land()` (actions/mod.rs:40, player.rs:64)
   - Reset in: `advance_step` when turn changes (state.rs:599)
   - **Impact**: Cannot undo "can play land" restrictions
   - **NOTE**: May be inferrable from MoveCard undo (if undoing Hand→Battlefield for a land, decrement count)

## Solution Approach

Some mutations can be **inferred** during undo:
- `lands_played_this_turn`: Decrement when undoing MoveCard(Hand→Battlefield) for a land
- `turn_entered_battlefield`: Clear when undoing MoveCard(*→Battlefield), restore when undoing MoveCard(Battlefield→*)

Others need **explicit logging**:
- `attached_to`: Add `AttachEquipment{equipment_id, old_target, new_target}` variant
- Damage tracking (see mtg-153)

## Files to Modify

- `src/undo.rs` - Add new GameAction variants as needed
- `src/game/actions/mod.rs` - Log mutations in play_land, resolve_spell, attach_equipment, detach_equipment
- `src/game/state.rs` - Implement undo logic for new variants
