# Description

Implement full CopySpellAbility for SubAbility context (Chain Lightning) — DONE

## Status: IMPLEMENTED (2026-06-09, claude/compat-1994-damage)

The `Defined$ Parent` CopySpellAbility (Chain Lightning's "...may pay {R}{R}. If
the player does, they may copy this spell and may choose a new target for that
copy") now actually puts a copy on the stack. Previously it only logged "Copy
not yet implemented".

## Root cause found (the real gap)

The blocker was NOT just "copy infrastructure missing" — the head ability's
`SVar:DBCopy1:DB$ CopySpellAbility | ... | UnlessCost$ R R | UnlessPayer$
TargetedOrController | UnlessSwitched$ True` was being lowered by
`follow_sub_ability_chain` (loader/card.rs) WITHOUT applying
`wrap_with_unless_cost`. So the optional {R}{R} gate was silently dropped and a
BARE CopySpellAbility was emitted. The head-ability path
(`params_to_effect_with_unless`) wraps; the SVar-aware sub-ability builders did
not. Fixed by wrapping every sub-effect in `follow_sub_ability_chain` — benefits
ALL sub-abilities carrying an UnlessCost.

## What was implemented

1. loader/card.rs `follow_sub_ability_chain`: apply `wrap_with_unless_cost` to
   sub-effects (the missing UnlessCost gate).
2. game/state.rs `copy_spell_onto_stack(original, new_controller, new_targets)`:
   single shared helper that clones the resolving spell, gives it a new id, sets
   the controller, flags it `is_token` (CR 707.10c), pushes it on the stack, and
   registers its `spell_targets`. BOTH the delayed-trigger path (Jeong Jeong,
   `Defined$ TriggeredSpellAbility`) and the new Parent path call it (DRY — the
   delayed path previously re-implemented the clone-push inline AND forgot to
   register the copy's targets, a latent bug now fixed).
3. game/actions/mod.rs `Effect::CopySpellAbility { Parent }`: copies
   `current_damage_source` (the resolving spell — still on the stack during its
   own SubAbility execution) under the resolved controller; with
   `MayChooseTarget$ True` and a DIFFERENT controller, retargets the copy at the
   original caster (the canonical "chain").
4. game/actions/mod.rs UnlessCostWrapper resolution: resolve `TargetedOrController`
   payer via the last-resolved target (decoding player-target sentinels), and
   pin the inner CopySpellAbility's `Controller$ TargetedOrController` to that
   same payer (the one paying {R}{R} controls the copy).
5. game/actions/mod.rs UnlessCost Mana payment: replaced the old "auto-success
   (no deduction)" with `pay_mana_cost_by_tapping` — proper colored-mana
   affordability (ManaEngine) + real source tapping + pool deduction. This is
   what makes a copy CHAIN TERMINATE (each copy costs {R}{R} again; the chain
   stops when a player runs out of untapped red).
6. game/actions/mod.rs resolve_spell_finalize: a non-permanent spell copy
   (`is_token`, would-go-to-graveyard) is sent to EXILE rather than the
   graveyard (CR 707.10c "ceases to exist"; entities are never deleted from the
   store, so Exile is the faithful sink — keeps graveyard-count/flashback honest).
7. game/game_loop/logging.rs: suppress the redundant generic "UnlessCost (if
   paid): NoTargetNeeded" line for the copy case (the real "X copies Chain
   Lightning" line covers it; also stops leaking the internal target-category
   enum into the player log).

## Follow-up (CONCERN, not a blocker)

mtg-884: the pay/no-pay + retarget decisions are made in-engine (deterministic,
information-independent — desync-safe) rather than routed through a
PlayerController. A human client can't decline/redirect. Tracked for a future
game-loop-interactive lift.

## Relationship to Java Forge

Forge-Java's CopySpellAbilityEffect copies the resolving SpellAbility and routes
the optional payment + new-target choice through its PlayerController. This Rust
version copies the spell card (CR 707.10) and lands the copy on the stack the
same way, but currently makes the optional decision deterministically in the
engine (mtg-884) instead of via the controller — a documented divergence chosen
to land the mechanic without the larger interactive-resolution refactor.
