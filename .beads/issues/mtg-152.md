# Description

**Phase 1.1 of mtg-151**: Audit all 185 String usages in mtg-engine/src/ and categorize them.

String does not support the Allocator API, so we need to understand which Strings
are acceptable (UI/errors) vs which should be converted to Arc<str> (hot-path data).

**Parent Issue**: mtg-151 (Allocator API implementation)

## Goals

1. **Inventory**: Categorize all 185 String occurrences
2. **Convert**: Hot-path Strings to Arc<str> if beneficial
3. **Accept**: UI/error Strings that don't affect performance
4. **Document**: Final String usage policy for allocator migration

## Expected Categories

Based on initial analysis:

1. **Arc<str> (Already Optimized)** - ~60%
   - CardName: Arc<str> ✅
   - PlayerName: Arc<str> ✅
   - Subtype: Arc<str> ✅
   - Action: None needed

2. **Temporary/UI Strings (Acceptable)** - ~30%
   - Error messages
   - Debug/logging output
   - TUI display text
   - Action: Accept as-is

3. **Hot-Path Strings (Needs Conversion)** - ~10%
   - Card descriptions (if cloned frequently)
   - Parser intermediate results
   - Controller state strings
   - Action: Audit and convert to Arc<str>

## Search Strategy

```bash
# Find all String usages
cd mtg-engine/src
grep -rn "String" --include="*.rs" | grep -v "// " | grep -v "to_string()" > /tmp/string_usage.txt

# Categorize by context
grep "CardName\|PlayerName\|Subtype" /tmp/string_usage.txt  # Already Arc<str>
grep "Error\|error\|format!" /tmp/string_usage.txt          # Errors (acceptable)
grep "struct\|pub " /tmp/string_usage.txt                    # Struct fields (audit)
```
