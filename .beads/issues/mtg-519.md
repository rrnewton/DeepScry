# Description

Test all behavioral aspects of Mana Drain in MTG Forge-rs.

Card: cardsfolder/m/mana_drain.txt
Set: LEG (Legends)
Deck: thedeck Peterschnidrig (mtg-413, main deck)

Card text:
  Mana Drain {U}{U} Instant
  Counter target spell. At the beginning of your next main phase, add
  an amount of {C} equal to that spell's mana value.

Script:
  A:SP$ Counter | TargetType$ Spell | RememberCounteredCMC$ True
   | ValidTgts$ Card | SubAbility$ DBDelTrig
  SVar:DBDelTrig:DB$ DelayedTrigger | Mode$ Phase | Phase$ Main1,Main2
   | ValidPlayer$ You | Execute$ AddMana | SpellDescription$ ...
   | RememberNumber$ True | SubAbility$ DBCleanup
  SVar:DBCleanup:DB$ Cleanup | ClearRemembered$ True
  SVar:AddMana:DB$ Mana | Produced$ C | Amount$ X
  SVar:X:Count$TriggerRememberAmount

Findings (2026-05-29_#2473(1048d207), compat-oldschool-wave9):

1. [x] Parses as {U}{U} Instant.
2. [x] SP$ Counter (the first half) works end-to-end: targets a spell,
       resolves, counters it, clears remembered cards on resolution.
3. [BROKEN] DB$ DelayedTrigger with Mode$ Phase is unsupported by the
       loader: effect_converter.rs::DelayedTrigger only handles
       Mode$ ChangesZone and logs "DelayedTrigger Mode$ 'Phase' not
       yet implemented" while returning None — the SubAbility chain is
       silently dropped. The "add {C} equal to spell's mana value at
       next main phase" rider never fires.
   Affected pattern (Bug: candidate): any card whose script registers
   a delayed trigger keyed off a phase event rather than a zone
   change. Mana Drain is the most prominent example; others probably
   exist in the LEG set.

Reproducer:

```

[no "produces 1 colorless mana" log line]
```
