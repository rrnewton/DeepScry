# Description

## Bug Description

When cards like Demonic Tutor execute a SearchLibrary effect, the game log shows the wrong player's library being searched.

## Evidence

From game log (old school decks game):
```
[GAMELOG Turn16 M2] Random2 casts Demonic Tutor (107) (putting on stack)
...
[GAMELOG Turn16 M2] Demonic Tutor (107) searches Random1's library for a Card card and puts it into Hand
```

Random2 cast Demonic Tutor but the log says it searches Random1's library.

## Root Cause Analysis

In effect_converter.rs:212-213, SearchLibrary uses PlayerId::new(0) placeholder. During execution in priority.rs:1088, code correctly uses current_priority. But logging.rs:324 still reads from the unresolved placeholder.

## Impact

Logging only - game logic correctly searches caster's library. But confusing for debugging.

## Fix

Pass resolved player ID to logging function, or resolve placeholder before logging.
