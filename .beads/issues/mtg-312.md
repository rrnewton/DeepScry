# Description

Files: mtg-forge-rs/mtg-engine/src/server.rs, mtg-forge-rs/mtg-engine/src/main.rs (or wherever CLI args are parsed)

Action:
1. Add a new CLI flag --trusted-bug-report-password (separate from the game server password)
2. Add a WebSocket message handler for bug reports that:
   a. Receives: user description text, game logs, console logs, optional trusted password
   b. Validates the trusted password if provided
   c. Stores the bug report locally as at least 3 files:
      - user_report.txt (the user's description text)
      - game_logs.txt (game logs from the active game)
      - console_logs.txt (chrome dev console logs)
      Store in a directory like bug_reports/<timestamp>/ or similar
   d. Returns a response to the client indicating success/failure

Why: Core server infrastructure for receiving and storing bug reports before filing them on GitHub.

Verify:
- Server starts with --trusted-bug-report-password flag
- Bug report WebSocket message is handled and files are written to disk
- cargo test passes for any new unit tests
- cargo build succeeds
