# Description

## Network Client Single-Channel Architecture Redesign

## Problem Statement

The current network client implementation has several architectural issues:
- Multiple channels with race conditions (reveal_tx/rx, etc.)
- Controllers contain network-blocking logic (should be pure)
- CardRevealed processing requires `&mut GameState` but controllers only have `&GameStateView`
- Shared `recv_rx` channel between controllers would give each a random subset of messages

## Design Overview

### Core Concept: Pre-Choice Hook + Helper Functions

Replace the current multi-channel architecture with:
1. **Pre-choice hook**: Called before each controller choice, has `&mut GameState`
2. **Helper functions**: Wrap every `controller.choose_X()` call
3. **Thin NetworkLocalController**: Only sends choices to output stream
4. **RemoteController**: Marker/placeholder, methods never called in network mode

### Pre-Choice Hook

```rust
type PreChoiceHook = FnMut(&mut GameState, PlayerId, ChoiceKind) -> PreChoiceResult;

enum ChoiceKind {
    SpellAbility,
    Targets,
    ManaSources,
    Attackers,
    Blockers,
    DamageOrder,
    Discard,
    FromLibrary,
    Sacrifice,
    Modes,
}

enum PreChoiceResult {
    /// Local player: proceed to call controller
    AskController,
    /// Remote player: use this choice (indices from OpponentChoice message)
    UseChoice(RawChoice),
    /// Game ended
    Exit,
}

struct RawChoice {
    indices: Vec<usize>,
    spell_ability: Option<SpellAbility>,  // Only for spell choices
}
```

**Behavior:**
- Blocks on input message stream
- Processes CardRevealed messages (updates GameState) as they arrive
- Stops when it receives choice signal:
  - For local player: ChoiceRequest → returns `AskController`
  - For remote player: OpponentChoice → returns `UseChoice`

### Helper Functions

One helper per choice type, all follow the same pattern:

```rust
fn choose_spell_ability_with_network(
    game_loop: &mut GameLoop,
    controller: &mut dyn PlayerController,
    view: &GameStateView,
    available: &[SpellAbility],
) -> ChoiceResult<Option<SpellAbility>> {
    // Non-network mode: direct call
    if !game_loop.is_network_mode() {
        return controller.choose_spell_ability_to_play(view, available);
    }
    
    // Network mode: call pre-hook first
    let player = controller.player_id();
    let is_remote = controller.get_controller_type() == ControllerType::Remote;
    
    match game_loop.call_pre_choice_hook(player, ChoiceKind::SpellAbility) {
        PreChoiceResult::AskController => {
            assert!(!is_remote, "AskController returned for remote controller");
            controller.choose_spell_ability_to_play(view, available)
        }
        PreChoiceResult::UseChoice(raw) => {
            assert!(is_remote, "UseChoice returned for non-remote controller");
            let choice = convert_indices_to_spell_ability(&raw, available);
            ChoiceResult::Ok(choice)
        }
        PreChoiceResult::Exit => ChoiceResult::ExitGame,
    }
}
```

**Invariants (enforced by assertions):**
- `UseChoice` ↔ remote controller
- `AskController` ↔ local controller

### NetworkLocalController (Simplified)

```rust
struct NetworkLocalController<C: PlayerController> {
    inner: C,
    choice_tx: Sender<ClientMessage>,  // Output stream only
}

impl<C: PlayerController> PlayerController for NetworkLocalController<C> {
    fn choose_spell_ability_to_play(&mut self, view: &GameStateView, available: &[SpellAbility]) 
        -> ChoiceResult<Option<SpellAbility>> 
    {
        // Pre-hook already called by helper, just ask inner controller
        let result = self.inner.choose_spell_ability_to_play(view, available);
        
        // Send choice to output stream
        if let ChoiceResult::Ok(choice) = &result {
            let indices = convert_choice_to_indices(choice, available);
            self.choice_tx.send(ClientMessage::SubmitChoice { 
                choice_indices: indices,
                // ... other fields
            }).ok();
        }
        
        result
    }
    // Similar for other choose_X methods
}
```

No incoming message handling - that's the hook's job.

### Message Flow

```
Input Stream (Server → Client)          Output Stream (Client → Server)
         |                                        ↑
         ↓                                        |
    Pre-Choice Hook                    NetworkLocalController
    (blocks, processes                  (sends choice after
     CardRevealed, returns               inner controller
     when choice signal)                 makes decision)
         |                                        ↑
         ↓                                        |
    Helper Function ─────────────────────────────→
    (calls controller if AskController,
     uses raw choice if UseChoice)
```

### Changes Required

1. **GameLoop additions:**
   - `with_pre_choice_hook()` builder method
   - `is_network_mode()` check
   - `call_pre_choice_hook()` method

2. **New helper functions** (in game_loop module):
   - `choose_spell_ability_with_network()`
   - `choose_targets_with_network()`
   - `choose_mana_sources_with_network()`
   - `choose_attackers_with_network()`
   - `choose_blockers_with_network()`
   - `choose_damage_order_with_network()`
   - `choose_discard_with_network()`
   - `choose_from_library_with_network()`
   - `choose_sacrifice_with_network()`
   - `choose_modes_with_network()`

3. **Replace all `controller.choose_X()` calls** with helper function calls

4. **Simplify controllers:**
   - NetworkLocalController: remove all blocking/message handling, just send choices
   - RemoteController: simplify to marker with panic implementations

5. **Remove old infrastructure:**
   - `SharedMessageReceiver`
   - `pending_reveals` buffer
   - `reveal_drainer` callback (replaced by pre-choice hook)
   - Multi-channel setup in client.rs

6. **client.rs changes:**
   - Create pre-choice hook closure that owns message receiver
   - Set up single input channel from WsReader
   - Set up single output channel to WsWriter
