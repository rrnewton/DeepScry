# Description

MEDIUM guard-rail gap (pre-existing), found in the mtg-677 carve-out desync-review.

PROBLEM: the wasm-gated unit tests under mtg-engine/src/wasm/ (cfg(feature="wasm")) COMPILE but never RUN under `make validate` or CI, because both invoke cargo tests with --features network only. wasm-gated tests need --features wasm-tui (or wasm). In particular the two safety tests guarding the mtg-677 reveal-timing-stable discard carve-out —
  replay_verifier.rs::test_verify_replay_discard_reveal_timing_is_stable (timing-only display diff is NOT fatal)
  replay_verifier.rs::test_verify_replay_discard_stable_key_still_catches_real_divergence (a diverging CardId IS still fatal — proves the carve-out does not blind the detector)
— were verified by hand this session (cargo test --features wasm) but are NOT exercised by the green gate. The same gap hides ALL other wasm-gated unit tests.

FIX: add a wasm-feature unit-test leg to scripts/validate.py AND .github/workflows/ci.yml, e.g.
  cargo nextest run -p mtg-engine --features network,wasm-tui   (or a dedicated --features wasm run)
so the wasm-gated tests actually execute in the gate. Mind the WASM_TARGET_DIR / build-lock separation already set up in the Makefile (mtg-717) and the per-validate cgroup isolation (mtg-743).
