# Description

## Network hand size desync with random controllers

### Problem
Cards drawn and then discarded within the same activated ability (e.g., Bazaar of Baghdad:
"draw 2, discard 3") would be re-added to the player's hand by the reveal processor,
causing a hand size desync detected as a fatal state hash mismatch.

### Root Cause (Fixed 2026-03-26_#1978)
In `reveal_processor.rs`, `process_card_reveal()` for Draw/OpeningHand reveals used an
"empty library mode" check: `!card_in_hand && !card_in_library`. This condition was also
TRUE when a card had been drawn (removed from library) and then discarded (removed from hand
to graveyard). The reveal processor would incorrectly re-add the card to hand.

### Fix
Added zone checks for graveyard, exile, and battlefield before the "empty library mode"
hand insertion. If a card has already moved to another zone, don't re-add it to hand.

Also added `pending_activation_effect_idx` to GameState for WASM effect resumption, ensuring
interrupted ability effects don't re-execute already-completed effects on re-entry.

### Files Modified
- `mtg-engine/src/network/reveal_processor.rs` - zone check fix
- `mtg-engine/src/game/game_loop/priority.rs` - WASM effect resumption
- `mtg-engine/src/game/state.rs` - new field
