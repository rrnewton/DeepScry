== Prioritized backlog (all LOW value — no maindeck breakage) ==

B1 [BROKEN, low value] Magical Hack + Sleight of Mind
  Cards: cardsfolder/m/magical_hack.txt, cardsfolder/s/sleight_of_mind.txt (both in 01_blumke sideboard, 1 copy each)
  Script: A:SP$ ChangeText | ValidTgts$ Card | ChangeTypeWord$/ChangeColorWord$ ...
  Root cause: ApiType ChangeText has NO parser variant in ability_parser.rs (it is not in the ApiType enum at all) -> the spell converts to Effect::Unimplemented. Casting it changes no text.
  Impact: niche text-changing tech (swap a basic land type / a color word). Strictly a sideboard answer card; near-zero metagame value. PARTIAL-toward-BROKEN.
  Fix shape: add ChangeText ApiType + a continuous text-modification effect (replace one basic land type / color word with another, Duration$ Permanent). Bigger feature for tiny payoff; lowest priority.

B2 [BROKEN, low value] Lhurgoyf
  Card: cardsfolder/l/lhurgoyf.txt (04_stern sideboard, 2 copies)
  Script: PT:*/1+* ; S:Mode$ Continuous | CharacteristicDefining$ True | SetPower$ X | SetToughness$ Y (X=creature cards in all graveyards, Y=X+1)
  Root cause: continuous CharacteristicDefining SetPower/SetToughness is NOT in the StaticAbility runtime enum (ModifyPT/GrantKeyword/ReduceCost/RaiseCost/GrantAbility/GainControl/SacrificeMatchingPresent/CantBeCast/CantPlayLand/CantBlockMatching/CastWithFlash). SetPower is only handled inside CopyPermanent. Lhurgoyf never recomputes its P/T from the graveyards; it sits at its base (effectively 0/1).
  Empirical: lhurgoyf_test.pzl (3 creature cards in graveyards -> expected 3/4). The AI declined to attack into an empty board with its only creature — consistent with a 0/1 valuation, not a 3/4.
  Fix shape: implement continuous CharacteristicDefining P/T as a real layer-7b CDA static recomputed from a Count$ expression. THIS IS THE SAME ENGINE GAP as 2000's Serra Avatar (B4) and Opalescence (B5) in backlog mtg-913 — drive it from there (those are high-value combo pieces; Lhurgoyf is a sideboard 2-of that rides along for free once the CDA static exists).

B4 [FIXED wave4] Island Sanctuary
  Card: cardsfolder/i/island_sanctuary.txt (02_hernandez, 2 copies)
  Script: R:Event$ Draw | ActivePhases$ Draw | PlayerTurn$ True | Optional$ True | ReplaceWith$ SanctuaryEffect
  Root cause (confirmed wave3): R:Event$ Draw replacement infrastructure does NOT exist. The parse_static_abilities loop over R: lines only handles Event$ Untap (DoesNotUntap lock), Event$ DamageDone (Torbran + Prismatic Ward), and Event$ Moved (ETB-tapped). There is NO "Draw" event arm. The Sanctuary R: line is silently dropped.
  Fix (wave4, slot34, c8103fe2b, 2026-06-12): 5-part change following Stasis/Orgg precedents:
    1. CardCache::is_island_sanctuary (core/card.rs): new precomputed flag (like skips_untap_step).
    2. CardDefinitionLoader::is_island_sanctuary_card() (loader/card.rs): tokenized structural parse of R:Event$ Draw | ActivePhases$ Draw | PlayerTurn$ True | Optional$ True shape.
    3. Player::island_sanctuary_protected (core/player.rs): new serialized bool, cleared in cleanup_step().
    4. draw_step() (game_loop/steps.rs): when active player controls an island sanctuary card, skip the mandatory draw and set player.island_sanctuary_protected = true. AI simplification: always activates (AI:RemoveDeck:All means only human players use this card). TODO(mtg-917): offer explicit yes/no choice to human controllers.
    5. Attack restriction enforcement: get_available_attacker_creatures() filters non-flying/non-islandwalk; declare_attacker() returns Err for non-evasion creatures attacking a protected player. New has_islandwalk() helper in continuous_effects.rs.
  Tests: test_load_island_sanctuary_cache_flag (card_loading.rs, parser) + test_island_sanctuary_attack_restriction (puzzle_e2e.rs, behavioral: 3 cases — non-evasion blocked ✓, attack works without sanctuary ✓, flying can attack through ✓).
  STATUS: FIXED (WORKING with AI simplification — always activates instead of offering choice).
