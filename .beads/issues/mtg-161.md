# Description

Create par_rewind_play_again benchmark to model future parallel MCTS search.

## Objective
Benchmark parallel game tree exploration by:
1. Playing halfway through a game
2. Creating snapshot
3. Cloning snapshot to N worker threads (pinned to cores)
4. Each thread plays iters/N games with thread-specific RNG seeds
5. Measures allocator contention in parallel setting

## Implementation
- Use rayon or std::thread with core affinity
- Mix thread ID into RNG seed for path diversity
- Aggregate metrics from all threads
- Compare vs sequential rewind_play_again

## Why Important
Models future MCTS where shared allocator contention matters more. Current optimizations may have different impact under parallel load.
