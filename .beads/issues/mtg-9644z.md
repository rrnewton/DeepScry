# Description

## Network Client

Implement the client that connects to server and maintains a shadow game state.

## Tasks

- [ ] Create `ClientGameState` struct (shadow game, known cards, expected hash)
- [ ] Implement WebSocket client connection
- [ ] Handle authentication handshake
- [ ] Process GameStarted: create shadow state with remote libraries
- [ ] Process CardRevealed: add to known_cards, queue in library buffer if draw
- [ ] Process ChoiceRequest: verify options match local computation, verify hash, prompt local controller
- [ ] Process OpponentChoice: apply to shadow state for sync
- [ ] Verify state hash at each choice point
- [ ] Integrate with existing TUI controllers (FancyTui, Interactive, etc.)
- [ ] Add `mtg connect` CLI subcommand with LocalPlayerArgs
- [ ] Handle disconnection gracefully

## CLI

\`\`\`bash
mtg connect deck.dck --server=HOST:17771 --password=SECRET --controller=fancy --name="Alice"
\`\`\`

## Key Design

Client maintains shadow GameState with LibraryMode::Remote. 
On each ChoiceRequest, verify:
1. Options list matches what we computed locally
2. State hash matches (excluding hidden info)
