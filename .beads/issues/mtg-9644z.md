## Implementation

Created `mtg-engine/src/network/client.rs` with:
- `ClientConfig`: connection settings (server, password, player_name, deck_path, cardsfolder)
- `GameStartInfo`: struct to hold game initialization parameters
- `ClientGameState`: shadow game state with remote libraries
- `NetworkClient`: WebSocket client with connect/run_game/disconnect

Key patterns:
- Uses `futures_executor::block_on` for sync CardDefinition lookup in async context
- Uses `LibraryMode::Remote` for both player libraries
- Processes CardRevealed, OpponentChoice, ChoiceRequest messages
- Basic stdin-based choice input (TODO: integrate with TUI controllers)
