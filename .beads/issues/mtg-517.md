# Description

Test all behavioral aspects of Library of Alexandria in MTG Forge-rs for
the 1994 Old School playtest goal.

Card file: cardsfolder/l/library_of_alexandria.txt

Card text:
  no cost  Land
  {T}: Add {C}.
  {T}: Draw a card. Activate only if you have exactly seven cards in hand.

Script:
  A:AB$ Mana | Cost$ T | Produced$ C
  A:AB$ Draw | Cost$ T | PresentZone$ Hand | IsPresent$ Card.YouOwn
             | PresentCompare$ EQ7
             | SpellDescription$ Draw a card. Activate only if you have
                                  exactly seven cards in hand.

Decks containing this card (1994 Old School playtest):
- 06_jeskai_aggro_joseantonioprieto (mtg-561)

== Findings (2026-05-29_#2462(132ce6cc)) ==

1. [x] Parses as a Land with no mana cost.
2. [x] Mana ability {T}: Add {C} parses and works — verified by
   gameplay log "Tap Library of Alexandria for {C}" feeding a Sol Ring
   cast successfully.
3. [BROKEN] Draw ability gating on "exactly seven cards in hand" is
   silently dropped at parse / activation time. The action menu offers
   "activate Library of Alexandria" REGARDLESS of the current hand
   size. Reproduced as follows:

   - Hand=7, untapped: activate appears, draw fires, hand goes to 8
     after the draw step on the next upkeep.
   - Hand=8 (after T2 draw step), untapped: activate STILL appears and
     succeeds. Card draws regardless of hand size. The "exactly seven"
     gate is unenforced.

   Root cause: the parser exposes IsPresent$ + PresentZone$ +
   PresentCompare$ on STATIC abilities (S:) and on cost-reduction
   conditions (see mtg-engine/src/loader/card.rs around line 3849),
   but the ActivatedAbility struct
   (mtg-engine/src/core/effects.rs:2985) has no
   activation_condition / present_filter field. The PresentCompare$
   EQ7 clause is read by the parser and then DROPPED. The
   loader/effect_converter.rs ApiType::Draw arm reads only NumCards$
   and Defined$, ignoring PresentZone$ / IsPresent$ / PresentCompare$.
   PluS the action-listing path in mtg-engine/src/game/game_loop/
   actions.rs (around line 940) checks cost-payability, summoning
   sickness, life cost, sacrifice cost — but has no hook for
   activation-condition predicates.

   Empirical evidence:

     ./target/release/mtg tui \
       decks/old_school/06_jeskai_aggro_joseantonioprieto.dck \
       decks/old_school/06_jeskai_aggro_joseantonioprieto.dck \
       --p1-draw 'Library of Alexandria;Mox Sapphire;Mox Ruby;Mox Pearl;Mox Jet;Black Lotus;Sol Ring' \
       --p2-draw Plains --seed 42 --p1 fixed --p2 fixed \
       --p1-fixed-inputs 'play library of alexandria;pass;pass;pass;pass;pass;pass;pass;activate library of alexandria' \
       --p2-fixed-inputs pass --stop-when-fixed-exhausted -v 3

   With this script the Library is played T1 (hand drops to 6 after
   play), then passes through T2 draw step (hand back to 7) +
   compounded draws → at activation time the hand is 8, and the
   activate still succeeds:

     [7] activate Library of Alexandria
     Library of Alexandria activates ability: Draw a card. Activate
       only if you have exactly seven cards in hand.
     Fixed1 draws Black Vise (59)
     Fixed1 draws Tundra (113)         <-- second player's draw
     Hand: 7 | Library: 52              <-- after activate at hand=8

4. [unverified] Interaction with cards that change hand size mid-turn
   (Wheel of Fortune, Timetwister, discards). Conditional gate is
   never enforced anyway, so this is moot until (3) is fixed.

== Required engine work (separate Bug: issue) ==

To make this WORKING the engine needs:
  (a) ActivatedAbility to carry an optional
      `activation_condition: Option<ActivationCondition>` carrying
      the IsPresent$/PresentZone$/PresentCompare$ tuple parsed by the
      loader.
  (b) The action-listing path in actions.rs to evaluate that condition
      against the current game state and skip the ability when it's
      not met.
  (c) The card_loading / effect_converter pipeline to thread the
      condition into the ActivatedAbility (similar to how the
      ReduceCost path threads `CostReductionCondition`).

This is a generic engine feature (other cards in the future will need
activation-condition gating; Library is just the most prominent
historical example). File as Bug: issue, then revisit this issue.

Reproducer (above) captures the bug; once fixed, expected log
evidence will show "activate Library of Alexandria" disappearing from
the action menu when hand size != 7, and reappearing when it equals 7.

CARD STATUS: BROKEN — mana ability {T}: Add {C} works correctly, but
the gated draw ability ignores its EQ7 hand-size condition, allowing
unconditional draws every untap step. Blocking on the engine gap
described above.
