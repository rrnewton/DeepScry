# Description

## Symptom

Live smoke test 2026-05-28 against https://deepscry.net (integration b03e22cc) finds the landing-page lobby never connects. Browser console:

```
WebSocket connection to 'wss://deepscry.net:8080/' failed: Error in connection establishment: net::ERR_SSL_PROTOCOL_ERROR
```

Port 8080 on the VM is plain TCP (the unified axum HTTP+WS server). It is NOT TLS-fronted by Cloudflare (CF proxies 443->origin), so the browser cannot speak wss:// directly to it. Curl confirms: `https://deepscry.net:8080/` -> SSL wrong-version-number error.

The correct same-origin WS endpoint is `wss://deepscry.net/lobby` (verified: `curl https://deepscry.net/lobby` with WS upgrade headers returns 400, meaning the WS handshake reached the server).

## Root cause

scripts/deploy-cloud.sh line 392 writes:
```
window.MTG_WS_URL = "${ws_scheme}://${PUBLIC_HOST}:${REMOTE_PORT}";
```

It should either:
1. Emit `${ws_scheme}://${PUBLIC_HOST}/lobby` for the same-origin unified deploy, or
2. Leave `window.MTG_WS_URL` unset so the in-page same-origin auto-detect in the committed default server-config.js runs.
