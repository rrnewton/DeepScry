# Description

## deploy-cloud.sh: server-config.js points lobby at wss://host:8080 (broken) — FIXED

Closed 2026-06-01 gardening: FIXED.

Evidence: scripts/deploy-cloud.sh lines 440-448 explicitly state:
  '--- 3. server-config.js: NO LONGER OVERRIDDEN at deploy time. ---
   The committed web/server-config.js self-detects the WS URL from window.location 
   ("wss://" + host + "/lobby") which works for every topology: direct-IP dev, 
   Cloudflare-proxied HTTPS, and any reverse proxy.'

web/server-config.js lines 13-16:
  if (!window.MTG_WS_URL) {
    const proto = window.location.protocol === 'https:' ? 'wss:' : 'ws:';
    window.MTG_WS_URL = proto + '//' + host + '/lobby';
  }

The old hardcoded 'wss://host:8080' generation was removed. The deploy script now echoes:
  '→ using committed web/server-config.js (self-detects ws/wss + /lobby path)'

Original bug: scripts/deploy-cloud.sh line 392 was writing window.MTG_WS_URL = '${ws_scheme}://${PUBLIC_HOST}:${REMOTE_PORT}' which broke when behind Cloudflare (port 8080 is not TLS-fronted). Fixed by removing the override entirely.
