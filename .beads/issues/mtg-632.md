# Description

## Flaky: agentplay determinism tests collide on per-second log filename — FIXED

Closed 2026-06-01 gardening: FIXED.

Root cause (from issue): Tests compared /tmp/mtg_game_<sec>.log, and parallel games ending in the same second would write to the same file, causing cross-contamination.

Fix: test_mode_equivalence.py now extracts the log path from the engine's stderr announcement ('Log saved to <path>') instead of guessing the /tmp path. Evidence: agentplay/test_mode_equivalence.py:482-531:
  'The engine writes a structured game log to a path it announces on stderr ("Log saved to <path>"). We parse that path out of stderr...'
  log_path = Path(match.group(1))
  (tmp_path / f"{log_path.name}").write_text(text, encoding="utf-8")

Each run gets its own unique path from the engine. The test no longer picks up a log from a concurrently-running game.

Timestamp format: main.rs:72 uses YYYYMMDD_HHMMSS (second precision), still theoretically collide-able but the test's path-from-stderr pattern makes the test correct regardless.
