# Description

The new systemd-based deploy leaves behind legacy log files from previous deploy iterations on the remote:

  ~/$REMOTE_DIR/server.log         (old python http.server)
  ~/$REMOTE_DIR/rust-server.log    (old standalone `mtg server`)
  ~/$REMOTE_DIR/deepscry.log       (ad-hoc tmux `mtg server-web` testing)
  ~/$REMOTE_DIR/deepscry-tls.log   (ad-hoc tmux TLS testing)

The current axum-via-systemd-user service logs to journalctl, not files. These leftovers waste disk + confuse readers.

`scripts/deploy-cloud.sh config` already kills legacy tmux sessions (mtg-server, mtg-rust-server, deepscry). Extend the same step to delete these stale .log files. Make it idempotent (rm -f). Optionally also remove any older legacy paths if discovered.

Note: keep `~/mtg-forge-rs/bug_reports/` and `~/mtg-forge-rs/cardsfolder/` intact — those are real data, not logs.
