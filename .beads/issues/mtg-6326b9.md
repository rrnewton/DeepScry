# Description

Part of: mtg-dba689

Improve the 2D card rendering to use available space more efficiently with intelligent layout and compaction.

## Current Problem

Cards truncate names too aggressively:
```
┌────────┐ ┌────────┐
│ Mou... │ │ Mou... │
│[TAPPED]│ │[TAPPED]│
│        │ │        │
│        │ │        │
│        │ │        │
└────────┘ └────────┘
```

Given a W×H character space, we're not using it efficiently.

## Target Behavior

Implement an intelligent layout system with progressive compaction:

1. **Full content definition:** Define all desired content elements
2. **Progressive compaction:** If content doesn't fit, compact elements one-by-one
3. **Compaction priority order:**
   - TAPPED marker: `[TAPPED]` → `[T]` (compact mode)
   - Power/Toughness: Always show in bottom-right (right-indented), no compact mode
   - Card name: Compact last, scalable truncation down to min 5 chars: "Mou.."

## Layout Requirements

- **Text wrapping:** Wrap all lines to fit full card width
- **P/T positioning:** Bottom-right, right-indented (e.g., `      2/2`)
- **Tapped marker:** Essential, shown as `[TAPPED]` or `[T]` when space is short
- **Name truncation:** Use ".." (two dots) for truncated names, min 5 chars

## Example Improved Layout

```
┌────────┐ ┌────────┐
│Mountain│ │Grizzly │
│  [T]   │ │ Bears  │
│        │ │        │
│        │ │        │
│        │ │        │
│      0 │ │    2/2 │
└────────┘ └────────┘
```

## Implementation

Location: `src/game/fancy_tui_controller.rs:render_card_box` (lines 656-724)

Strategy:
1. Calculate available height/width (area minus borders)
2. Build content list with priority/compact modes
3. Measure total height needed
4. If doesn't fit, apply compaction in priority order
5. Render with proper alignment (P/T right-aligned)
