# Description

Part of: mtg-dba689

Improve the 2D card rendering to use available space more efficiently with intelligent layout and compaction.

## Status

✅ **COMPLETED** (2025-11-04)

## Problem Solved

Cards were truncating names too aggressively, showing "Mou..." instead of "Mountain" even when vertical space was available. The TAPPED indicator also used excessive horizontal space on narrow cards.

## Implementation

### 1. Progressive Compaction for TAPPED Marker

Changed threshold for compact display:
- `[TAPPED]` (8 chars): Only when content_width >= 12 (was >= 8)
- `[T]` (3 chars): When content_width >= 3
- `T` (1 char): Ultra-compact for very narrow cards (< 3 width)

This frees up 5 extra characters on narrow cards for better name display.

### 2. Intelligent Name Layout Strategy

Implemented multi-strategy layout in `render_card_box()`:

**Strategy 1**: If name + cost fit on one line → use one line (ideal)

**Strategy 2**: If name would be truncated BUT we have vertical space → use two lines (name on line 1, cost on line 2)

**Strategy 3**: If name fits alone and cost doesn't fit on same line AND we have vertical space → use two lines  

**Strategy 4**: Fallback → single line with minimal truncation

This maximizes use of vertical space to avoid truncating card names unnecessarily.

### 3. Better Truncation Logic

- Uses ".." suffix for truncation (standard convention)
- Minimum 5 chars before truncation
- Only truncates when absolutely necessary (no vertical space available)

## Code Changes

File: `src/game/fancy_tui_controller.rs`
