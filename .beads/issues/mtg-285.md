# Description

Context:
- Date: 2026-04-03
- Decks: decks/old_school/01_rogue_rogerbrand.dck vs decks/old_school2/the_deck_classic.dck
- Mode: fixed-vs-zero, seed 42

Steps to Reproduce:
1. Run: ./target/release/mtg tui --p1=fixed --p2=zero --p1-fixed-inputs="play Bazaar of Baghdad;activate Bazaar of Baghdad" --p2-fixed-inputs="" --stop-on-choice=10 --json --log-tail=80 --seed 42 --verbosity 3 decks/old_school/01_rogue_rogerbrand.dck decks/old_school2/the_deck_classic.dck
2. Observe that Bazaar of Baghdad's ability draws 2 cards but does NOT discard 3 cards

Expected Behavior:
- Bazaar activation: draw 2, then discard 3 (net hand -1)
- Per MTG CR 608.2c: effects resolve sequentially as part of the same ability

Actual Behavior:
- Only Draw resolves. SubAbility$ Discard chain is silently dropped.
- Hand increases by +2 instead of -1

Root Cause:
- mtg-engine/src/loader/card.rs: parse_activated_abilities() (~line 2360) calls params_to_effect_with_svars() which returns a single effect
- It does NOT call follow_sub_ability_chain() to resolve SubAbility$ references
- Spell parsing (line ~1461) DOES call follow_sub_ability_chain(), so spells work correctly
- Fix: add follow_sub_ability_chain() call after primary effect extraction in parse_activated_abilities()

This affects ALL activated abilities with SubAbility$ chains, not just Bazaar.

Evidence:
- Hand starts at 6 after playing Bazaar. After activation: hand = 8 (drew 2, no discard). Should be 5.
- Graveyard has 1 card (cleanup discard), should have 3 from Bazaar ability.
