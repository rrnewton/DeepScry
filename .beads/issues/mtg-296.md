# Description

## Damage log shows incorrect life total (double subtraction) — FIXED

Closed 2026-06-01 gardening: FIXED.

Evidence: mtg-engine/src/game/game_loop/logging.rs lines 180-188:
  // This logging hook runs AFTER resolve_spell has already applied
  // the damage, so the player's stored life is the post-damage
  // total. Do NOT subtract amount again (that double-counted,
  // e.g. Lightning Bolt / Chain Lightning at a 20-life player
  // logged '(life: 14)' instead of 17).
  let life_after = self.game.get_player(*target_player_id).map(|p| p.life).unwrap_or(0);

The fix reads the stored life AFTER damage application, not life_before - amount. Fixed as part of the Chain Lightning compat work (mtg-489 describes the fix on this branch). Tests: chain_lightning e2e + lightning_bolt_targets_opponent_player_e2e verify correct life totals.
