# Description

## Description

When multiple copies of the same card (e.g., 3 Swamps) are in play, display them as a visually stacked group rather than individually.

## Visual Design

- Draw cards stacked from upper-left to lower-right
- Only the topmost card shows full content  
- Other copies show as border outlines behind
- When N cards are tapped, the TOP N appear tapped with untapped ones visible behind

## Implementation Challenges

- More complex geometry calculations
- Need to track card stacks vs individual cards
- Hit testing for mouse clicks becomes more complex
- Layout algorithm must account for stack depth
- Offset calculation for border outlines

## Benefits

- More compact battlefield representation
- Easier to see total count of each card type
- Visually similar to physical MTG gameplay
- Reduces horizontal space usage
