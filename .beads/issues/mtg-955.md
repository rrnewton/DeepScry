# Description

Found during mtg-948 Part B card-audit puzzle backfill (claude/card-audit-wave1).

## Summary
The puzzle assertion DSL predicate `trigger fired from <CardName>` (LogEvent::TriggerFired) was only emitted by the engine for a NARROW set of death/leaves-battlefield watcher triggers. It was NOT emitted for other trigger kinds that DO fire correctly in-engine, so puzzles could not assert those triggers via the event log even though the underlying card behavior was correct.

## RESOLVED (claude/widen-trigger-events, mtg-955)
TriggerFired emission was widened to the TWO shared trigger-firing chokepoints that previously emitted nothing, so it now covers ALL trigger kinds uniformly (in addition to the pre-existing death/LTB/watcher sites):

1. `check_triggers_inner` (mtg-engine/src/game/actions/mod.rs) — the shared firing path for EVERY event-based trigger: DamageDone, DealsCombatDamage, EntersBattlefield, Sacrificed, landfall, spell-cast-derived, etc. One TriggerFired is emitted per fired trigger at the top of the Phase-3 execute loop, carrying the source permanent's name + the trigger description. This covers Hypnotic Specter (DamageDone discard).
2. `check_triggers_for_controller` — the shared firing path for phase/upkeep triggers (BeginningOfUpkeep, AttackersDeclared, etc.). The trigger-match predicate was extracted into a free fn `phase_trigger_matches` (shared by the emission pass and the effect-collection pass so they never drift), and one TriggerFired is emitted per matched trigger. This covers Serendib Efreet (beginning-of-upkeep self-damage).

The pre-existing death-trigger emissions in `check_death_triggers` are unchanged.

## Determinism (observability only — no gameplay change)
push_event is a no-op when the event log is disabled, and the logger's `event_log` Vec is EXCLUDED from GameLogger's custom Serialize impl, so emitting these events cannot perturb game state, the undo log, or any snapshot / network-shadow / WASM rewind hash. Verified:
- native determinism sweep: 480/480 PASS (rewind/replay bit-identical).
- native-vs-WASM equivalence sweep: 31/32 PASS; the single divergence (seed=6 deck=03_robots @#119) is PRE-EXISTING — it reproduces identically on the base commit WITHOUT this change.
- puzzle-golden-check: PASS with NO re-bless needed — the change adds NO human-readable game-log line (only structured events), so all goldens are byte-identical.
