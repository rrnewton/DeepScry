# Description

Test all behavioral aspects of Chain Lightning in MTG Forge-rs (1994 Old School; decks mtg-561, mtg-562).

Card file: cardsfolder/c/chain_lightning.txt
Card text:
  {R} Sorcery. Chain Lightning deals 3 damage to any target. Then that player
  or that permanent's controller may pay {R}{R}. If the player does, they may
  copy this spell and may choose a new target for that copy.

Script:
  A:SP$ DealDamage | ValidTgts$ Any | NumDmg$ 3 | SubAbility$ DBCopy1
  SVar:DBCopy1:DB$ CopySpellAbility | Defined$ Parent | Controller$ TargetedOrController | UnlessPayer$ TargetedOrController | UnlessCost$ R R | MayChooseTarget$ True

== Findings (2026-05-31_#2549(221c7867), compat-finish-jeskai) ==

1. [x] Parses as {R} Sorcery; SP$ DealDamage(3) + chained DB$ CopySpellAbility
   (Defined$ Parent). Parser-shape unit test added:
   test_card_compat_chain_lightning (mtg-engine/src/game/actions/tests/effects.rs)
   — asserts DealDamage{amount:3} + CopySpellAbility{defined_source:Parent}.
2. [WORKING] PRIMARY mode: deals exactly 3 damage to any target. NEW e2e puzzle
   test_chain_lightning_deals_three_to_player (mtg-engine/tests/puzzle_e2e.rs):
   Player 2 20 -> 17. Gamelog: 'Chain Lightning (3) deals 3 damage to Player 2 (life: 17)'.
3. [FIXED this branch] The misleading sentinel gamelog 'Chain Lightning copies
   spell (may choose new targets)' is SUPPRESSED. Root cause: logging.rs
   log_effect_execution unconditionally logged 'copies spell' for
   Effect::CopySpellAbility regardless of defined_source. Now only the
   IMPLEMENTED CopySpellSource::TriggeredSpellAbility path (Jeong Jeong-style
   delayed SpellCast copy) logs; the unimplemented Defined$ Parent path is
   silent (compatibility_tracking SKILL §2.2). The e2e test asserts no
   'copies spell' line leaks.
4. [FIXED this branch] Gamelog life total for non-combat DealDamage at a player
   was double-subtracting (logged '(life: 14)' for Bolt/Chain Lightning at a
   20-life player instead of 17). log_effect_execution runs AFTER resolve_spell
   has already applied the damage, so the stored life is already the post-damage
   total; the Player / None / DealDamageDivided arms no longer subtract `amount`
   again. Generalizes to ALL direct-damage-to-player burn (Lightning Bolt,
   Psionic Blast, Fireball-at-player). docs/EFFECT_SUPPORT.md row added.
5. [BLOCKED] The optional 'pay {R}{R}, then copy this spell and may re-target'
   sub-ability (DB$ CopySpellAbility | Defined$ Parent) is NOT implemented:
   ENGINE GAP mtg-152 (still open). The copy is never put on the stack; an
   INFO-level developer log notes 'Copy not yet implemented - see mtg-152'
   (env_logger only, NOT a player-facing gamelog). Implementing it needs
   spell-stack copy infrastructure (clone resolving spell + new controller +
   optional re-target) which is OUT OF SCOPE for the compatibility fence
   (touches resolve_spell / stack). Tracked under mtg-152.

CARD STATUS: PARTIAL — primary 3-damage burn fully WORKING (correct life log,
no sentinel); the optional pay-RR spell-copy chain is a documented hard blocker
(engine gap mtg-152). This is the only non-WORKING aspect remaining for the
mtg-561 Jeskai Aggro deck.
