# Description

GAMEPLAY BUG: Turn headers missing from tui_game.html log; P1 actions appear during P2's turn / 'Turn 11 skipped'.

ROOT CAUSE (found 2026-06-02, branch fix-log-rendering): the fancy-TUI wrapped-log cache (LogWrapCache, mtg-engine/src/game/fancy_tui_renderer.rs) invalidated ONLY on width/perspective change (needs_rebuild) or append-only growth (needs_update: processed_count < total). It had NO signal for a rewind/undo TRUNCATING the log buffer (logger.truncate_to in rewind_to_turn_start) and replay re-growing it. When the replayed log length lands at or past the old processed_count, the count-only check skips the change -> stale wrapped lines (old turn headers retained, replayed ones missing / misordered -> 'Turn N skipped', P1 actions shown during P2's turn). Turn-navigation also keys off the stale cache (lines scanning '>>> Turn'), compounding it. The UNWRAPPED log path rebuilds from the live buffer every frame and was never affected; only the cached wrapped (default) path.

FIX (commit 54e84c31): GameLogger now has a monotonic log_epoch bumped on every real truncate_to + clear_logs. LogWrapCache stores the epoch it was built against; draw_log_view_wrapped forces a full rebuild when the epoch changed (epoch_stale), in addition to the existing width/perspective + growth checks. Epoch (not a mere shrink check) is required because truncate+replay can re-grow the buffer to an identical count, invisible to count-only logic.

TESTS: game::logger::tests::test_log_epoch_bumps_on_truncate_and_clear + game::fancy_tui_renderer::tests::log_wrap_cache_rebuilds_after_truncate_regrow (both pass). Shares root with mtg-570 (network TUI dup banners). Pending full make validate + push; not self-merged.
