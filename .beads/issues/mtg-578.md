# Description

PROFILE (2026-05-28): two binaries are ~80%+ of test time.
- determinism_e2e: 65 decks, each runs the game TWICE. Standalone on a QUIET 8-core box ~10s/test; under full nextest validate it inflates to ~180-200s/test (~18x) from core oversubscription (each test spawns full-game subprocesses; 65+27 game-tests in parallel thrash).
- shell_script_tests: 27 tests/*.sh full games; SEVERAL rebuild the release binary mid-test (~1046s serial measured).
CI Test = ~125min because ci.yml (~L63) runs `cargo test --verbose --workspace --features network` SERIALLY on a 2-vCPU runner (+ cold-cache rebuilds on Cargo.lock change).

PLAN — PRONG B (split CI, highest leverage): break the single serial Test job into PARALLEL GitHub jobs (separate runners): test-determinism (cargo nextest --test determinism_e2e), test-shell (the 27 .sh), test-unit (rest, fast). Wall = max() not sum(). Use a LARGER runner (4-8 vCPU) + nextest for the heavy jobs. Target 125min -> ~15-25min. (Naive nextest-on-2vCPU was WORSE — split + bigger runner is the fix.)
PRONG A (tent poles): (1) shell_script_tests reuse the pre-built binary (don't rebuild mid-test); (2) cap nextest test-threads / use test-groups for game-subprocess tests (kills the 18x inflation); (3) optional determinism SUBSET on PR + full matrix nightly.
