# Description

PROFILE (2026-05-28): two binaries are ~80%+ of test time.
- determinism_e2e: 66 tests, each runs the game TWICE. ~10s/test standalone; inflates ~18x under full parallel nextest from core oversubscription.
- shell_script_tests: 28 tests/*.sh full games; each script's ensure_mtg_binary() ran a fresh cargo build (~1046s serial measured).
CI Test = ~125min because ci.yml ran cargo test --verbose --workspace --features network SERIALLY on a 2-vCPU runner.

IMPLEMENTED (branch ci-split-test-jobs):

PRONG B — split CI Test job into 3 PARALLEL GitHub jobs (separate runners) in .github/workflows/ci.yml. Partition verified via 'cargo nextest list -E ...' with NO gaps/overlap (1230 total = 66 + 28 + 1136):
- test-determinism: cargo nextest run -E 'binary(determinism_e2e)' (66). Also carries the explicit commander_e2e + snapshot_resume_e2e steps (full-game determinism scripts).
- test-shell: cargo nextest run -E 'binary(shell_script_tests)' --test-threads=1 (28). Serial because cross-test parallelism oversubscribes the 2 cores for subprocess-heavy full-game scripts.
- test-unit: cargo nextest run -E 'not binary(determinism_e2e) and not binary(shell_script_tests)' (1136) — exact complement, no gaps. Also carries run_examples.sh, agentplay pytest + agent_game, and the mode-equivalence orchestrator (all fast, were in the old Test job).
Each job: checkout(submodules) + Install Rust + same cargo cache keys + free-disk + build release binary (network) + targeted command. Wall = max() not sum().

PRONG A tent-pole (1): shell_script_tests REUSE the pre-built binary. tests/lib/test_helpers.sh::ensure_mtg_binary() now short-circuits (skip cargo build) when MTG_REUSE_PREBUILT=1 AND target/release/mtg exists. CI sets MTG_REUSE_PREBUILT=1 on the shell/determinism-script steps after the one-time 'Build release binary' step. Local 'make validate' does NOT set the flag, so it keeps always-fresh rebuild behaviour. (network_vs_local_equivalence_e2e.sh already reused the prebuilt binary; cycle_ability_network_sync_e2e.sh delegates to it; heuristic_royal_assassin_e2e.sh is a no-op exit 0.)
