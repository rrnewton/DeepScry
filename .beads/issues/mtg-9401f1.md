# Description

## Problem (user-reported)

In the native card-style web game (web/native_game.html), watching a
random-vs-random (both-AI) game, the Actions pane went SILENT: the engine's
semantic prompt is "Game ready. Press Space to advance turn." /
"Network AI game running..." / "Waiting for server..." with an EMPTY choice
list, so the pane only showed "No actions available". The sole hint was a tiny
bottom status-bar line and a controls panel hidden until 'C'. In manual-choice
modes the pane shows a prompt + choices, so only the AI-watch / waiting states
were affected. Users want the Actions pane to ALWAYS tell them the next move.

## Fix

Added a persistent affordance footer (#actions-affordance) to the Actions pane,
implemented purely in the RENDERER layer (web/native_game.html) — NOT in the
engine prompt strings (fancy_tui.rs / gui_view_model.rs). Rationale: Space/A are
HTML-page keybindings; embedding them in the shared engine current_prompt would
couple the view-model to one renderer's keys and collide with concurrent engine
work (netarch / refactor slots).

The footer is driven off REAL state (no prompt string-matching):
- game_over OR a human choice is pending (choices.length > 0) -> hidden (the
  choice list is the guidance; nothing left at game over).
- otherwise (watch/auto-advance/waiting; choices empty) -> shown:
  - auto-running    -> "⏸ Auto-running  [A] pause"  (green treatment)
  - paused, network -> "⏳ Waiting for the other player  [Space] continue · [A] auto-run"
  - paused, local   -> "▶ [Space] continue · [A] auto-run"

## DRY / single source of truth

Centralized the auto-run toggle into applyAutoRunState(on, toggleWasm, kickLoop).
Previously the 'A' keydown handler, #btn-auto click handler, and
syncInitialRunState each duplicated the same set-flag/derive-pause/flip-WASM/
relabel-button/kick-loop dance — the drift risk that made the affordance hard to
keep accurate. The affordance, button label, and pause gate now share one flag.
Folded `autoRunning` into the updateUI render-skip key (mtg-6vzht) so a toggle
re-renders the footer even when the view-model JSON is byte-identical.

Network launch behaviour preserved exactly: syncInitialRunState uses
kickLoop=false so it does NOT add a second startAutoRun driver to the network
path (which advances via onMessageProcessed); the per-mode launch code owns the
initial kick.

## Tests

New focused Playwright e2e web/test_action_affordance.js, wired into
scripts/validate.py's WASM browser suite (now 17 tests). Asserts: pane not
silent in paused AI-watch state; footer names Space + auto-run; stays visible
across 12 turn advances (the core complaint); toggling 'A' flips footer + button
to auto-running and back; hidden at game over; no panics. All pass.
test_render_skip.js (render-key change) and test_game_gui_rebuild.js
(auto-run/Space) still green.
