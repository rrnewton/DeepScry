# Description

Found during the 2026-06-03 deploy of main 94c868b5 to deepscry.net. Live `curl -skI https://deepscry.net:8080/data/sets/index.json` returns `cache-control: public, max-age=60`, but the deploy-release SKILL + CAS caching model specify index.json (the logical->hashed asset RESOLVER) must be NO-CACHE. With a 60s TTL, a returning visitor within 60s of a deploy can use a STALE index.json -> resolve to OLD hashed asset names that the rsync --delete removed -> 404 until the cache expires. Investigate: is this an intentional recent change (e.g. mtg-720 CAS-manifest resolution work) or a regression in the server's cache-control for index.json? Fix to no-cache (or document why max-age=60 is safe). Also: the deploy-cloud.sh post-deploy probe printed only '✓ / 200' — confirm whether it actually asserts the index.json no-cache + hashed-asset immutability the skill claims, since it passed despite this deviation.
