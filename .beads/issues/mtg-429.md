# Description

## Purpose

Persistent tracker for the network fuzz-and-fix cycle. Records each fuzz run (date, commit, mode-by-mode pass rates) and links every NEW bug discovered by `bug_finding/network_fuzz_test.py` to its dedicated minibeads issue. Use this issue as the single entry point when re-running fuzz, comparing pass rates over time, or triaging follow-up work.

## Scope

- ONLY tracks failures surfaced by the network fuzz harness (state-hash desync, FATAL choice mismatches, gamelog drift, test-infra races).
- Each fuzz run gets its own dated section under "History".
- Per-bug issues are filed separately and back-linked here.

## History — 2026-06-09_#3049(e0c33784) (branch claude/netarch-desync-sweep, netarch desync sweep)

### Native determinism sweep (fuzz.py determinism)
- Corpus: decks/old_school/*.dck + decks/old_school2/*.dck (chain pairing); controllers heuristic+random+zero; ~5760 configs.
- Result: **5743 PASS / 17 FAIL — ZERO gamelog-divergence desyncs.** All 17 FAIL are `local_exit_1_1` = deterministic IDENTICAL crashes (both runs of the seed crash the same way), NOT desyncs. Dominant crash class: a Maze of Ith + Basalt Monolith mutual-untap loop hitting `MAX_ACTIONS_PER_PRIORITY (1000)` under the `zero` controller (e.g. seed 920324, mono_black_control vs power_monolith_combo). This is a gameplay/AI-loop bug, out of netarch scope — file separately if pursued.
- Conclusion: native same-seed determinism is SOLID.

### WASM-vs-WASM network sweep (fuzz.py network --client wasm, random)
- This is the live-bug class (rewind/replay shadow). Two desyncs found:
  1. **mtg-212 (FIXED, e0c33784):** opening-hand reveal stamped at action_count 0 was dropped by the shadow's `*ac > last_applied_reveal_ac` apply filter (cursor init 0). The first-drawn card's identity stayed `???`; if it was a land the shadow omitted its PlayLand → "Local abilities (4) != server abilities (5)" FATAL at the first choice. Fixed in BOTH wasm + native clients via an `any_reveal_applied` flag.
  2. **mtg-885 (OPEN, NEW):** after the mtg-212 fix the same game runs to Turn 1 EndCombat (ac 66) and then hits a P2 state-hash mismatch (server=92c4ed0e... client=84c90bf6...). Structural state is byte-identical in the summary; the divergent field is in turn-structure/priority/combat/rng bookkeeping at the EndCombat boundary. This was previously MASKED by bug #1. Likely the known-hard WASM-rewind class. Repro logs in debug/netarch_fuzz/repro_p2_statehash_ac66/.

### Pass-rate delta
- WASM-vs-WASM network previously desynced at the OPENING HAND (ac ~45, choice 1). Post-fix it advances ~20 more choices (to ac 66) before the next (separate) desync. Net: the opening-hand class is eliminated; mtg-885 is the new frontier.
