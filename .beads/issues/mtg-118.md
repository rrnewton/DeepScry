# Description

## Summary

Tournament testing with old_school decks revealed a **228 error** count across tournament games when running Heuristic vs Random controller games.

**Major Fix (2025-11-03)**: The most common error (90/228 = 39.5%) has been FIXED! See "Fixed Issues" section below.

## Error Categories

### 1. ✅ FIXED: Infinite Loop with Activated Abilities (39.5% of errors)
**Error**: "Priority round exceeded max actions (1000), possible infinite loop"  
**Count**: 90 errors (39.5% of all errors)  
**Status**: ✅ **FIXED in commit c144036**

**Root Cause**: Activated abilities with mana costs (like Shivan Dragon's {R}: +1/+0) didn't auto-tap lands before paying costs. The mana engine said "you can pay this" but the payment failed because mana wasn't in the pool.

**Fix**: Added auto-tap logic before paying ability costs, mirroring spell casting behavior.

**Verified Fixed**: 
- Original reproducer (seed 9067726178950943642) ✓
- Error 3 reproducer (seed 1879725853889203507) ✓
- Error 10 reproducer (seed 10965042154540734908) ✓

### 2. Counterspell Targeting Errors (13.2% of errors)
**Error**: "Cannot counter a spell that is not on the stack"  
**Count**: 30 errors (13.2% of all errors)  
**Status**: ⚠️ NOT YET FIXED

This indicates bugs in counterspell timing/targeting logic - the game is trying to counter spells that have already resolved or were never on the stack.
