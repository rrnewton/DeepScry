# Description

## Network desync: choose_card_to_discard skips uninstantiated cards in shadow game

## Root Cause

GameState::choose_card_to_discard() in mtg-engine/src/game/actions/mod.rs uses
self.cards.get(card_id).ok()? which returns None for uninstantiated (reserved but
not-yet-revealed) cards. This causes the AI heuristic to skip these cards when computing
discard priorities.

## Fix (2026-03-26)

Two fixes applied:

### 1. SubAbility chain following for activated abilities
`parse_activated_abilities()` in `card.rs` was NOT following SubAbility$ chains, so
Bazaar of Baghdad's Draw ability would parse but the chained Discard SubAbility was
silently dropped. Fixed by calling `follow_sub_ability_chain()` after parsing the
primary effect.

### 2. Controller-routed discard choices for network safety
Effect-driven discard choices (DiscardCards, Loot) are now routed through the
controller protocol via `choose_discard_with_hook()` instead of using the internal
`choose_card_to_discard()` heuristic. This ensures:
- In network mode: server's choice is authoritative (sent via ChoiceRequest)
- In local mode: controller's `choose_cards_to_discard()` is used
- Reveals are pushed between draw and discard effects so shadow state can see drawn cards

Applied in both:
- Activated ability effects loop (priority.rs)
- Spell resolution path (resolve_top_spell_with_discard_hook in priority.rs)

Also refactored `resolve_spell` into `resolve_spell_execute_effects` + `resolve_spell_finalize`
+ `resolve_spell_collect_effects` to support effect-by-effect interception during spell
resolution.
