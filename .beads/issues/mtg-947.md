# Description

Puzzle DSL v2: self-checking, isolated, feature-gated, zero-overhead-when-off puzzle tests built on the existing game log.

SCOPE / FEATURES:
- Event-backed assertions driven off the structured game log: trigger fired / spell cast / creature died / life gained, etc.
- Final-state assertions (board, zones, life, counters) at end of scripted play.
- Golden-log oracle: capture/compare canonical game logs; refresh via `make puzzle-bless`.
- Bulk parallel runner: execute all .pzl files in parallel for CI.
- Anti-overfitting semantic controller commands so puzzles assert intent, not brittle exact transcripts.

ACCEPTANCE BAR (user-set): an INDEPENDENT reviewer testifies that the implementation and the guide match in every respect.
