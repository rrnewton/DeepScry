# Description

Puzzle DSL v2: self-checking, isolated, feature-gated, zero-overhead-when-off puzzle tests built on the existing game log.

SCOPE / FEATURES:
- Event-backed assertions driven off the structured game log: trigger fired / spell cast / creature died / life gained, etc.
- Final-state assertions (board, zones, life, counters) at end of scripted play.
- Golden-log oracle: capture/compare canonical game logs; refresh via `make puzzle-bless`.
- Bulk parallel runner: execute all .pzl files in parallel for CI.
- Anti-overfitting semantic controller commands so puzzles assert intent, not brittle exact transcripts.

ACCEPTANCE BAR (user-set): an INDEPENDENT reviewer testifies that the implementation and the guide match in every respect.

## 2026-06-14_#3428 — Action scripts land (claude/puzzle-action-script, task #16)

The bulk runner used to drive BOTH seats with the heuristic AI, so puzzles could only assert passive/combat-keyword cards — the AI will not reliably cast a spell at a chosen target on cue. ACTION SCRIPTS fix this: a .pzl may now carry optional [p0_script] / [p1_script] sections. Each line is one semantic command in the EXISTING rich-input/fixed-input vocabulary (cast/target/activate/attack/pass/PASS_UNTIL/*); a scripted player is driven by a RichInputController instead of the AI. DRY: no new parser/controller — reuses RichInputController wholesale via a single shared PuzzleFile::build_controller() wiring point used by the bulk runner, the assert-e2e harness, and the golden-check.

Anti-overfitting: added an inline 'cast <card> targeting <selector>' clause to RichInputController. A standalone 'target' line strands+errors when the engine auto-targets a single-legal-target spell (CR 601.2c forced choice, no prompt); the inline clause is consumed by choose_targets when asked and is a harmless no-op when the target was forced. Selectors use the prefix/case/space-insensitive card matcher or a pN player sentinel — never raw menu indices.

Zero-overhead-when-absent: a puzzle with no script sections yields p0_script=p1_script=None and behaves EXACTLY as before (build_controller returns HeuristicController). has_no_scripts()/script_for() let runners branch with no allocation.

Determinism: a script issues only the same public commands a human types and matches only against the engine's offered options — no hidden info (opponent hand, library order, RNG). Identical decisions on server/client/native/WASM. RichInputController gained a serialized pending_target field (snapshot round-trips).

Showcase: test_puzzles/script_lightning_bolt_kills_creature.pzl — P0 scripted to bolt P1's Grizzly Bears. Assertions now PASS in the bulk runner: 'spell cast Lightning Bolt', 'creature died Grizzly Bears', 'opponent graveyard contains Grizzly Bears'. Golden log committed (shows 'casts Lightning Bolt -> targeting Grizzly Bears' / 'deals 3 damage' / 'dies from lethal damage'). Bulk runner assert-pass count 15 -> 16; the demo was a PANIC before the targeting fix.

Docs: docs/FIXED_INPUT_SYNTAX.md (targeting clause) + docs/guide/src/part3/puzzle_language.md (new 'Action scripts' section). Unit tests: format section parse, no-script default, split_targeting_clause.
