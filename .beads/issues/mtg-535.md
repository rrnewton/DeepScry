# Description

Test all behavioral aspects of Recall in MTG Forge-rs.

Card: cardsfolder/r/recall.txt
Set: ARN (Arabian Nights)
Deck: thedeck Peterschnidrig (mtg-413, main deck)

Card text:
  Recall {X}{X}{U} Sorcery
  Discard X cards, then return a card from your graveyard to your hand for each card discarded this way. Exile Recall.

Script:
  A:SP$ Discard | Defined$ You | Mode$ TgtChoose | NumCards$ X
   | RememberDiscarded$ True | SubAbility$ DBChangeZone
  SVar:DBChangeZone:DB$ ChangeZone | Hidden$ True | Mandatory$ True
   | ChangeType$ Card.YouOwn | ChangeNum$ Y | Origin$ Graveyard
   | Destination$ Hand | SubAbility$ DBExile
  SVar:DBExile:DB$ ChangeZone | Origin$ Stack | Destination$ Exile
   | SubAbility$ DBCleanup
  SVar:X:Count$xPaid
  SVar:Y:Remembered$Amount

## Fix (compat-thedeck)

Root cause: the `ChangeZone | Origin$ Graveyard | Destination$ Hand` converter
branch matched this sub-ability as `MoveSelfBetweenZones` (self-return), since
the condition was `Origin$ Graveyard && Defined$.is_none()`. But Recall's
DBChangeZone has `ChangeNum$ Y` (= Remembered$Amount), not self-return semantics.

Fix: added a new `Effect::ReturnCardsFromGraveyardToHand { player: PlayerId }`
variant that reads `game.remembered_cards.len()` at resolution to determine
how many cards to return (= cards actually discarded). The converter now matches
`Origin$ Graveyard | Destination$ Hand | ChangeNum$` BEFORE the self-return
branch, emitting the new effect.

The AI picks cards from the graveyard in lowest-CardId order for determinism
(network-safe; graveyard is public).

Parser shape verified by `test_card_compat_recall` (in
`mtg-engine/src/game/actions/tests/effects.rs`):
  [x] {X}{X}{U} Sorcery
  [x] DiscardCardsXPaid with remember_discarded=true
  [x] ReturnCardsFromGraveyardToHand present (was MoveSelfBetweenZones before fix)
  [x] SelfExileFromStack present (Recall exiles itself)
