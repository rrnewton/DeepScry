# Description

## Background

CPU profiling with both Callgrind (instruction count) and Linux perf (wall-clock time) identified the top hotspots.

**NEW: Linux Perf Profiling (2025-11-29_#966)**

Wall-clock sampling (5000 games, 8.09B cycles, 2,838 games/sec):

1. **priority_round** (17.75% CPU) - game_loop/priority.rs
2. **ManaEngine::update** (15.00% CPU) - mana_engine.rs
3. **String operations** (8.89% combined)
4. **bounds_check_payment** (4.79% CPU) - mana_payment.rs

## Completed Optimizations

### OPT-1: Cache card type flags in CardCache ✅ (a11c14b, 2025-11-30_#992)

**Status:** COMPLETE

**Changes:**
- Added is_land, is_creature, is_artifact fields to CardCache
- Pre-compute type checks at card load time
- O(1) lookup instead of Vec::contains() at runtime

**Results:**
- Most benchmarks improved 10-18%
- fresh_games: -10.1%
- robots_mirror/rewind_play_again: -17.9%
- simple_bolt/rewind_play_again: -9.9%

### OPT-7: Reuse candidates Vec in mana payment ✅ (2025-11-30_#996)

**Status:** COMPLETE

**Changes:**
- Extracted tap_for_color() as a separate method
- Hoisted candidates SmallVec outside the per-color loop
- Clear and reuse buffer instead of creating 5 SmallVecs per payment

**Results:**
- Reduces SmallVec allocations from 5 to 1 per mana payment
- Benchmark results within noise range (~3%) - main benefit is allocation reduction
- 45,197 games/sec (simple_bolt rewind_play_again)
