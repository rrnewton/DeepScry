# Description

Context:
- Date: 2026-04-03
- Puzzle: /tmp/disk_test.pzl (Disk + Sedge Troll vs Grizzly Bears)
- Seed: 42

Steps to Reproduce:
1. Create a puzzle with Nevinyrral's Disk on the battlefield (untapped, since it's been there a turn)
2. Run game - Disk's activated ability is NEVER offered as a choice

Expected Behavior:
- "activate Nevinyrral's Disk" should appear as an available action when Disk is untapped and controller has 1 mana available
- Activation should destroy all artifacts, creatures, and enchantments

Actual Behavior:
- Disk has 0 activated abilities in snapshot (activated_abilities: [])
- "activate Nevinyrral's Disk" never appears in available actions
- The ability is silently dropped during card parsing

Root Cause:
- mtg-engine/src/loader/ability_parser.rs line 92: ApiType::DestroyAll is defined
- mtg-engine/src/loader/effect_converter.rs: params_to_effect() handles ApiType::Destroy but NOT ApiType::DestroyAll
- When the ability is parsed, params_to_effect returns None, so the ability gets no effects
- parse_activated_abilities() (card.rs line 2392) skips abilities with empty effects
- Fix: Add ApiType::DestroyAll handler to effect_converter.rs that creates an appropriate Effect variant

This also affects:
- Shatterstorm (Sorcery version of DestroyAll)  
- Any card using AB$ DestroyAll or SP$ DestroyAll

Evidence:
- Snapshot shows: Card[5]: name=Nevinyrral's Disk, tapped=False, abilities=0
- Card definition: A:AB$ DestroyAll | Cost$ 1 T | ValidCards$ Artifact,Creature,Enchantment
