# Description

FOLLOW-UP (deck-editor UX batch2 item 11, option b): add a STABLE server-side direct-link route for a user's cloud deck collection.

CONTEXT: R2 deck-collection objects are PRIVATE — served only via short-TTL presigned SigV4 URLs (GET /api/deck-storage/credentials mints get_url/put_url/download_url). A permanent public link would require making the bucket public, which is a security risk (every user's decks become world-readable). 

SHIPPED NOW (option a): the deck editor's "Direct link" button (web/deck_editor.html + DeckStorage.directLink() in web/deck_storage.js) copies the CURRENT presigned GET URL to the clipboard with a clear "(temporary link, expires)" note. Works but the link lapses after the presign TTL.

WANTED (option b, this bead): a STABLE app route — e.g. GET /api/deck-storage/link (or /decks/me) — that, on each load, authenticates the session, re-mints a fresh presigned GET, and 302-redirects to it. The user can then bookmark ONE durable URL that always resolves to their latest collection without exposing the bucket. Auth = the same session cookie /auth/status uses; 401 when logged out. Keep the existing presigned-credentials + download paths working unchanged.

SCOPE: server-side (Rust web server route + the existing presign helper) + swap the editor's "Direct link" button to point at the stable route once it exists. Out of scope for the web-only batch2 pass (no server work), hence option (a) shipped first.

## Partial implementation landed (claude/r2-public-bucket, 2026-06-11)

A simpler approach to permanent URLs was implemented without needing the server-side redirect route: by making the R2 bucket public (Cloudflare dashboard → R2 → Settings → Public access → Allow Access), objects get a permanent `pub-<hash>.r2.dev/<key>` URL with no expiry.

Changes shipped in commit 9c89c209 (branch claude/r2-public-bucket):
- R2Config gains `public_base_url: Option<String>` from `R2_PUBLIC_BASE_URL` env var
- Credentials endpoint includes `"public_url"` field when configured
- `DeckStorage.directLink()` returns `{url, permanent}` — uses public_url when present, presigned fallback otherwise
- "Direct link" button disabled when no cloud decks exist (prevents NoSuchKey error)
- Button shows permanent vs temporary status message accordingly

REMAINING for option (b) route (this bead): If the operator prefers NOT to make the bucket public, the server-side 302-redirect route (GET /api/deck-storage/link → presigned GET redirect) is still valuable as an alternative. The bucket-public approach is simpler and ships no server complexity, but requires a Cloudflare dashboard step. Either approach satisfies the "stable bookmark" use-case.

OPERATOR STEP for public bucket (one-time, out-of-band): Cloudflare dashboard → R2 → deepscry-decks → Settings → "Public access" → Allow Access. Copy the pub-<hash>.r2.dev subdomain and set R2_PUBLIC_BASE_URL=https://pub-<hash>.r2.dev in .r2.env.
