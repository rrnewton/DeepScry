# Description

## Context

Part of the web-server-unification design. Depends on **mtg-468** (the unified axum process) — TLS is bolted on once axum exists.

## Port + TLS plan

- Port 80 is taken by another web server on the VM. DeepScry already runs on **8080**.
- Cloudflare has an Origin Rule mapping `deepscry.net` → VM port 8080.
- Switch CF SSL/TLS mode to **Full (strict)**. CF will speak HTTPS to origin on 8080.
- axum binds 8080 with rustls. On every connection, the certificate presented is the Cloudflare Origin Certificate (15-year, free, only trusted by CF — perfect for this topology).

## Cert provisioning

- User runs a separate script (provided by coordinator) that calls the Cloudflare API with an account API token, generates an origin certificate, and writes:
  - `/etc/ssl/deepscry/deepscry.crt` (full chain)
  - `/etc/ssl/deepscry/deepscry.key`
- Mode: `0640`, owned `root:deepscry` (or whatever group runs the service — see #mtg-N3 systemd unit).
- Renewal is a 15-year non-issue; document the re-run procedure.

## Code change in mtg-engine

Small. Inside the new `mtg server-web` entry point (added in mtg-468):

```rust
let cert = std::env::var("MTG_TLS_CERT").ok();
let key  = std::env::var("MTG_TLS_KEY").ok();
match (cert, key) {
    (Some(c), Some(k)) => {
        let tls = RustlsConfig::from_pem_file(c, k).await?;
        axum_server::bind_rustls(addr, tls).serve(app.into_make_service()).await?;
    }
    _ => {
        axum::serve(listener, app).await?;
    }
}
```

Either `axum-server` (simplest TLS story) or hand-rolled `tokio-rustls` acceptor. `axum-server = { version = "0.7", features = ["tls-rustls"] }` adds one crate.

Graceful degradation: if either env var is unset, the server runs plain HTTP. This keeps local dev and pre-cert deploys working.

## Frontend change

`web/server-config.js` becomes truly same-origin (already noted in mtg-468 but spelled out here because the rustls flip is what enables it):

```js
window.MTG_WS_URL = (location.protocol === "https:" ? "wss://" : "ws://")
                  + location.host + "/lobby";
```

No more hardcoded `:17810`. No mixed-content risk because the WS scheme tracks the page scheme.

## Cloudflare dashboard steps (one-time, manual)

1. SSL/TLS → Overview → set mode to **Full (strict)**.
2. SSL/TLS → Origin Server → confirm the origin cert is listed (or create via API per script).
3. Verify the existing Origin Rule still maps host:443 → origin:8080.

## Acceptance criteria

- With cert installed: `curl https://deepscry.net/` returns 200 and a valid CF-edge cert chain (verified by the browser).
- DevTools shows zero mixed-content warnings on the landing page.
- `wss://deepscry.net/lobby` connects from the deployed page with no port number in the URL.
- With env vars unset locally: `curl http://localhost:8080/` still works (graceful degrade).
- `curl -k --resolve deepscry.net:8080:<vm-ip> https://deepscry.net:8080/` from outside CF returns 200 served by axum-rustls directly (origin reachable from CF).

## Dependencies

- Depends on **mtg-468** (unified axum process must exist).
- Related to **mtg-N3** (systemd unit needs to grant read access to the key file).
