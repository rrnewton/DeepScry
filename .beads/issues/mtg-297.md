# Description

## Swords to Plowshares: exile + dynamic life gain (= creature power) — FIXED

RESOLVED 2026-05-30_#2489 on branch compat-dynamic-gainlife-mtg297.

Root cause: `DB$ GainLife | LifeAmount$ X` with `SVar:X:Targeted$CardPower`
is a DYNAMIC amount (the targeted creature's power, captured at resolution).
The old loader path (`params_to_effect` -> `get_i32("LifeAmount")`) failed to
parse "X" and silently dropped the GainLife sub-effect.

Fix — general dynamic-amount construct (not a card hack):
- New `DynamicAmount` enum (core/effects.rs): TargetPower / TargetManaValue /
  DamageDealt / Fixed, parsed from `Targeted$<Characteristic>` SVars.
- New `Effect::GainLifeDynamic { player, amount: DynamicAmount, reference }`.
- New `PlayerId::target_controller()` sentinel for `Defined$ TargetedController`.
- Amount captured as last-known information (CR 608.2g/2h) via a pre-resolution
  power/mana-value snapshot using the CR 613 layer system, so continuous static
  buffs (e.g. Sedge Troll +1/+1 while controlling a Swamp) are counted before
  the exile strips them.
- `ExilePermanent` now records last_resolved_target so the chained GainLife
  resolves against the exiled creature.

Evidence (repro from this issue):
  target/release/mtg tui decks/old_school/01_rogue_rogerbrand.dck --p1 zero --p2 zero --seed 42 \
    --p1-draw 'Badlands;Sengir Vampire;Swords to Plowshares;Scrubland;Shivan Dragon;Bayou;Disenchant' \
    --p2-draw 'Badlands;Sedge Troll;Animate Dead;Scrubland;Triskelion;Swamp;City of Brass' -v verbose --no-color-logs
  -> 'Sedge Troll (66) is exiled' then 'Zero2 gains 3 life (life: 22)'
     (3 = Sedge Troll's power as a 3/3 with the Swamp buff; LKI per CR 608.2h).
