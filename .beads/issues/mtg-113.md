# Description

## Status: IMPLEMENTED

X-cost spell casting is now fully functional. Implementation includes:

### What was added:
1. **Card.x_paid field** - stores the chosen X value when casting
2. **ManaCost.has_x() / with_x_value()** - methods for X cost detection and resolution
3. **PlayerController.choose_x_value()** - controller interface for X selection (default: max available)
4. **Priority loop integration** - X value selection happens during step 2 of 8-step casting, after modal selection
5. **Effect variants** - DealDamageXPaid, DrawCardsXPaid, DiscardCardsXPaid for X-dependent effects
6. **Effect converter** - Handles NumDmg$ X and NumCards$ X when SVar X = Count$xPaid
7. **Resolution** - resolve_x_paid_effect() patches effects with concrete X value at resolution time
8. **ReplayChoice::XValue** - supports snapshot/replay of X value decisions

### Cards now working:
- Fireball (XR) - X damage
- Braingeyser (XUU) - draw X cards
- Mind Twist (XB) - discard X at random
- Any card with SVar:X:Count$xPaid

### Verified:
- Fireball X=2 correctly deals 2 damage to opponent
- X value correctly clamped to available mana minus colored costs
- Affordability check allows X=0 casting

### Still TODO:
- Heuristic AI doesn't proactively cast X damage to player face (only targets creatures)
- Logging shows 'X damage' instead of resolved amount (cosmetic)
