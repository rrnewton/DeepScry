# Description

All Hallow's Eve (Legends, cardsfolder/a/all_hallows_eve.txt) now correctly self-exiles with two scream counters, ticks them down each of the controller's upkeeps from exile, and at counters==0 puts itself into the graveyard and mass-resurrects every player's creature cards.

RESOLVED 2026-05-28 (#2362 / fix-allhallows-eve branch).

Both originally-listed bugs:
1. ChangeZone Origin$ Stack | Destination$ Exile redirect — was ALREADY fixed in integration prior to this work (Effect::SelfExileFromStack + resolve_spell_finalize skip-if-not-on-stack). Verified working.
2. RememberChanged$ True populating the Remembered list — was ALREADY fixed (remembered_cards push in SelfExileFromStack; CardId::remembered_card() sentinel consumed by the chained PutCounter). Verified working. The misleading "puts counters on (none)" log was a post-resolution logging artifact (logger ran after DBCleanup cleared remembered_cards) and is now fixed to name the source card.

The remaining BLOCKED behaviour from the compat issue (the exile-resident upkeep trigger + counter tick + exile→graveyard self-move + mass resurrection) was unimplemented and is the bulk of this fix:
- Trigger struct gained trigger_zones (TriggerZones$) and present_self_condition (IsPresent$ Card.Self+counters_<CMP>_<TYPE>).
- check_phase_triggers now scans the exile zone (in addition to battlefield) and honours the intervening-if counter condition (CR 603.4 / 603.6e).
- New general effects: Effect::MoveSelfBetweenZones (DB$ ChangeZone | Defined$ Self non-stack) and Effect::ConditionalSelfCounter (ConditionDefined$ Self + ConditionPresent$ Card.counters_<CMP>_<TYPE> gating).
- The phase-trigger parser now falls back to the generic extract_effects_from_svar SVar-chain extractor for non-hardcoded chains (RemoveCounter → ChangeZone Self → ChangeZoneAll).

Regression tests:
- Unit: mtg-engine/src/game/actions/tests/spell_casting.rs::test_all_hallows_eve_upkeep_trigger_parse_shape (parser shape: trigger_zones, present_self_condition, RemoveCounter + EQ0-gated MoveSelfBetweenZones + EQ0-gated ChangeZoneAll). The pre-existing test_all_hallows_eve_self_exile_with_two_scream_counters still covers bugs 1+2.
- E2E: tests/all_hallows_eve_resurrection_e2e.sh (auto-discovered by shell_script_tests.rs glob, wired into make validate). Asserts self-exile, 2 SCREAM counters, >=2 exile-zone upkeep triggers, all three creatures resurrected, Lightning Bolt (non-creature) NOT resurrected.

Reproducer: ./target/release/mtg tui --start-state test_puzzles/all_hallows_eve_resurrection.pzl --p1=fixed --p2=zero --p1-fixed-inputs="cast All Hallow's Eve" --stop-on-choice=6 --json --seed 42 --verbosity 3
