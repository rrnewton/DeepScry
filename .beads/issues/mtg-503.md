== Owner interpretation (2026-06-01_#2639(f4402e9b), compat-oldschool-finish) ==
Falling Star is a paper DEXTERITY card. Per the project owner we approximate it as:
  "{2}{R} Sorcery: Falling Star deals 3 damage to target creature. If that
   creature survives, tap it."
This INTENTIONALLY DIVERGES from Forge-Java (physical flip + DamageAll + TapAll
over the "touched"/Remembered set). Reasoning: a smart player spreads their
board so a flip catches one creature, and a dextrous flipper hits exactly the
one aimed at. The divergence is expressed in the Rust ENGINE (effect_converter),
NOT by editing the card script.

Card script (cardsfolder/f/falling_star.txt) is UNEDITED upstream Forge-Java:
  A:SP$ FlipOntoBattlefield | SubAbility$ DBDamageTouched | AILogic$ DamageCreatures | ...
  SVar:DBDamageTouched:DB$ DamageAll | ValidCards$ Creature.IsRemembered | NumDmg$ 3 | SubAbility$ DBTapAllDamaged | ...
  SVar:DBTapAllDamaged:DB$ TapAll | ValidCards$ Creature.IsRemembered+DamagedBy | SubAbility$ DBCleanup | ...
  SVar:DBCleanup:DB$ Cleanup | ClearRemembered$ True

== Engine divergence (where the work lives) ==
- effect_converter ApiType::Unknown("FlipOntoBattlefield") arm branches on the
  tokenized SubAbility$ reference: SubAbility$ DBDamageTouched (Falling Star) ->
  Effect::DealDamage { amount: 3, target: None } (3 damage to the chosen
  creature target).
- effect_converter TapAll arm: ValidCards$ Creature.IsRemembered+DamagedBy
  (requires_remembered) converts the mass-tap into a single survival-gated
  TapPermanent { reuse_previous } (there is exactly one "touched" creature: the
  damage target).
- effect_converter Tap arm: Defined$ Targeted -> reuse_previous sentinel
  (mirrors the existing Untap arm).
- resolve_effect_target (actions/mod.rs): new TapPermanent{reuse_previous} arm,
  gated on SURVIVAL (battlefield.contains && current_toughness > 0 &&
  toughness > damage). A creature that took lethal damage is NOT tapped (fizzles
  to a placeholder).
- Display logger (game_loop/priority.rs) mirrors the same survival-gated
  reuse_previous resolution; logging.rs skips placeholder Tap targets so a dead
  creature emits no "taps Unknown" line.

== Relationship to Java Forge ==
Java models the physical flip (FlipOntoBattlefield + DamageAll(Remembered) +
TapAll(DamagedBy)). We diverge in the converter to a single creature target with
a survival-gated chained tap. The upstream card script in forge-java is left
untouched; the divergence is in the Rust engine.
