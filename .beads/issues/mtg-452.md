# Description

##  Card: Donatello, the Brains

**Card text:** If one or more tokens would be created under your control, those tokens plus an additional Mutagen token are created instead. Partner—Character select.

**Card script:** `cardsfolder/d/donatello_the_brains.txt`
```
Name:Donatello, the Brains
ManaCost:2 U
Types:Legendary Creature Mutant Ninja Turtle
PT:2/4
R:Event$ CreateToken | ActiveZones$ Battlefield | ValidToken$ Card.YouCtrl | ReplaceWith$ DBReplace | Description$ ...
SVar:DBReplace:DB$ ReplaceToken | Type$ AddToken | Amount$ 1 | TokenScript$ c_a_mutagen_sac
K:Partner:Character select
```

**[x] Body (combat):** 2/4 blocked Grizzly Bears (2/2), dealt 2 damage, took 2 damage.
WORKING. Log evidence:
```
Donatello, the Brains (7) deals 2 damage to Grizzly Bears (11)
Grizzly Bears (11) deals 2 damage to Donatello, the Brains (7)
```

**[x] Replacement effect (R:Event$ CreateToken):** When Raise the Alarm created 2 Soldier
tokens, Donatello's replacement also created 1 Mutagen Token.
WORKING (FIXED in this session). Root cause was:
- `R:Event$ CreateToken` was parsed/stored in `raw_abilities` but never converted to a game
  effect — the replacement was silently dropped.
- Fix: Added `StaticAbility::TokenCreationBonus` variant, parsed from
  `R:Event$ CreateToken | ValidToken$ Card.YouCtrl | ReplaceWith$ <svar>` where `<svar>`
  resolves to `DB$ ReplaceToken | Type$ AddToken | Amount$ 1 | TokenScript$ <script>`.
- Applied in `execute_create_token` via non-recursive bonus minting (CR 614.5d).
- Also extended `extract_token_scripts()` to include `ReplaceToken` SVars so the token
  definition (`c_a_mutagen_sac`) is preloaded by the puzzle/game loader.

WORKING log evidence:
```
Created Soldier Token under Player 1's control
Created Soldier Token under Player 1's control
Created Mutagen Token under Player 1's control
```
