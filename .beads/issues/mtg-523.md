CARD STATUS: not-tested (skeleton placeholder)

Related: 1994 Old School playtest umbrella (filed in same commit)

Findings (2026-05-30_#2533(b052ce01), compat-wave16-robots):

1. [x] Parses as {0} Land. {T}: Add {C}{C}{C} mana ability with Amount$ 3.
2. [WORKING] Mana ability: taps for {C}{C}{C} per activation (was BROKEN —
   produced a single {C}, ignoring Amount$ 3).
   ROOT CAUSE: GameState::tap_for_mana_for_cost (actions/mod.rs) had a LAND
   path that always added exactly 1 pip, ignoring the cached
   mana_production.amount. Only the NON-land explicit-mana path (Black Lotus)
   and the any-color land path respected `amount`. Workshop is a Land, so it
   hit the single-pip branch.
   FIX: capture `land_amount = mana_production.amount.max(1)` before the player
   borrow and multiply the produced pips (and the gamelog render). General fix
   — lifts ANY land that produces >1 mana per tap.
3. [PARTIAL] `RestrictValid$ Spell.Artifact` (the "spend only to cast artifact
   spells" restriction) is NOT enforced — Workshop mana is currently spendable
   on any spell. This is strictly more permissive than printed. Filed as a
   follow-up (mana spend-restriction engine gap). The robots deck only spends
   Workshop mana on artifacts, so play is unaffected. Tracked: see new follow-up.
