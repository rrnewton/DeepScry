# Description

REGRESSION introduced by the mtg-722 CDN image migration (merged @d6897f05); turned integration CI RED.

ROOT CAUSE: native_game.html unconditionally does `fetch('./data/card-lookup.bin')` at boot (task #7 CDN table load). That file is GITIGNORED (derived asset) and only built at DEPLOY time by `mtg build-card-lookup` (deploy-cloud.sh §2b). On a hermetic checkout — CI, or a fresh dev clone, or validate after `git clean`/new worktree — the file does not exist → the fetch 404s → the browser logs a console.error → the no-console-error browser tests (test_decouple_step3_launch_game_session.js, and any other asserting zero console errors) FAIL.

WHY IT SLIPPED: the authoring machine had a stale UNTRACKED web/data/card-lookup.bin (1.5MB, left by a prior `deploy-cloud.sh deploy` run). Because it is gitignored, `git status` looked clean, so local `make validate` passed while CI (clean tree) failed. Classic untracked-file determinism trap (see the feedback-decide-by-principles / untracked-files-break-determinism memory).

FIX (commit 2cef04ea, branch fix-cardlookup-hermetic): `mtg export-wasm` now ALWAYS writes a valid EMPTY 14-byte SCDT table (`encode_card_lookup(&[])`) to web/data/card-lookup.bin. `make wasm-dev`/`wasm` (which every browser-test validate step depends on via validate-wasm-step) runs export-wasm, so the placeholder is regenerated every validate → fetch 200s, table loads 0 entries, lookups cascade to Gatherer. Deterministic, no network, no reliance on any untracked file. Overwrites unconditionally so a lingering real table can't reintroduce non-determinism. Deploy runs `make wasm-export` BEFORE §2b, so §2b overwrites the placeholder with the real CDN table; the placeholder is only ever the hermetic default.

RED→GREEN PROVEN: rm web/data/card-lookup.bin → test_decouple_step3 fails on the 404 (exit 1); run export-wasm → placeholder written → test passes clean (exit 0).

STANDING GUARD: the existing browser tests (assert zero console errors) catch any reintroduction; export-wasm regenerating the placeholder keeps validate hermetic.

STATUS: FIXED, fix-forward on fix-cardlookup-hermetic; validate running; ff-merge pending to restore integration green (unblocks slot05).
