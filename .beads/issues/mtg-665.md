# Description

Bug: SP$ FlipOntoBattlefield + DBDamageTouched (Falling Star) mis-routes to destroy; needs deterministic damage model

RESOLVED — already fixed before this issue could be acted on. This bug was filed
describing the OLD broken behavior (Falling Star reusing Chaos Orb's
destroy-touched path → "Falling Star destroys Mountain", and the empty
Remembered set). Both gaps were subsequently fixed by the Falling Star
compatibility work tracked in mtg-503 / mtg-563, which landed the deterministic,
replay-safe, native==WASM damage model this issue asked for.

== What landed (the deterministic model) ==
Per the project owner, Falling Star (a paper DEXTERITY card with no faithful
digital analog) DIVERGES to a single-target approximation:
  "{2}{R} Sorcery: Falling Star deals 3 damage to target creature.
   If that creature survives, tap it."
Implemented in the engine (NOT by editing the upstream card script):
- effect_converter.rs ApiType::Unknown("FlipOntoBattlefield") arm branches on the
  TOKENIZED SubAbility$ reference (no substring scan):
    SubAbility$ DBDamageTouched (Falling Star) -> Effect::DealDamage { amount: 3 }
    (target creature), DISTINCT from
    SubAbility$ DBDestroyTouched (Chaos Orb)    -> Effect::DestroyPermanent.
  This resolves gap #1 (no more land-destroy mis-route; DamageAll routed
  distinctly from the destroy branch).
- effect_converter.rs TapAll arm: ValidCards$ Creature.IsRemembered+DamagedBy
  (requires_remembered) converts the mass-tap into a single survival-gated
  TapPermanent{reuse_previous}. This resolves gap #2 (no reliance on a
  physically-"landed-on" Remembered set the digital engine can't simulate).
- resolve_effect_target (actions/mod.rs): TapPermanent{reuse_previous} is gated
  on SURVIVAL — a creature that took lethal 3 damage is NOT tapped.

== Verification ==
- e2e: tests/falling_star_damage_tap_e2e.sh (PASSES; auto-discovered by
  mtg-engine/tests/shell_script_tests.rs glob "*.sh" in tests/, so it runs in

  4/4 Sengir Vampire survives 3 damage + IS tapped.

Closing as a duplicate of the completed mtg-503 (Card Compatibility: Falling
