CONSOLIDATED onto ONE shared strong type `ChoicePayload { choice_indices: Vec<usize>, spell_ability: Option<SpellAbility>, library_search_result: Option<CardId>, target_card_ids: Option<Vec<CardId>> }` (defined in network/choice_entry.rs, re-exported via network::*).

WIRE-FORMAT INVARIANCE: project speaks JSON on the wire (serde_json, order-independent). #[serde(flatten)] keeps the four fields at the SAME JSON object level; only intra-object key ORDER shifts (payload fields now after description), which JSON semantics + every deserializer ignore. Verified by serde round-trip experiment (flat decodes inline bytes and vice versa) and by the existing test_protocol_encoding_decoding re-encode-equality test. The rewind/replay buffer comparison + dedup (push_opponent_choice / record_opponent_choice) is STRUCTURAL (choice_indices field compare), never a JSON-string compare — so flatten is replay-stable.
