# Description

Track instances where players are not offered choices they should receive, and implement missing choice points.

## Problem

Players are not being asked to make certain choices that should be presented:

### 1. Keep/Mulligan Choice

**Issue**: No mulligan decision at game start
- Players should be able to mulligan (redraw) their opening hand
- Should follow official MTG mulligan rules

**Example**: Every game starts, no choice to mulligan

**Solution**:
- Add `choose_mulligan()` to PlayerController trait
- Implement for all controller types (heuristic, TUI, fancy TUI)
- Add `--keep-hand` CLI argument to skip mulligan choice and auto-keep

**Implementation**:
```rust
fn choose_mulligan(&mut self, view: &GameStateView, hand: &[CardId]) -> bool {
    // Return true to mulligan, false to keep
}
```

**CLI argument**:
```bash
## Skip mulligan choice, always keep opening hand
./mtg tui --p1=fancy --p2=heuristic --keep-hand
```

### 2. Activated Abilities Not Offered

**Issue**: Activated abilities are not presented as choices during priority

**Example**: Vibrant Cityscape
- Card text: "{T}, Sacrifice Vibrant Cityscape: Search your library for a basic land..."
- Expected: Should be offered as choice during main phase when untapped
- Actual: No choice presented, cannot activate

**Root Cause**: 
- `choose_spell_ability_to_play()` only offers PlayLand and CastSpell
- Activated abilities (ActivateAbility) may not be properly enumerated
- Need to check if `get_available_abilities()` includes activated abilities

**Investigation needed**:
1. Does `GameStateView::get_available_abilities()` return activated abilities?
2. If yes, why aren't they in the choice list?
3. If no, need to implement activated ability discovery

### 3. Other Missing Choices (Future)

Create a systematic way to track missing choices:

**Methodology**:
1. Play games with fancy TUI
2. Note when expected choices don't appear
3. Cross-reference with MTG rules
4. Document in this issue

**Categories**:
- [ ] Mulligan decisions (highest priority)
- [ ] Activated abilities during priority
- [ ] Triggered abilities (may trigger but no choices for modes/targets)
- [ ] Replacement effects (e.g., "draw a card" → "may draw 2 discard 1")
- [ ] Special actions (play land, activate mana abilities)
