# Description

FIXED + improved 2026-05-12 (compat1).

Set: LEA (mtg-383)
Deck: rogue_rogerbrand (mtg-387)
Card script: cardsfolder/w/wheel_of_fortune.txt

Findings:
- The SP$ Discard Mode$ Hand Defined$ Player effect AND the chained SubAbility DBEachDraw both work correctly. Both players discard their hands, then both draw 7. Order is correct (discards before draws).
- The post-resolution effect-summary log was using raw sentinel values: 'Wheel of Fortune (3) causes Player 4294967295 to discard 255 card(s)'. The 4294967295 = u32::MAX is the ALL_PLAYERS_ID sentinel for Defined$ Player; 255 = u8::MAX is the Mode$ Hand sentinel for 'discard entire hand'.

Fix:
1. mtg-engine/src/game/game_loop/logging.rs: get_player_name() now returns 'each player' for ALL_PLAYERS_ID and 'remembered players' for REMEMBERED_PLAYERS_ID instead of falling through to the 'Player {N+1}' formatter (which produced the offensive 4294967295).
2. mtg-engine/src/game/game_loop/logging.rs: DiscardCards summary log now special-cases count == u8::MAX as 'discard their hand' instead of 'discard 255 card(s)'.

Behavioral aspects verified:
1. [x] Castable for {2}{R} as Sorcery
2. [x] Discard Mode$ Hand Defined$ Player discards EACH player's hand (both)
3. [x] DBEachDraw Defined$ Player draws 7 for EACH player (both)
4. [x] Discard precedes draw (CR 608.2c)
5. [unverified] Behavior when library has fewer than 7 cards (would loss-by-decking trigger)
6. [x] Wheel of Fortune itself goes to graveyard after resolution
7. [implicit] Hand size limit doesn't apply mid-resolution
8. [x] Game log shows clear sequence with friendly 'each player' message
9. [implicit] Replacement effects (none in test deck)

Reproducer:
  ./target/release/mtg tui --start-state test_puzzles/wheel_of_fortune_each_player.pzl --p1=fixed --p2=zero --p1-fixed-inputs='cast Wheel of Fortune' --stop-on-choice=4 --json --seed 42 --verbosity 3
