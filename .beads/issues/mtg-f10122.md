# Description

Files: mtg-forge-rs/mtg-engine/src/server.rs (or new module)

Action: When a bug report passes the trusted-password check:
1. Launch claude --dangerously-skip-permissions -p with the bug report content as the prompt
2. The Claude agent should attempt to fix the bug and file a PR via gh
3. Link the original bug report issue to the PR (cross-reference in both directions)
4. This is fire-and-forget from the server's perspective — don't block the response to the user
5. Log the auto-fix attempt and its status

Why: Trusted users can trigger automatic bug fixes. For now, just launch the agent — no auto-merge/rebuild/relaunch.

Verify:
- claude command is spawned as a child process with correct args
- PR is linked back to the original issue
- Server doesn't block waiting for Claude to finish
- cargo build succeeds
