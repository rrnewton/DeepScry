INCREMENT 2 — OAUTH LOGIN + UI (this commit):
- Server (web_server/oauth.rs): GitHub+Google authorization-code flow. /auth/login/<p>, /auth/callback/<p>,
  /auth/logout, /auth/status. CSRF state cookie (one-shot), HttpOnly session cookie, in-memory session store.

  (read from id_token). OAuthIdentity derives R2 prefix decks/<provider>-<sub>/ via the SAME Identity seam

- Landing page (index.html): THREE login choices — "Sign in with GitHub", "Sign in with Google",
  "Ephemeral account" (renamed guest; name-only, no server account, globally-unique-while-connected name
  already enforced by lobby try_register_name/release_name; VISIBLE warning that ephemeral CANNOT save decks
  to cloud → localStorage only). OAuth buttons shown only for configured providers (via /auth/status).
- Custom Deck URL (launcher.html): client-side fetch+parse of any plain-text .dck link, available to EVERYONE
  (no OAuth/R2). Loads into the shared custom-decks localStorage + selects it. Specific CORS-error and
  invalid-.dck messages. Self-hosted format explainer at web/docs/dck-format.html, linked from the input.
  No server proxy (no SSRF surface) — client-side only.
