OAUTH-INDEPENDENT STORAGE LEG — IMPLEMENTED on branch `claude/r2-deck-storage` (2026-06-10):

SERVER (mtg-engine/src/web_server/r2.rs + mod.rs):
- `R2Config::from_env()` reads AWS_ACCESS_KEY_ID / AWS_SECRET_ACCESS_KEY / R2_ENDPOINT / R2_BUCKET
  (never hardcoded; secret redacted in Debug). Absent → endpoint 503s, rest of server unaffected.
- Local AWS SigV4 presigned-URL signing (PUT/GET/HEAD + attachment-download GET) using `ring`
  (HMAC-SHA256/SHA256, already pulled by rustls) + `percent-encoding`. NO aws-sdk, NO R2 round-trip.
- `Identity` trait + `DevIdentity` stub (fixed `decks/dev/` prefix) = swappable seam for OAuth.
- `is_valid_user_id` guards prefix traversal (future hostile identity provider).
- New route GET /api/deck-storage/credentials → JSON {put_url,get_url,head_url,download_url,object_key,ttl,content_type}.
- VERIFIED LIVE against real bucket (r2.rs::tests::live_round_trip, #[ignore]d, manual w/ creds): PUT ok,
  GET bytes match, stale If-Match → 412. 7 hermetic unit tests pass.

CLIENT (web/deck_storage.js + deck_editor.html):
- .tgz pack/unpack via native CompressionStream/DecompressionStream + a tiny hand-rolled USTAR tar (no dep).
- Collection ⇄ plaintext <name>.dck members (human-readable; data-liberation).
- hydrate() GET → edit → If-Match conditional PUT (debounced ~1/sec); 412 → "reload to merge" hint.
- IndexedDB offline read/edit cache (NOT source of truth). Opaque bytes: Content-Type application/gzip, no Content-Encoding.
- "Download my decks" button → fresh presigned attachment-GET.
- ADDITIVE localStorage migration (mtg-forge-custom-decks) behind flag localStorage['mtg-deck-cloud']='1';
  production behaviour unchanged until OAuth + user flip it on. Existing localStorage flow untouched.

TESTS: web/test_deck_storage.js (16 checks, hermetic mocked-R2 browser e2e) wired into validate.py browser suite.

CORS: parent token LACKS bucket-config admin (PutBucketCors/GetBucketCors → 403). Policy shipped at
scripts/r2-cors.json; bucket OWNER must apply via Cloudflare dashboard or an admin token (steps in
ai_docs/reference/R2_DECK_STORAGE.md). AllowedOrigins deepscry.net/www, methods GET/PUT/HEAD, ExposeHeaders ETag.

DEPLOY: deploy-cloud.sh forwards R2_* env into the systemd EnvironmentFile; example in deepscry-deploy.env.example.
