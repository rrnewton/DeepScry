# Description

Durable deck storage on Cloudflare R2 (OAuth + prefix-scoped temp creds + per-user .tgz). APPROVED 2026-06-10; R2 creds live, OAuth app live (github+google).

## Cloud-deck UX rework (branch claude/deck-cloud-decks, user-requested 2026-06-11)

Built on the deck-editor-3col UX. Server-side R2/OAuth was already complete (presigned creds endpoint, /auth/status, provider-scoped prefix decks/<provider>-<sub>/collection.tgz). This pass reworked the BROWSER UX:

A. AUTHORITATIVE LOGIN + AUTO-HYDRATE. deck_storage.js now exposes authStatus()/isLoggedIn() reading the server /auth/status (session cookie) — NOT inherited sessionStorage (fixed the Safari+Google "logged in but saw nothing" bug). The editor + launcher auto-fetch the R2 collection when logged in.

B. TWO LABELED LISTS in the editor: "Saved Decks — Cloud" (auto-hydrated R2) + "Saved Decks — Local Browser" (localStorage). Signed-out shows a sign-in note instead of the cloud account. Cloud section is accent-tinted with a ☁ badge; local has a 💾 badge. Load/delete act on the correct store. Same-name in both is allowed.

C. HONEST SAVE INDICATOR: "☁ Saved to your cloud account" / "💾 Saved to local browser" / "⚠ Cloud save failed (<reason>) — kept local". Durable "View / download my cloud collection" link re-mints a fresh presigned URL via downloadMyDecks (not a raw expiring URL) + shows the object path decks/<id>/collection.tgz.

D. MANUAL MIGRATION: removed the auto-migrateLocalStorage()-on-load. Added a "Migrate All →" button (DeckStorage.migrateAll overwrites cloud slots with every local deck) + a "Delete local copies after migrating" checkbox (deletes local only after a successful cloud write).

E. REAL R2 ROUND-TRIP verified. Added a DEV-ONLY gated route GET /auth/dev-login (env MTG_DEV_LOGIN=1; 404 otherwise; non-Secure cookie for local HTTP) to mint a session without live OAuth. Proven: dev-login → /auth/status logged_in → /api/deck-storage/credentials returns provider-scoped presigned URLs; direct R2 PUT 200 + GET 200 + byte-identical (curl). The BROWSER PUT/GET is blocked ONLY by missing bucket CORS (preflight 403) — see follow-up. App-level round-trip (save→PUT→hydrate→GET, two lists, migrate) verified hermetically with a fake-R2 route + the REAL server credentials.

F. LAUNCH "missing cards / empty deck" RESOLVED. Editor saveDeck() guards 0-card decks ("⚠ This deck is empty — add some cards before saving") and writes nothing. Launcher shows a clearer empty/too-small message. The 86-byte metadata-only .dck was an empty deck saved before this guard; deckToDck serialization was always correct.

G. LAUNCHER two-group picker + cloud PLAYABILITY (user-requested). launcher.html now splits custom decks into "Custom Decks (Cloud)" + "Custom Decks (Local Browser)" optgroups (cloud only when logged in; sign-in hint otherwise). A cloud deck's option value is `cloud:<name>`; buildDeckSubmission resolves its cards from the hydrated R2 collection (so the network CreateGame gets real cards), and selecting it mirrors the deck into localStorage so the game-page boot path (game_boot_params.js) finds it too — fixing "not found / missing cards" for cloud decks.

H. Import dialog: added an "Upload .dck…" button (hidden file input + FileReader) that imports a file through the SAME path as paste.

Tests: web/test_deck_storage.js +authStatus/credentialsInfo/migrateAll. web/test_deck_editor.js +sections 9b (two lists, empty-save guard, upload), 9c (logged-in account + auto-hydrate + cloud save + migrate), 9d (launcher two groups + cloud playability). Rust unit: dev_login_is_404_without_env_gate. make validate green (34 passed).

REMAINING (ops, see follow-up): apply scripts/r2-cors.json to the bucket with an admin token so the BROWSER PUT/GET works in production (object-scoped deploy keys cannot set bucket CORS). Rotate R2 token before public launch.
