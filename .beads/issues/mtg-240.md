# Description

## Bug Description

When Balance resolves, the "Creature equalize to N" message was logged 3 times instead of once.

## Root Cause

The card parser (`follow_sub_ability_chain`) was flattening Balance SubAbility chains into multiple Effect objects during parsing. Then the game loop (`resolve_balance_effect_chain`) was ALSO following SubAbility chains for each Balance effect.

This caused:
1. Balance card parsed with 3 effects: [Land, Hand, Creature]
2. Game loop processed Land effect → followed chain → logged Land, Hand, Creature
3. Game loop processed Hand effect → followed chain → logged Hand, Creature
4. Game loop processed Creature effect → logged Creature

Result: Creature logged 3 times (1+1+1).

## Fix

Modified `parse_effects()` in card.rs to skip `follow_sub_ability_chain()` for Balance effects. Balance effects store their SubAbility reference internally and are handled by the game loop's `resolve_balance_effect_chain()`.

## Verified

`make validate` passes.
