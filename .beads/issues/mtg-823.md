# Description

In check_spellcast_triggers() (mtg-engine/src/game/actions/mod.rs), the PutCounter logging branch hardcodes "+1/+1 counter" in the log message, regardless of the actual counter type. This causes Ral, Crackling Wit's SpellCast loyalty counter to log as "[card] gets a +1/+1 counter" even though the counter placed is a loyalty counter.

The game state is correct (the right CounterType is used in the Effect); only the log message is wrong.

FIXED in branch compat-2025-champ-izzet alongside the Prowess fix (mtg-822): The PutCounter logging match now passes counter_type to c.get_counter() and uses counter_type.to_string() in the format string, producing e.g. "Ral gets a loyalty counter (now 5 counters)" instead of "+1/+1 counter".

Status: Fixed, pending make validate and merge.
