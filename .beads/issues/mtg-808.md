# Description

web/test_image_flicker_memo.js asserts each battlefield <img src> settles on a NON-local (CDN) URL — 'stuck on local' = FAIL. Assumes a hermetic env with NO local image cache (web/images/small + web/images/normal are gitignored, ~4GB). When a dev checkout HAS that cache, the local image loads -> card settles on local -> test FAILS ('stuck on local: Mox Pearl, City of Brass') even though settling-on-local is perfectly STABLE (no flicker). IMPACT: spurious wasm.browser RED in a local make validate 2026-06-05 (gating the mtg-799 seed-7 merge). Proven env-only: relocating web/images/small+normal -> test PASSED ('6 cards held a single settled src'); hermetic CI green; merge landed innocent @6a708dda. FIX: (a) test clears/ignores local web/images cache before run, OR (b) PREFERRED: accept local-settle as 'stable/no-flicker' — the test's purpose is 'settles to ONE stable state, no flicker'; a card holding ONE local URL across 8 re-renders is NOT flickering. Current logic wrongly conflates 'stuck on local' with 'flickering'. Related: mtg-751.
