# Description

RECURRED 2026-06-10 (landing wave): the release delegate hit this as a scary-looking validate failure — Island, Multiversal Passage, Gran-Gran loaded from the local web/images/ cache and were mis-flagged 'stuck'. Confirmed environment-only: 17/17 with the cache moved aside; would pass in GitHub CI. Cost real debugging time, so bumping visibility.

RESOLVED 2026-06-10 on branch claude/web-card-render-robustness (commit 97571c86).
test_image_flicker_memo.js now installs a page.route('**/images/**') interceptor
that forces every local card-image asset (jpg/png/gif/webp/avif) to 404 at the
browser-context level, independent of the on-disk web/images/ cache. The
local->CDN->gatherer cascade is exercised by construction, so the test behaves
identically WITH or WITHOUT the gitignored 67k-file cache present. Proven:
 - fixed + seeded local cache present: 9/9 PASS
 - fixed + no cache: 9/9 PASS
 - OLD test + same seeded cache: FAIL ('stuck on local Swamp.jpg') — reproduces
   the false failure the fix removes.
Full make validate green at 97571c86 (validate_97571c86...log).
