# Description

Card Compatibility: Maze of Ith (cardsfolder/m/maze_of_ith.txt)

Per .claude/skills/compatibility_tracking/SKILL.md.

== Card Status ==

CARD STATUS: WORKING — primary 'Untap target attacking creature' and the
SubAbility$ DBPump 'prevent all combat damage to and by that creature this turn'
both parse and execute correctly.

== Parser-shape test ==

mtg-engine/src/game/actions/tests/effects.rs::test_card_compat_maze_of_ith
verifies BOTH effects are present in the activated ability:
  [x] Effect::UntapPermanent (primary)
  [x] Effect::PreventAllCombatDamageThisTurn (SubAbility, fixed in compat-thedeck)

== Fix (compat-thedeck) ==

Root cause: `DB$ Effect | ReplacementEffects$ RPrevent1,RPrevent2 |
RememberObjects$ Targeted` was not supported by the ApiType::Effect converter
branch; it fell through to None (silently dropped). The Untap worked but the
creature still dealt/received damage.

Fix: added new `Effect::PreventAllCombatDamageThisTurn { target: CardId }`
variant that sets `Card::prevent_all_combat_damage_this_turn = true` on the
targeted creature. The flag is:
  - Set by the effect at resolution (target bound from last_resolved_target
    = the creature that was just untapped)
  - Checked in `assign_combat_damage`: skip dealing AND receiving combat damage
  - Cleared in cleanup_step at end of turn

The ApiType::Effect converter detects the Maze pattern via
`ReplacementEffects$ containing "RPrevent" AND RememberObjects$ Targeted`.

MTG Rules Review: PASS
  CR 615 (replacement effects), CR 506.5 (untapping attacker stays in combat),
  CR 701.31 (prevent), network-determinism preserved (flag is public card state)
