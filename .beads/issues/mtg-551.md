# Description

Test all behavioral aspects of Time Walk in MTG Forge-rs for the 1994 Old School playtest goal.

Card file: cardsfolder/t/time_walk.txt
Deck: 06 Troll Disk Daniellebrunazzo (mtg-562)
Test puzzle: test_puzzles/time_walk_extra_turn.pzl

Card text:
  {1}{U} Sorcery
  Take an extra turn after this one.
Script:
  A:SP$ AddTurn | NumTurns$ 1

Findings (2026-05-29_#2456(e30f4ce1), compat-oldschool-wave5):

1. [x] Parses as {1}{U} Sorcery (CMC 2) with an SP$ AddTurn spell ability
   producing Effect::AddTurn { num_turns: 1 }. Asserted by unit test
   test_card_compat_time_walk.
2. [BROKEN->FIXED] Extra turn was never taken. ROOT CAUSE: the AddTurn
   effect handler (actions/mod.rs) pushed the extra-turn player onto
   `self.turn.extra_turns` (a TurnStructure Vec field that NOTHING ever read
   — write-only/dead), while the turn-rotation code in
   GameState::advance_step drains a DIFFERENT field, `self.extra_turns`
   (a VecDeque). So the queued extra turn was dropped and play rotated
   straight to the opponent. Fix: push to `self.extra_turns.push_back(..)`;
   removed the dead `TurnStructure::extra_turns` field entirely (DRY).
3. [BROKEN->FIXED] Log mis-attribution. The post-resolution logger in
   game_loop/logging.rs receives the UNRESOLVED effect, whose `player` is the
   placeholder PlayerId(0). Because 0 is also a real player id (not a
   sentinel like ALL_PLAYERS), the "grants <player> N extra turn(s)" line
   named Player 1 even when Player 2 cast it. Fix: added an AddTurn arm to
   both pre-logging placeholder-resolution blocks in priority.rs (mirroring
   the existing Mill/SearchLibrary/CreateToken arms) so the logged player ==
   the resolved caster.
4. [x] Runtime (CR 500.7): caster takes a consecutive extra turn inserted
   immediately after the current turn, BEFORE the opponent. Verified by
   puzzle (P0 turn 1 -> extra turn 2 P0 -> turn 3 P1) and by deck-vs-deck
   play (seed 13: P2 turn 26 -> "Extra turn for AI-Heuristic2!" -> turn 27 P2).
5. [N/A] No keywords / triggers / static / replacement abilities.
