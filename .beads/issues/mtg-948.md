# Description

Two coupled deliverables to converge "# closed card-tracking issues" toward "# puzzle-backed".

(a) DURABLE REUSABLE PYTHON SCRIPT (replaces the ad-hoc per-collection scripts):
- No args => global card-compatibility stats derived from the minibeads DB: count of cards with tracking issues, % closed, % puzzle-backed. Broken down by year, then by set.
- `--deck-collection <name>` arg whose values MATCH the website deck-collection options (the same collections offered in web/index.html).
- SUBSUMES AND DELETES the old per-collection scripts (e.g. the old_school one). One script, parameterized — no per-collection copies.

(b) AUDITOR AGENT over CLOSED card-tracking issues:
- For each closed card issue, enumerate the card aspects (passive/triggered/activated abilities, costs, zones, timings, interactions).
- Ensure each aspect is covered by a puzzle (.pzl) file; WRITE one if missing.
- Stamp `PUZZLE_FILE: <path>` into the issue description and apply a `puzzle-tested` label.

GOAL: drive "# closed" toward "# puzzle-backed"; eventually new card issues do not close until puzzle-backed.

AUTHORIZATION: user authorized STARTING this work BEFORE the human review of the puzzle DSL v2 design.
