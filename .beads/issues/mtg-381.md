# Description

Reproducer:
  bash tests/network_vs_local_equivalence_e2e.sh 42 random random

Symptom: at Turn 31 M2 the network server.log emits 4 extra GAMELOG entries that the local game does NOT emit:

    [GAMELOG Turn31 M2] Boar-q-pine (33) takes 1 damage (total: 1)
    [GAMELOG Turn31 M2] Master Piandao (78) takes 1 damage (total: 1)
    [GAMELOG Turn31 M2] Suki, Kyoshi Warrior (76) takes 1 damage (total: 1)
    [GAMELOG Turn31 M2] Turtle-Duck (67) takes 1 damage (total: 1)

These follow:
    Iroh's Demonstration deals 1 damage to each matching creature

Both runs end with the same winner and same number of turns (34) and the actual game state is consistent — only the per-creature damage GAMELOG line emission differs between local and server.

Hypothesis: the per-creature damage event is logged in network mode (server-side resolution path) but suppressed in the local 'mtg tui' main loop, OR vice versa — duplicate emission in network mode.

The Python equivalence test passes for seed=42 random random because tests/network_vs_local_equivalence.py uses fixed seed_p1=seed_p2=3 (different game) and filters 'damage.*life:' patterns. The shell test does not filter and so trips on this mismatch.

Files to inspect:
- mtg-engine/src/game/effects/damage.rs (or wherever DealDamage to each creature resolves)
- The GAMELOG emission point for 'takes N damage (total: N)' lines
- Network server resolution path in mtg-engine/src/network/server.rs

Also see mtg-292 ('Simultaneous combat damage log shows incorrect intermediate life totals') and mtg-296 ('Damage log shows incorrect life total') for related damage-logging issues.
