# Description

## Summary

Fuzz testing reveals network game desync mid-game. The LibraryReordered fix (dca83296) ensures initial library order is synced, but there are separate desync issues occurring during gameplay involving complex effects.

## Symptoms

From fuzz test with seed=2:
- Server: Turn 11, Hands: [3, 5], Hand CardIds: [68, 69, 71, 75, 76]
- Client: Turn 10, Hands: [4, 8], Hand CardIds: [65, 66, 67, 68, 69, 71, 75, 76]
- Client has 3 extra cards (65, 66, 67) that server already played/discarded
- Client is stuck on Turn 10 while server is on Turn 11

## Error Context

Server log shows:
```
[GAMELOG Turn11 UP] Created Clue Token under Ryan's control
[GAMELOG Turn11 UP] Cunning Maneuver (39) gives Ostrich-Horse (72) +3/+1 until end of turn
[GAMELOG Turn11 UP] Tap Mountain for {R}
[GAMELOG Turn11 UP] Tap Swamp for {B}
[WARN  pump] PumpCreature fizzled: target is still placeholder 0
```

Then sync mismatch:
```
NETWORK SYNC MISMATCH DETECTED - P2 choice_seq=112
Server hash: 52141d1979c54f77  Client hash: 063e02fe583d3ed5
DESYNC DETECTED: NetworkController 1 received invalid choice index 3 (only 1 options available)
```

## Root Cause Hypothesis

The desync happens around effects that:
1. Create tokens (Investigate/Clue tokens)
2. Pump creatures (Cunning Maneuver)
3. Involve targeting that fizzles

The client's shadow game state may not be properly handling these effects, causing it to fall behind the server.

## Reproducer

```bash
./tests/network_vs_local_equivalence_e2e.sh 2 random random
```
