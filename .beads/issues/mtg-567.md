# Description

CLUSTER: UI/infra.

The "Auto-run" button in the game UI (tui_game.html / native_game.html) spams the same turn-banner line repeatedly when a HUMAN controller is in the game:

>>> Turn 1 - P1 20 (P2 20) <<<
>>> Turn 1 - P1 20 (P2 20) <<<
... (repeats)

With heuristic/heuristic it works fine. Root cause likely: auto-run advances the game loop without waiting for the human controller's input, so it busy-loops re-rendering the same waiting-for-input state.

FIX: disable (grey out / hide) the Auto-run button whenever any player in the game is a human controller. Only enable it for full-AI (heuristic/heuristic, random/random, etc.) games. Detect controller types from game setup.
