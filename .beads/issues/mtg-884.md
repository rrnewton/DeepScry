# Description

Follow-up: route the "may pay {R}{R} to copy" / UnlessCost optional-payment decision through the PlayerController

Context: implementing the Chain Lightning spell-copy (mtg-152) exposed that the
shared `Effect::UnlessCostWrapper` makes the pay / don't-pay decision INSIDE
`GameState::execute_effect` (the engine), not via a `PlayerController`
ChoiceRequest. This predates the copy work — every "unless you pay" card
(Counterspell-family, Power Sink, etc.) uses the same in-engine decision.

Current behavior (after mtg-152):
- The decision is a hardcoded `should_pay = true` when affordable.
- Affordability is now a correct colored-mana check (ManaEngine) and the cost is
  actually DEDUCTED (pay_mana_cost_by_tapping), so a copy chain terminates on
  mana exhaustion.
- For the copy, the retarget is a deterministic "aim back at the original
  caster".

Why this is acceptable for now (NOT a desync risk):
- The decision is INFORMATION-INDEPENDENT: `should_pay` is constant, `can_pay`
  reads only the payer's own public mana sources, the retarget is a public
  player sentinel. So server and client produce identical results — verified by
  the network-vs-local equivalence test (0 public-zone divergence) and by
  byte-identical replay of the copy-chain puzzle. The network invariant (desync
  always fatal) is preserved.

Why it is still a CONCERN (decision authority, mtg-rules-review item 4):
- A human / interactive client cannot DECLINE the optional payment, nor choose a
  DIFFERENT new target for the copy. The AI always pays-if-able and always
  retargets the caster. That is rules-legal (you MAY pay) but not full decision
  authority.

Proper fix (future): lift the UnlessCost optional-payment + the
CopySpellAbility MayChooseTarget retarget into the game-loop interactive
resolution layer (like mtg-272 discard / Clone / SearchLibrary in
`resolve_top_spell_from_stack_interactive`), adding controller methods
"choose_to_pay_optional_cost" and reusing the existing targeting choice for the
copy. Then HeuristicController owns the pay/retarget strategy and a human client
is prompted.

Affected: Chain Lightning (mtg-489/152) and every UnlessCost card. Tracked as a
follow-up so the copy mechanic can land now with a documented, determinism-safe
simplification.
