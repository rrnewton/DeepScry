# Description

Card Compatibility: Psionic Blast — WORKING after cross-player self-damage FIX.

Card file: cardsfolder/p/psionic_blast.txt
Card text: {2}{U} Instant — Psionic Blast deals 4 damage to any target and 2 damage to you.
Deck: troll-disk (mtg-562), thedeck (mtg-413), jeskai (mtg-561), robots (mtg-559).

CARD STATUS: WORKING.

== Bug (was REOPENED 2026-06-01, false-CLOSED) ==
On a CROSS-PLAYER cast (P2 casts at P1) the "2 damage to you" rider
(DB$ DealDamage | Defined$ You) landed on P1 (the targeted player) instead of
the CASTER P2. The original close only tested the self-cast direction
(caster == P1 == player 0), where the bug was masked.

== Root cause ==
effect_converter encodes Defined$ You as TargetRef::Player(PlayerId::new(0)).
PLACEHOLDER_ID == 0, and there is ALSO a real player 0 (P1). The
spell-resolution path resolve_effect_target (actions/mod.rs) had a
PreventDamage{Player(placeholder)} arm but NO DealDamage{Player(placeholder)}
arm — so the unresolved PlayerId(0) fell straight through to
deal_damage(PlayerId(0), ..) and hit the literal player 0. Coincidentally
correct only when the caster was P1. NOTE: DamageMap$ True was a red herring —
it is a no-op in our eager-damage engine (DamageResolve is a no-op marker); the
bug was the missing placeholder-resolution arm and would have fired without
DamageMap.

== Fix (engine, minimal) ==
1. resolve_effect_target (actions/mod.rs): new arm
   DealDamage{Player(player_id)} if player_id.is_placeholder() -> resolve to
   card_owner (the caster). Mirrors the existing PreventDamage Defined$ You arm
   and the trigger-path resolve_effect_placeholder DealDamage arm.
2. Display logger (game_loop/priority.rs): matching arm so the gamelog names the
   caster, not the targeted player's controller.
Activated-ability path was already correct (uses resolve_effect_placeholder).
Sibling cards: every spell with a Defined$ You DealDamage SubAbility benefits;
painlands (Yavimaya Coast, Murmuring Bosk) use the activated-ability path and
were already correct.
