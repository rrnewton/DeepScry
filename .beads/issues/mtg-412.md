# Description

Companion to bug-draw-reveals-opponent-hand. The fix added LogEntry::private_to + GameLogger::gamelog_private and applied it to the per-card draw line. Other gamelog events likely also leak hidden information when viewed from the opponent's perspective in the WASM/web UIs:

- Tutored cards (search library, find a card) — the chosen card's name appears in the gamelog
- Shuffled-into-library reveals
- Looted/discarded cards (when discard target is the active player's choice)
- 'P scries' — bottom-of-library decisions
- Reveal-from-hand effects (Hymn to Tourach, Duress, Cabal Therapy)

Action: grep for `logger.gamelog` in src/game/state.rs, src/game/actions/, and src/game/game_loop/ and re-tag each entry as `gamelog_private(..., owner)` whenever the message contains hidden info about a single player's choice/state. Filtering at the WASM exporter (gui_view_model.rs / fancy_tui.rs) is already in place via LogEntry::message_for(perspective) — only the producer side needs updating.

Verify with similar regression tests to test_opponent_draws_do_not_leak_card_name.
