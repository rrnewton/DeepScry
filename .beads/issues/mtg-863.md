# Description

Test all behavioral aspects of Torch the Tower in MTG Forge-rs.

Card: cardsfolder/t/torch_the_tower.txt
Set: WOE (Wilds of Eldraine) — reprinted in WC2025 decks 01 (SB), 02, 04
Deck: 2025 World Championship 01_manfield (SB), 02_shibata, 04_henry (tracker mtg-881)

Card text:
  R  Instant
  Bargain (You may sacrifice an artifact, enchantment, or token as you cast this spell.)
  Torch the Tower deals 2 damage to target creature or planeswalker. If this spell was bargained, instead it deals 3 damage to that permanent and you scry 1.
  If a permanent dealt damage by Torch the Tower would die this turn, exile it instead.

Script: A:SP$ DealDamage | ValidTgts$ Creature,Planeswalker | NumDmg$ X | RememberDamaged$ True | ReplaceDyingDefined$ Targeted | SubAbility$ DBScry
        SVar:X:Count$Bargain.3.2   (Bargain ? 3 : 2)   ; SVar:DBScry:DB$ Scry | ScryNum$ 1 | Condition$ Bargain

Findings (2026-06-05_#3008(50175e06)): parses; bargain/conditional-damage unverified.

Findings (2026-06-10_#3177(9e100acf), agent compat-2025-deepverify slot07) — CONFIRMED BROKEN:
1. [x] Parses: cost R, Instant, K:Bargain.
2. [BROKEN] PRIMARY damage does NOT happen. In EVERY game it was cast (dozens of
   occurrences across all native heuristic + random logs for decks 01/02/04), the
   log shows:
     "Torch the Tower (N) deals X damage to None"
   — the literal token "X" is unresolved (X=Count$Bargain.3.2 never computes to a
   scalar) AND the recipient is "None" even though targeting succeeded
   (a separate "→ targeting Enduring Vitality (6)" line appears just before).
   PUZZLE CONFIRMED (debug/compat2025/vitality_test.pzl): P2 casts Torch the Tower
   targeting a 3/3 Enduring Vitality; "deals X damage to None"; the creature takes
   0 and SURVIVES. So Torch the Tower is a DEAD removal spell — it resolves, scries
   (the bargained scry sub-ability fires even un-bargained — also suspect), and
   deals nothing.
3. [BROKEN root cause class] The `Count$Bargain.3.2` X-expression (conditional
   "3 if this spell was bargained, else 2") is not resolved at DealDamage time —
   it reaches damage application as an unresolved X and is treated as 0 / None.
   Same FAMILY as the Combustion Technique X-bug (Count$ValidGraveyard
   Lesson.YouOwn/Plus.2) — EXCEPT Combustion's was FIXED since 2026-06-05
   (Combustion now correctly deals 2+lessoncount; see mtg-881 update). Torch's
   `Count$Bargain.N.M` conditional-count variant is still broken.
4. [unverified] Bargain ADDITIONAL cost (sacrifice artifact/enchantment/token) — the
   AI never bargained in these games; needs a puzzle that forces the bargain cost
   and asserts 3 damage + scry.
5. [unverified] Exile-if-would-die replacement (RememberDamaged$ True /
   ReplaceDyingDefined$ Targeted) — cannot verify while the damage itself is 0.
6. [NOTE] The "deals X damage to None" lines were ALSO a prominent (downstream)
   symptom in the WC2025 04-vs-02 network desync (mtg-908) — but this puzzle proves
   the breakage is PRESENT IN LOCAL PLAY TOO, so it is an independent card bug, not
   merely a network artifact.

## WAVE-1 FIX (2026-06-11_#3229(f2983584), agent slot11 claude/compat-2025-wave1)

Primary damage FIXED: added CountExpression::Bargain{bargained_value, unbargained_value}
variant to parse "Count$Bargain.3.2" correctly. The expression evaluates conservatively
to unbargained_value=2, so Torch the Tower now always deals at least 2 damage.

Evidence: puzzle test_torch_the_tower_base_damage_unbargained passes (Grizzly Bears 2/2
dies from 2 damage). deals_zero assertion is false.
