### Code-review findings (this session, recorded in the design doc §3/§9)
- FIXED: stale doc-comment on SetManifestEntry.hash said "blake3-derived" but content_hash_hex uses std DefaultHasher (SipHash). Corrected.
- OPEN (PR): two hash functions in one pipeline — bins use SipHash (64b), pkg uses `sha256sum|cut -c1-16` (64b). DRY/consistency smell; recommend unifying on one algo (blake3 in Rust for both, or sha256-trunc for both). Not a correctness bug.
- OPEN (PR): DefaultHasher not guaranteed stable across Rust versions; acceptable for cache-bust but flagged.
- OPEN (PR): GC has no grace window (rsync --delete is immediate); client mid-load across redeploy could 404. Acceptable for single-origin hobby deploy; "keep last N" is a possible enhancement.
- OPEN (PR): no local-tree GC between exports (orphans accumulate locally; gitignored, deploy-staging-swept so never reach VM).
