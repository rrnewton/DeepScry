# Description

FIXED in commit on fix-rng-determinism branch.

Centralized seed derivation (mtg-engine/src/game/seed_derivation.rs):
- New `derive_player_seed(master, slot)` helper with the canonical native salt scheme (P1 = master + 0x1234_5678_9ABC_DEF0; P2 = master + 0xFEDC_BA98_7654_3210)
- Used at every callsite: native main.rs (CLI + resume + tournament profile), wasm/mod.rs (single-process), wasm/network/ai_harness.rs (network harness), wasm/fancy_tui.rs, tournament.rs
- Eliminates the WASM seed/seed+1 hack and the divergent JS-passed-seed scheme that silently caused the same --seed to produce different games across modes (a network desync risk per docs/NETWORK_ARCHITECTURE.md).

HeuristicController seed wiring:
- HeuristicController now uses Xoshiro256PlusPlus (matches RandomController) and is Clone+Serialize+Deserialize so its RNG state survives snapshot/restore.
- All 6 production callsites (main.rs network client + 4 P1/P2 fresh+resume sites + ai_harness.rs + fancy_tui.rs + tournament.rs) call HeuristicController::with_seed instead of ::new (which was silently using seed 0).
- ControllerState gained a Heuristic variant so stop-and-go preserves heuristic RNG state across snapshots.

Python --mock now uses engine RandomController:
- agent_game.py's _controller_for_player(mock=True) returns 'random' (was 'mock'), so all three drivers (stop-and-go / persistent / wasm) route mock decisions through the engine's RandomController seeded via derive_player_seed.
- Stop-and-go got a new bypass code path (_run_stop_and_go_engine_only) for fully-engine-side games — the legacy iterative loop with --p1=fixed --p2=fixed and a Python random.Random(seed) is gone.
- MockSession.ask() now raises NotImplementedError; the class is kept only so old imports don't break.

Cross-mode equivalence test (agentplay/test_mode_equivalence.py):
- New test_engine_self_determinism_random_vs_random: two direct mtg tui --p1=random --p2=random --seed=N invocations produce byte-identical engine logs.
- New test_drivers_byte_identical_mock_seed: stop-and-go and persistent drivers produce the same engine choice stream for --mock --seed=42.
- The old test_filtered_action_streams_overlap (which tolerated divergence) is replaced.

Verified: same --seed produces identical heuristic and random games across CLI invocations.
