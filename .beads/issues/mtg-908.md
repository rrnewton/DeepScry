# Description

## Summary

A real determinism bug surfaced during the WC2025 04-vs-02 game: the WASM rewind/replay verifier detected a genuine divergence at turn 13 because the Dig effect heuristic (`dig_card_score`) reads hidden library card identities. On a network game the server scores the real top-N while each client shadow scores its own (different) hidden view → different picks → fatal state-hash desync.

## Root Cause (confirmed 2026-06-10_#3177)

Thundertrap Trainer's ETB trigger fires via `check_triggers_inner` → `execute_effect` → `execute_dig`, bypassing the priority.rs `choose_from_library_with_hook` mechanism. `execute_dig` used an inline AI heuristic that reads real (hidden) card identities. Both clients run this heuristic on their own shadow state, picking different cards.

## Fix (2026-06-11, slot19 branch claude/fix-dig-desync-mtg677)

Added `StateSyncEntry::DigDecision` to the state-sync log mechanism (same pattern as mtg-420 LibraryReordered). Implementation:
- SERVER: `execute_dig` records the heuristic decision in `SubActionScratch::pending_dig_decisions`. `NetworkController::request_choice` drains it into `ChoiceRequest::dig_decisions`. The coordinator broadcasts `ServerMessage::DigDecision` to BOTH clients before the ChoiceRequest (same as LibraryReordered).
- SHADOW: `apply_state_sync_at` / `apply_state_sync_up_to_frontier` deliver the kept-list as `StateSyncEntry::DigDecision` → `SubActionScratch::pending_dig_authoritative_decision`. `execute_dig` on the shadow checks this field FIRST and uses it if present, bypassing the heuristic.
- Non-network / local AI games: heuristic runs as before (no shadow, no hidden-info leak).
- Also added `BufferedFact::DigDecision` for the minimal-lazy-protocol buffer path (mtg-752).
