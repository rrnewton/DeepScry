# Description

Bug: AB$ Tap auto-targets its own source (Icy Manipulator)

When activating an AB$ Tap ability with ValidTgts$ Artifact,Creature,Land (Icy Manipulator), both the fixed and heuristic controllers auto-selected the FIRST legal target, which was the source artifact itself. The ability resolved but tapped Icy Manipulator instead of an opponent permanent, making it useless.

Observed (pre-fix, seed 42):
  Icy Manipulator activates ability: Tap target artifact, creature, or land.
    -> targeting Icy Manipulator (3)

== FIXED (2026-06-01_#2587(1faf5ae6), compat-monoblack-v2) ==

Two changes in mtg-engine/src/game/heuristic_controller.rs:

1. **should_cast_spell**: Added arm for utility non-creature artifacts with
   non-mana activated abilities. The heuristic now casts Icy Manipulator when
   the opponent has permanents that the Tap ability can affect.

2. **choose_targets** (has_tap_effect guard): When a spell or activated ability
   has a TapPermanent effect, the heuristic now filters valid_targets to prefer
   opponent permanents. The fallback chain is:
   - opponent permanents (preferred)
   - any valid target (fallback, covers corner cases)
   CR 602.1b: target choice is part of activation; no rules violation regardless
   of which legal target is selected.

This generalizes to all AB$ Tap activated abilities (Royal Assassin, Prodigal
Sorcerer, etc.) — any ability that produces Effect::TapPermanent benefits from
this fix.

Bug closed by: mtg-511 (Icy Manipulator WORKING with e2e evidence).
