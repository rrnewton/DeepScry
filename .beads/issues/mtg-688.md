# Description

User-reported 2026-06-01 (playtest; "not immediate priority"): casting Mind Twist did not ask the player to choose X. Expected: when casting any spell with {X} in its cost, the choice tree should offer X = 0..N where N is the maximum the player can pay (available/producible mana, after other costs). This appears to be MISSING for ALL X-cost spells, not just Mind Twist — i.e. X is not being solicited from the controller at cast time.

This is the GENERAL X-cost choice-at-cast support. Distinct from mtg-113 (closed — that added X-cost PARSING to ManaCost; this is the runtime CHOICE/payment-of-X step). Mind Twist card-compat tracker mtg-564 (closed) should be revisited once this lands — its closure did not exercise the X-choice path.

Scope: locate where spell cast cost is assembled and where the controller chooses costs; add an X-selection choice (ChoiceResult) bounded by max payable mana; thread the chosen X into NumCards/NumDmg/etc. Determinism: the X choice must be a normal controller choice so it replays identically native==WASM==network. Add a card-compat test that casts an X spell and asserts X is solicited and applied (e.g. Mind Twist discarding X cards). Cross-check against forge-java AbilityUtils X handling.
