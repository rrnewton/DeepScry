# Description

Implement phase-triggered abilities that fire during specific phases/steps.

## Status

**Significantly expanded** - Now supports DealDamage and GainLife effects with ValidPlayer$ You filtering. Cards like Juzám Djinn now work correctly.

## What's Implemented

1. ✅ TriggerEvent enum now derives Copy (src/core/effects.rs:128)
2. ✅ Phase trigger parsing in card loader (src/loader/card.rs)
   - Detects `T:Mode$ Phase` triggers
   - Supports `Phase$ Upkeep` → BeginningOfUpkeep
   - Supports `Phase$ End` or `Phase$ EndOfTurn` → BeginningOfEndStep
   - ✅ **NEW: Parse Execute$ SVar references for DealDamage effects** (2025-11-30)
   - ✅ **NEW: Parse Execute$ SVar references for GainLife effects** (2025-11-30)
   - ✅ **NEW: ValidPlayer$ You filtering (controller-only triggers)** (2025-11-30)
3. ✅ Phase trigger checking in game loop (src/game/game_loop/steps.rs)
   - `check_phase_triggers()` method with active player filtering
   - Called from `upkeep_step()` and `end_step()`
   - ✅ **NEW: [controller_only] flag for ValidPlayer$ You** (2025-11-30)
4. ✅ Phase trigger execution (src/game/actions/mod.rs)
   - ✅ **NEW: check_triggers_for_controller() method** (2025-11-30)
   - ✅ **NEW: Placeholder PlayerId(0) resolution for Defined$ You** (2025-11-30)
   - ✅ **NEW: Logging of trigger effects** (2025-11-30)
5. ✅ Tests
   - test_upkeep_trigger_damage_to_controller - Juzám Djinn style damage
   - test_upkeep_trigger_controller_only - Verifies trigger doesn't fire on opponent's turn
   - test_juzam_djinn_from_cardsfolder - E2E test with real card
6. ✅ All validation passes (533 tests passed, 1 skipped)

## Working Cards (NEW 2025-11-30)

- **Juzám Djinn** - "At the beginning of your upkeep, Juzám Djinn deals 1 damage to you."
  - Verified working in gameplay: Player2 takes 1 damage each of their upkeeps

## What's Still Missing

1. ❌ **No condition checking** - CheckSVar$ and SVarCompare$ not supported
   - Land Tax needs: "if opponent controls more lands"
2. ❌ **No optional triggers** - OptionalDecider$ not supported
   - Land Tax is optional: "you may search..."
3. ❌ **No opponent targeting** - Defined$ Opponent not implemented
4. ❌ **Complex effects** - Only DealDamage and GainLife parsed from SVars
   - No DrawCards, Search, Pump, etc.
