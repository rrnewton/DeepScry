# Description

## Separate seed for initial shuffle vs subsequent game

**COMPLETED 2025-11-29**: The `--deck-seed` argument has been implemented.

### Original Request
We want to retain deterministic testing by controlling RNG seeds, but sometimes sample many DIFFERENT games from the space of all possible games between two decks. To this end, `--deck-seed` controls only the initial shuffling, allowing `--seed` to vary independently.

### Implementation

- Added `--deck-seed` CLI argument to the `tui` command
- When `--deck-seed` is provided:
  - Library shuffling uses the deck seed
  - Game RNG is re-seeded with `--seed` after shuffling
- If only `--deck-seed` is provided (no `--seed`), the deck seed is used for everything

### Usage Examples

```bash
## Run two games with same hands but different RNG
mtg tui deck1.dck deck2.dck --deck-seed=42 --seed=1
mtg tui deck1.dck deck2.dck --deck-seed=42 --seed=2

## Both start with identical hands, different game outcomes
```

### Note on from_entropy

The issue also mentioned adding support for `--seed=clock` to seed from entropy. This is already implemented as `--seed=from_entropy`:
```bash
mtg tui deck1.dck deck2.dck --seed=from_entropy
```
