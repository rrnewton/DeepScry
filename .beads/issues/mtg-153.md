# Description

## Current Implementation

The current `deal_damage_to_creature` implementation uses a **simplified model** (actions/mod.rs:1171):

```rust
// If damage >= toughness and no indestructible, creature instantly dies
let dies = amount >= toughness as i32 && !has_indestructible;
if dies {
    self.move_card(target_id, Zone::Battlefield, Zone::Graveyard, owner)?;
}
```

**There is no damage field on creatures.** Damage doesn't accumulate - creatures either die instantly or take no effect.

## MTG Rules Requirements

Per MTG Comprehensive Rules:
- Damage dealt to creatures is marked on them (CR 120.3)
- Damage remains until cleanup step (CR 514.2)
- Multiple sources can deal damage to same creature in one turn
- Creature dies as state-based action when damage >= toughness (CR 704.5g)

## Implementation Plan

### Phase 1: Add damage field
1. Add `pub damage: i32` to Card struct (core/card.rs)
2. Initialize to 0 in Card::new()
3. Add to serialization/deserialization

### Phase 2: Modify damage dealing
1. Change `deal_damage_to_creature` to accumulate damage instead of instant-kill
2. Move death check to state-based actions
3. Add `DealDamageToCreature{card_id, amount}` variant to undo::GameAction
4. Log damage to undo log (with prior_log_size)

### Phase 3: Cleanup step
1. Modify `cleanup_step` to clear damage from all creatures (state.rs:509)
2. This can use existing `cleanup_temporary_effects` or a new helper
3. Clear damage AFTER checking SBAs (so lethal damage creatures die first)

### Phase 4: Undo logic
1. Implement undo for DealDamageToCreature (subtract damage)
2. Ensure damage field is included in state hash comparison

## Example Scenario

```
Turn 1, Main Phase:
- Cast Lightning Bolt (3 damage) targeting 5/5 creature
  - Creature now has 3 damage marked
  - Creature does NOT die (3 < 5 toughness)
- Cast Shock (2 damage) targeting same creature
  - Creature now has 5 damage marked
  - SBA: Creature dies (5 >= 5 toughness)

Cleanup Step:
- All damage cleared from surviving creatures
```

# Resolution

Implemented full MTG-compliant damage tracking:

## Implementation

1. **Added damage field to Card struct**
   - Tracks damage marked on creatures
   - Initialized to 0, cleared in cleanup step

2. **Updated deal_damage_to_creature**
   - Accumulates damage instead of instant-kill
   - Logs damage for visual feedback

3. **Added check_lethal_damage state-based action**
   - Checks for damage >= toughness
   - Respects indestructible ability
   - Implements MTG CR 704.5g

4. **Updated cleanup_temporary_effects**
   - Clears damage at end of turn per MTG CR 514.2
   - Resets P/T bonuses

## MTG Rules Compliance

- CR 120.3: Damage remains marked on creatures
- CR 514.2: Damage cleared in cleanup step
- CR 704.5g: Lethal damage state-based action
- CR 702.12b: Indestructible creatures immune to lethal damage

## Testing

- Updated test_etb_trigger_damage to call check_lethal_damage
- All 340 tests pass
