# Description

Chaos Orb's activated ability ({1}, {T}: flip + destroy touched + self-destroy) is implemented by mapping FlipOntoBattlefield to Effect::DestroyPermanent with a placeholder target (CardId::new(0)) at mtg-engine/src/loader/effect_converter.rs:1302. In play, the target resolves to Chaos Orb itself, so only Chaos Orb is destroyed. Opponent permanents are never affected.

Reproducer: ./target/release/mtg tui --start-state test_puzzles/chaos_orb_destroys_target.pzl --p1=fixed --p2=zero --p1-fixed-inputs='activate Chaos Orb' --stop-on-choice=8 --json --log-tail=120 --seed 42 --verbosity 3

Expected: P2's Grizzly Bears (or some random nontoken permanent) should be destroyed in addition to Chaos Orb. Java Forge simulates the random touch; mtg-forge-rs degrades to 'destroy target nontoken permanent' but never prompts for the target.

Suggested fix options:
(a) Add a TargetChoice prompt: change the FlipOntoBattlefield branch in effect_converter.rs to require a TargetChoice (Card.IsNontoken) restricted to opponent permanents preferred by AI.
(b) Use RNG to pick 1-3 random nontoken permanents from anyone's battlefield and destroy them (closer to Java behaviour).

Related compat issue: mtg-ad79fd (Card Compatibility: Chaos Orb).
