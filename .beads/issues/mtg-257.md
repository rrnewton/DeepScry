# Description

## RemoteController missing choose_blocker_for_lethal_damage implementation

## RESOLVED - commit c6d4cedd

Implemented `choose_blocker_for_lethal_damage` and `choose_blocker_for_remaining_damage`
in both `RemoteController` (server-side) and `NetworkLocalController` (client-side).

Network games with random controllers now produce identical gamelogs to local games.

## Root Cause

`RemoteController` was using default implementations that just picked the first blocker
instead of forwarding the choice to clients.

## Fix

- `RemoteController` now forwards damage ordering choices via `get_opponent_choice()`
- `NetworkLocalController` handles `ChooseBlockerForLethalDamage` and `ChooseBlockerForRemainingDamage` request types
- Added `--controller` filter to `network_fuzz_test.py` for targeted testing

## Verification

```bash
python3 bug_finding/network_fuzz_test.py --local-equivalence --controller random
```
All completed games showed 0 gamelog divergence.
