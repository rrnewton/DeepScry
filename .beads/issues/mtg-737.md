# Description

scripts/check_clean_environment.py (the conflicting-process precheck run at the start of validate.sh / make validate) produces FALSE POSITIVES that abort validation, costing real wall-clock time. Recurs for every slot/agent.

OBSERVED (slot03, compat-1994-sweep, 2026-06-03):
Two separate validate launches aborted with 'Environment not clean - conflicting processes detected', flagging:
  - 'cargo (PID NNN): /bin/bash -c source .../snapshot-bash...sh ... eval cd .../slot03; ... pgrep -f "cargo test -p mtg-engine" ...' — a MONITORING bash shell whose cmdline merely CONTAINS the substring 'cargo test -p mtg-engine' (inside a pgrep argument). It is not a cargo process.
  - 'validate.sh (PID NNN): /bin/bash -c ... ./scripts/validate.sh ...' — the bash -c WRAPPER that launched validate.sh (cmdline contains './scripts/validate.sh'); flagged as a conflicting validate.sh process even though it is the current invocation's own parent.

ROOT CAUSE: detection keys on cmdline substrings ('cargo', 'validate.sh') of processes whose /proc/<pid>/cwd is within the worktree, but does NOT exclude (a) the current process's own ancestry (the bash -c wrapper that invoked validate.sh), nor (b) non-cargo shells that merely mention 'cargo'/'validate' in their argv (monitoring/poll loops, pgrep args).

REPRO: from a worktree, run a background poll loop like  while kill -0 $(pgrep -f "cargo test -p mtg-engine"); do sleep 15; done  then launch  ./scripts/validate.sh  via a bash -c wrapper -> precheck aborts citing those shells.

WORKAROUND USED: invoke via 'make validate' (wrapper cmdline is 'make validate', no 'validate.sh' substring) and kill leftover monitoring shells first.

SUGGESTED FIX: scope detection to ACTUAL cargo/mtg/validate.sh processes — verify the executable (readlink /proc/<pid>/exe basename == cargo/mtg, or argv[0]), not a cmdline substring; AND exclude the precheck's own process ancestry (walk PPID chain up from self and skip those PIDs). validate-infra — likely belongs with the validate-overhaul work (mtg-717).
