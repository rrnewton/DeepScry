# Description

1994 World Championship compat sweep (mtg-684 / mtg-713 B12). Kismet: "Artifacts, creatures, and lands your opponents control enter tapped."

Script (GLOBAL form of the ETB-tapped replacement):
  R:Event$ Moved | ValidCard$ Artifact.OppCtrl,Creature.OppCtrl,Land.OppCtrl | Destination$ Battlefield | ReplaceWith$ ETBTapped | ActiveZones$ Battlefield

ROOT CAUSE (was BROKEN): the loader detected `ReplaceWith$ ETBTapped` with a substring match (`value.contains(...)`) and set the HOST card's own `enters_tapped` flag. So Kismet itself entered tapped and the global 'opponents' permanents enter tapped' effect was never applied (no engine support for the global replacement).

FIX: loader/card.rs now classifies the replacement STRUCTURALLY (tokenize | then $):
  - ValidCard$ Card.Self  -> host self-taps (enters_tapped; the ~846 tapped lands)
  - global predicate       -> stored as CardDefinition::etb_tapped_global (TargetRestriction)
Only predicates we can faithfully evaluate are installed (types + OppCtrl/YouCtrl/Any, ActiveZones Battlefield, no IsPresent/ValidCause); unmodeled qualifiers are refused (no-op) rather than over-matched. At ETB (game/state.rs move_card) the engine scans the battlefield for a source carrying a matching etb_tapped_global and taps the entering permanent, resolving OppCtrl relative to the SOURCE's controller (CR 614).
