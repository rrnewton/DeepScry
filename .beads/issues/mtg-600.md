# Description

## Hypnotic Specter: Mode Random + Defined TriggeredTarget — PRIMARY BUG FIXED

Closed 2026-06-01 gardening: PRIMARY BUG (wrong player discards) FIXED. 

Evidence:
1. effect_converter.rs line 167: maps TriggeredTarget/TriggeredPlayer to PlayerId::target_opponent() sentinel
2. triggers.rs lines 230-244: resolves target_opponent to ctx.opponent (the damaged player)
3. Dedicated regression test: tests/hypnotic_specter_discard_e2e.sh verifies 'Player 2 (opponent) discards a card' and attacker does NOT discard

REMAINING MINOR GAP (not blocking): Mode$ Random — the DiscardCards effect in effects.rs has no 'random' boolean field. In a 1-card-discard scenario (Hypnotic Specter: NumCards 1) this doesn't matter since there's only one card to choose. For multi-card triggers on a target with multiple cards, the discard is deterministic (lowest CardId) rather than truly random. This is the mtg-589 fix outcome described in this issue's description. Filed separately as mtg-648 covers Discarded trigger events.

The original bug report (Brand discards Swamp instead of Roger after Specter hit) is resolved: TriggeredTarget now resolves to the damaged player.
