# Description

## deploy-cloud.sh: web_stage unbound variable — FIXED

Closed 2026-06-01 gardening: FIXED. This was one of the two deploy-cloud.sh footguns fixed in commit 48a28eff (per mtg-590 description).

Evidence: scripts/deploy-cloud.sh lines 527-532:
  local web_stage
  web_stage="$(mktemp -d)"
  # Bake the path into the trap NOW (double-quotes): web_stage is `local`, so
  # a deferred '$web_stage' would be unbound at EXIT under `set -u` and make
  # the script exit 1 AFTER a successful deploy (lying exit code).
  trap "rm -rf '$web_stage'" EXIT

The trap now uses single-quotes inside double-quotes to expand the value immediately, avoiding the unbound variable error at EXIT.
