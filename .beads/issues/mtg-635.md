# Description

Activated abilities whose script declares 'ActivationZone$ Graveyard' (activatable while the card is in the graveyard) are never enumerated as available actions. ActivatedAbility (mtg-engine/src/core/effects.rs:3110) has no activation_zone field, so all activated abilities are implicitly battlefield-only; the action enumerator only walks battlefield permanents for activations.

Discovered via Earthquake Dragon (mtg-502, cardsfolder/e/earthquake_dragon.txt):
  A:AB$ ChangeZone | Cost$ 2 G Sac<1/Land> | Origin$ Graveyard | Destination$ Hand | ActivationZone$ Graveyard | SpellDescription$ Return CARDNAME from your graveyard to your hand.

Repro: with Earthquake Dragon in p0graveyard and lands on battlefield, the
{2}{G},Sac-a-land: return-to-hand ability does NOT appear in available actions
(only 'pass' and 'play Forest' are offered).

Bug class: ALL graveyard/exile/hand-activated abilities (unearth, flashback-as-
activated, graveyard recursion engines). Fix needs:
1. an activation_zone field on ActivatedAbility (default Battlefield),
2. parser support for ActivationZone$ in card.rs,
3. action enumeration to also walk the activating player's graveyard for
   abilities whose activation_zone matches.

Affects: Earthquake Dragon (mtg-502, rogerbrand mtg-387).

## FIXED (2026-05-31_#2558(7af638d8), compat-finish-rogue)

Added activation_zone: Zone field to ActivatedAbility (default Battlefield).
Parsed ActivationZone$ in card.rs via Zone::from_str_lenient.
Extended push_activatable_abilities to walk player graveyard for abilities
with activation_zone == Zone::Graveyard, checking mana/sacrifice/sorcery-speed
costs. effect_converter.rs: AB$ ChangeZone | Origin$ Graveyard | Destination$
Hand now maps to MoveSelfBetweenZones. priority.rs: new match arm for
MoveSelfBetweenZones { source: self_target() } patches source to card_id.
HeuristicController: ZoneReturn ability type activates during main phase.

Verified via Earthquake Dragon (mtg-502).
