# Description

The web deck builder (web/deck_editor.html) card-details pane now shows card ART alongside the text fields (name, mana cost, type line, P/T, oracle). Closed together with three UX fixes from a user live-test (claude/deck-editor-3col).

## WRONG-IMAGE BUG (user live-test) — root cause + fix

User added "Lightning Bolt"; the card-details DESCRIPTION was correct but the IMAGE was a different card ("Thrum of the Vestige" — not even a real card). Root cause: the art was resolved via Gatherer's `Image.ashx?name=<name>` handler, which does a SERVER-SIDE FUZZY/partial name match and returns an arbitrary card. The text fields come straight from the catalog entry (correct); only the image used the unreliable fuzzy-name service, so description and image could drift apart.

Fix: resolve art through Scryfall's EXACT-named image endpoint
  https://api.scryfall.com/cards/named?exact=<name>&format=image&version=normal
`exact=` is a case-insensitive EXACT name lookup — no fuzzy collisions — so the image is always the SAME card whose details render. On a miss Scryfall returns 404 (handled by the <img> onerror, which hides the art; text details stay intact). Verified Lightning Bolt, an Adventure card (Bonecrusher Giant), and a basic land (Forest) each show their OWN correct art.

Guard added: renderCardDetails stamps the panel (`dataset.selectedCard`) and the <img> (`dataset.cardName`) with the selected card name, and an image-load handler drops a stale image if the selection changed before it loaded — belt-and-braces against any future async-selection race.

## Other UX fixes in the same branch
- Search box was rendering ~160px tall (a `flex: 1 1 160px` on a flex-COLUMN child made 160px a HEIGHT); now a normal single-line input (~29px).
- Restructured into a responsive 3-column full-height layout: col1 = controls/chrome (deck name + stats, search + filters, Saved Decks + Save/Export/Import/Clear, Card Details); col2 = search results (full-height, scrolls); col3 = Mana Curve on top + deck/sideboard list scrolling below. Degrades to 2-col (<=980px) then single-column (<=680px).

## Tests
web/test_deck_editor.js section 4b updated to assert a Scryfall exact-name <img>; new section 4c asserts the per-card image MAPPING (details name, panel/img name-stamps, and exact= art URL all match the selected card) for several specific cards.
