## ROOT CAUSE: TCP_NODELAY missing (NOT rewind/replay quadratic)
The >1000× slowdown is NOT quadratic. ms/choice is constant across game lengths (CV=4.7% across seeds 7/19/42) — O(N) linear, not O(N²). The rewind/replay machinery is NOT the bottleneck.

Root cause: MISSING  on the TcpStream before WebSocket handshake. Each choice is a small message; Nagle's algorithm holds it until the receiver's delayed-ACK timer fires (Linux: ato=40ms). Actual loopback RTT=78µs; observed per-choice: 28.3ms. Every ChoiceRequest↔ChoiceResponse exchange hits this.

FIX (3 lines, landed on integration): server.rs line ~708 before  + client.rs after  on the  TCP stream. Expected speedup: ~28× on native-network (14.9s → ~0.5s per game on loopback).

## Attribution breakdown
- TCP_NODELAY (Nagle+delayed-ACK): ~95% of network overhead
- Log spam (RUST_LOG=off): ~5-10% reduction
- network_debug/serde_json: <0.5%
- Rewind/replay: NONE
