# Description

The post-deploy HTTP probe added in deploy-healthcheck-and-probes (6df2d191) curls https://deepscry.net:8080/ WITHOUT -k/--insecure. With the Cloudflare proxy turned off (direct origin access), the server presents the Cloudflare Origin Cert, which is NOT in the system CA bundle, so curl fails with curl: (60) SSL certificate problem: unable to get local issuer certificate and the probe reports HTTP 000 / connection refused / empty /health for EVERY endpoint -- a FALSE failure. Verified 2026-05-28: a manual curl -sk https://deepscry.net:8080/health returns {"sha":"6df2d191",...} and landing returns 200; the service is healthy.

FIXED (branch infra-deploy-hardening): run_post_deploy_probe in scripts/deploy-cloud.sh now builds curl_opts with -sSk (added the k/--insecure flag), so the direct-origin probe skips TLS-trust verification of the CF Origin Cert and stops false-failing. All content assertions are unchanged and remain the real proof of a good deploy: landing 200, /pkg JS > 50000 bytes, /pkg wasm > 1000000 bytes, /data/sets/index.json valid JSON, /health has a sha (with local-vs-served mismatch warning), /lobby reachable (101/400/426/405/404). Comment block updated to explain the CF-origin-cert rationale. Bash-only change; not exercised by make validate.
