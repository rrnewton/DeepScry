# Description

The post-deploy HTTP probe added in deploy-healthcheck-and-probes (6df2d191) curls https://deepscry.net:8080/ WITHOUT -k/--insecure. With the Cloudflare proxy turned off (direct origin access), the server presents the Cloudflare Origin Cert, which is NOT in the system CA bundle, so curl fails with 'curl: (60) SSL certificate problem: unable to get local issuer certificate' and the probe reports HTTP 000 / connection refused / empty /health for EVERY endpoint — a FALSE failure. Verified 2026-05-28: a manual 'curl -sk https://deepscry.net:8080/health' returns {"sha":"6df2d191",...} and landing returns 200; the service is healthy. FIX: the probe should use 'curl -sk' (or --cacert pointing at the CF origin CA) for the deepscry.net:8080 direct-origin case. Until fixed, treat probe failures as advisory and verify manually with -k.
