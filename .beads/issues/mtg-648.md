# Description

FIXED in claude/compat-1994-enchant-triggers (2026-06-09_#3045(bb0fdd18)).

There is now a TriggerEvent::Discarded distinct from CardDiscarded:
  - CardDiscarded (pre-existing): a battlefield permanent watching ITS
    controller's discards (Monument to Endurance: 'Whenever you discard a card').
  - Discarded (NEW): a SELF-discard punisher carried on the card BEING discarded
    (Psychic Purge: 'When a spell or ability an opponent controls causes you to
    discard CARDNAME, that player loses 5 life'). Fires from the discarded card's
    last-known-information as it moves Hand->Graveyard (CR 603.10a -- a
    leaves-the-hand trigger that looks back in time).

Engine work delivered:
1. TriggerEvent::Discarded added (mtg-engine/src/core/effects.rs).
2. Loader (loader/card.rs Mode$ Discarded branch) now splits on the tokenized
   ValidCard$ value: 'Card.Self' -> Discarded (self-punisher, trigger_self_only),
   anything else -> CardDiscarded. ValidCause$ SpellAbility.OppCtrl sets a new
   Trigger.requires_opponent_cause flag.
3. PlayerId::triggered_cause_controller() sentinel (core/entity.rs) +
   Defined$ TriggeredCauseController parsing in the LoseLife converter
   (loader/effect_converter.rs).
4. Firing site GameState::check_self_discarded_triggers (game/actions/mod.rs),
   called from discard_card after the card reaches the graveyard. The CAUSE of
   the discard is read from current_damage_source (the resolving spell/ability,
   public information). resolve_spell_with_choices (game_loop/priority.rs) now
   sets current_damage_source for the interactive-discard path (Mind Rot, Hymn,
   etc.) exactly as resolve_spell_execute_effects already does. The
   requires_opponent_cause gate fires only when the cause's controller is an
   OPPONENT of the discarding player; a cleanup-step hand-size discard (CR 514.1,
   a turn-based action with no spell/ability cause) does NOT fire it.
5. TriggerContext.cause_controller + resolve_effect_placeholder arm resolve the
   LoseLife to that cause controller (game/actions/triggers.rs).

CARDNAME substitution: the Discarded trigger description now substitutes the
card's real name (cosmetic, so the log reads 'discard Psychic Purge', not
'discard CARDNAME').

Game-log evidence (opponent-forced discard fires; self-discard does NOT):

```sh
## Positive: opponent's Mind Rot forces P0 to discard Psychic Purge -> P2 (the
## Mind Rot caster, an opponent) loses 5 life.
./target/release/mtg tui --start-state test_puzzles/psychic_purge_discarded_by_opponent.pzl \
  --p1=zero --p2=fixed --p2-fixed-inputs='cast Mind Rot;target Player 1;pass;pass' \
  --stop-on-choice=10 --json --seed 42 --verbosity 3
##   Player 1 discards Psychic Purge
##   Trigger: When a spell or ability an opponent controls causes you to discard Psychic Purge, that player loses 5 life.
##   Player 2 loses 5 life (life: 15)

## Negative: cleanup-step self-discard -> NO punish (both players stay at 20).
./target/release/mtg tui --start-state test_puzzles/psychic_purge_self_discard_no_punish.pzl \
  --p1=fixed --p2=zero --p1-fixed-inputs='pass;pass;pass;pass;pass;pass;pass;pass' \
  --stop-on-choice=14 --json --seed 42 --verbosity 3
##   Player 1 discards Psychic Purge   (no 'loses 5 life' line)
```

Regression tests:

    (mtg-engine/src/game/actions/tests/effects.rs) asserts the Discarded trigger
    parses with requires_opponent_cause + a LoseLife(5) to the
    TriggeredCauseController sentinel.
  - E2e: tests/psychic_purge_discard_punisher_e2e.sh (auto-discovered into
    make validate + CI) covers primary 1-damage mode + opponent-punisher fires +
    self-discard no-op.

Bug-class generalization: the SAME Mode$ Discarded | ValidCard$ Card.Self trigger
template is now PARSED (no longer silently dropped) for the whole class --
Guerrilla Tactics (deal 4 to any target), Quagnoth (return to hand),
Ajani's Last Stand, Mangara's Blessing, etc. The LoseLife payload (Psychic Purge)
is fully working end-to-end. Payload-specific resolution for the OTHER members
(DealDamage-to-any-target needs target selection; ChangeZone-self-return needs
Defined$ Self) may need follow-up but the trigger event/firing infrastructure --
the actual mtg-648 gap -- is closed. Tracked per-card under mtg-534.
