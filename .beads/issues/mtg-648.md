# Description

RESOLVED for the spell-discard path (2026-06-09_#3106(b3281a33), claude/compat-psychic-purge).

Added TriggerEvent::Discarded — a self-trigger fired on the DISCARDED CARD's
own LKI as it moves Hand->Graveyard (CR 603.6/603.10), distinct from the
existing CardDiscarded battlefield-watcher (Monument to Endurance). Wired
Psychic Purge's opponent-forced-discard punisher:
  T:Mode$ Discarded | ValidCard$ Card.Self | ValidCause$ SpellAbility.OppCtrl
    | Execute$ TrigLoseLife
  SVar:TrigLoseLife:DB$ LoseLife | Defined$ TriggeredCauseController | LifeAmount$ 5

Design (per user override of the original plan): the discard CAUSE is threaded
as an EXPLICIT `cause: Option<PlayerId>` parameter on GameState::discard_card —
NOT mutable transient GameState state. This is the safest possible shape for a
determinism-critical value: there is nothing to reconstruct on a network shadow
or a WASM rewind/replay (an immutable function argument has no
serialize/restore problem at all). Cause sources:
  - resolve_top_spell_with_discard_hook -> Some(spell_owner)  [Mind Twist /
    Hymn to Tourach / Mind Rot — the discard SPELLS]
  - priority_round activated-ability discard + loot -> Some(current_priority)
  - execute_effect DiscardCards/Loot arms + cleanup-step over-limit discard ->
    None / self (no opponent cause)

Pieces:
  1. TriggerEvent::Discarded (core/effects.rs) + Trigger.requires_opponent_cause
     flag (ValidCause$ ...OppCtrl).
  2. PlayerId::triggered_cause_controller() sentinel
     (TRIGGERED_CAUSE_CONTROLLER_ID, Defined$ TriggeredCauseController),
     resolved at fire time in check_discarded_self_trigger to the concrete
     cause.
  3. Parser split (loader/card.rs Mode$ Discarded): ValidCard$ Card.Self ->
     Discarded self-trigger; ValidCard$ Card.YouOwn -> the existing
     CardDiscarded watcher (unchanged).
  4. discard_card fires check_discarded_self_trigger AFTER the LKI move; the
     opponent gate is cause.is_some_and(|cc| cc != owner).

Gate semantics (CR 701.9 Discard): self-discards (cleanup, own looting; cause
== owner) and cause-less discards do NOT fire the punisher.

KNOWN REMAINING GAP (filed as mtg-894): forced discards from an opponent's
TRIGGERED ABILITY (Hypnotic Specter's random discard) still resolve through
the shared execute_effect path with cause=None, so the punisher does not fire
for that sub-case. The dominant 1994 vector (discard spells) is covered;
threading cause through execute_effect's 52 callers was out of scope. Behavior
is stricter-than-printed (punisher silently absent), never wrong-direction.

Tests:
  - Unit: game::actions::tests::effects::test_card_compat_psychic_purge.
  - E2E: tests/psychic_purge_discard_punisher_e2e.sh (Mind Rot -> punisher
    fires, caster loses 5 life; cleanup self-discard -> no fire).
