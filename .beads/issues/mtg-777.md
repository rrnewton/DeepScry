# Description

LOW (debug/diagnostic-only), found in the mtg-677 desync-review.

PROBLEM: the rewind-verifier [VERIFIER FIELD DIFF] dump in mtg-engine/src/wasm/fancy_tui.rs truncates each side with a BYTE slice `&cs[..8000]` (the cap raised 600->8000 in the seed-2 byte-pin work). A byte slice at an index that is not a UTF-8 char boundary PANICS. Card names / serialized JSON are ASCII in practice today, but a multi-byte char (e.g. an accented card name, or future unicode in a log) at the boundary would panic the diagnostic path.

FIX: truncate on a CHAR boundary — use char_indices()/floor_char_boundary (or take(8000) over chars), not a raw byte slice. Diagnostic-only, no gameplay impact.

CROSS-REF: mtg-677 (seed-2 byte-pin diagnostic).
