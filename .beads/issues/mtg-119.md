# Description

Implement intelligent evaluation and timing for activated abilities in HeuristicController.

## Current State

`should_activate_ability()` returns false for all abilities to prevent infinite loops.

## Required Work

### 1. Ping Abilities (Prodigal Sorcerer)
- Reference: DamageDealAi.java:196-200, 682-703
- Logic: Only ping when stack is clear
- Timing:
  - End of turn (if reusable and our turn is next)
  - Main 2 (if PW ability or leaves at EOT)
  - When can kill valuable creature
- Target: Best opponent creature we can kill

### 2. Pump Abilities (Shivan Dragon)
- Reference: PumpAi.java
- Logic: Pump before combat to enable/improve attacks
- Timing: Main1 before combat
- Target: Best attacker

### 3. Mana Abilities
- Skip these (let mana system handle)
- Check `is_mana_ability` flag

## Implementation Plan

1. Detect ability type from effects
2. Implement ping ability evaluation
3. Implement pump ability evaluation  
4. Add timing restrictions (stack empty check)
5. Test with Prodigal Sorcerer, Shivan Dragon
