# Description

Implement activated ability evaluation and timing for heuristic AI.

## Current State (2025-11-03)

IMPLEMENTED! The heuristic AI now intelligently evaluates and times activated abilities.

## Implementation Complete

### 1. Ping Abilities (Prodigal Sorcerer) ✅
- Reference: DamageDealAi.java:196-200, 682-703
- Logic: Only ping when stack is clear (placeholder - always returns true for now)
- Timing:
  - End of turn (if reusable and ability doesn't require sacrifice)
  - Main 2 (for immediate value)
  - When can kill valuable creature
- Target: Any opponent creature we can kill with damage

### 2. Pump Abilities (Shivan Dragon) ✅
- Reference: PumpAi.java
- Logic: Pump before combat to enable/improve attacks
- Timing: Main1 before combat
- Evaluation: 
  - Checks if creature can attack (not tapped, no summon sickness)
  - Significant power boost (3+) always worth it
  - Otherwise checks for positive power and non-negative toughness

### 3. Mana Abilities ✅
- Correctly skipped (let mana system handle)
- Check `is_mana_ability` flag

## Implementation Details

Created new enum `ActivatedAbilityType` with three variants:
- `Ping { damage: i32 }` - for damage-dealing abilities
- `Pump { power: i32, toughness: i32 }` - for stat-boosting abilities
- `Other` - for not-yet-categorized abilities

Added methods:
- `classify_activated_ability()` - detects ability type from effects
- `is_stack_empty()` - stack empty check (placeholder implementation)
- `has_valuable_ping_target()` - finds opponent creatures killable with ping
- `can_kill_valuable_creature()` - checks if ping can kill valuable target
- `would_pump_enable_attack()` - evaluates pump value for combat

Added `Cost::requires_sacrifice()` method to check if cost sacrifices permanent.

## Known Limitations

1. Stack empty check is placeholder - always returns true
   - TODO: Expose game.stack through GameStateView
2. Ping targeting doesn't prioritize "best" creature yet
   - Currently just finds any killable creature
3. Pump evaluation is simplified
   - Doesn't consider specific blockers or combat math
   - Just checks for significant power boost

## Next Steps
