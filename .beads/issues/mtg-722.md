# Description

DONE (web-ui-backlog): token card art 404 fixed. Root cause: predefined tokens are named '<X> Token' but Scryfall/Gatherer index them under the BARE name (verified live: named?exact=Clue%20Token→404, named?exact=Clue→Clue token card). So all sources in the [local,scryfall,gatherer] cascade missed. Fix: new single-source-of-truth image_overlay::card_image_url_cascade appends stripped-name fallbacks (scryfall+gatherer for 'Clue') when a name ends in ' Token'; tui_get_image_urls delegates to it. JS onerror cascade is generic so it auto-tries them; graceful degradation preserved (card-name div shows when all art fails). sync-web-assets.sh is NOT the lever (it syncs site chrome, not card art). No MTG-rules impact. Unit tests added in image_overlay (wasm32-gated, run via wasm-pack test). cargo check + clippy-wasm clean. Commit on web-ui-backlog.
