REMAINING BLOCKER (network gate RED): network GUI e2e (monored mirror seed=13, and default deck) FAILS with a P2 server/shadow state-hash mismatch at ~choice_seq=12, action_count=96, on the FIRST turn>=2 undo-rewind. SAME action count (96==96) but different content hash => a card is in a different zone between the client's replayed state and the server. The debug turn-start verifier does NOT fire (turn-start hash stable across rewinds), so the divergence is in FORWARD replay producing a state the server didn't, surfaced when the local controller submits compute_view_hash at the new choice.

DIAGNOSIS: native snapshot/resume rewind+replay PASSES, so rewind_to_turn_start itself is sound for the local path. The divergence is NETWORK-specific (the harness rewinds every step in an AI-vs-AI game, far more often than the human path which rewinds only on user input). Prime suspects, in order:
1. Library-reorder replay: sync_callback drains server library reorders destructively (client.drain_library_reorders, NOT undo-logged). After a turn>=2 rewind, the queue is empty, so a replayed draw can pop a different library card than the original run -> zone/hash divergence.
2. Opponent (remote) choice replay side-channels: WasmRemoteController sets last_spell_ability / last_library_search_result when consuming live OpponentChoice; ReplayController short-circuits the inner remote, so those side-channels are not repopulated on replay.
3. Reveal re-application: drain_reveals is destructive; likely OK because shadow card instantiation persists across rewind, but unverified.

This is exactly mtg-610's linchpin "is rewind_to_turn_start COMPLETE for network/mid-resolution state?" -> answer so far: NOT complete for the network harness's destructively-consumed inputs (library reorders / opponent-choice side-channels). PROPER FIX: make consumed network inputs (library reorders, reveals, opponent choices) REPLAYABLE — record them keyed by action_count and re-feed during replay instead of re-draining the live queue — OR snapshot full GameState at each turn boundary (like turn-1) instead of undo-rewinding (heavier but sidesteps undo-completeness entirely).
