# Description

INCIDENT 2026-05-28 ~12:03-12:51 UTC: deepscry.net:8080 went down for ~48min. Root cause: the deepscry.service runs as a systemd --user service but the service user (newton) did NOT have lingering enabled (loginctl show-user → Linger=no). systemd user managers (and their services) are torn down when the user's last login session ends, so the server died whenever the admin/deploy SSH session closed, and only restarted when a new SSH session re-created the user manager. Journal confirms: 'shutdown signal received' at 12:03:22 UTC (session end), no restart until 12:51:16 UTC (next SSH = my status check). MITIGATED live by running 'loginctl enable-linger newton' (now Linger=yes; survives logout + reboot). FIX: scripts/deploy-cloud.sh 'config' phase must run 'loginctl enable-linger $REMOTE_USER' (idempotent) right after installing the systemd-user unit, so every VM bootstrap is durable and this can't regress. Also consider a post-deploy assertion that linger=yes. Related: the deploy hardening (mtg-581 probe -k bug) and the /health endpoint added in d1636cfa now make this detectable.
