# Description

Test all behavioral aspects of Karma in MTG Forge-rs for the 1994 Old School playtest goal.

Card file: cardsfolder/k/karma.txt
Set: LEG. Deck: 06_jeskai_aggro_joseantonioprieto sideboard (mtg-561).
Test puzzle: test_puzzles/karma_upkeep_swamp_damage.pzl

Card text:
  {2}{W}{W} Enchantment
  At the beginning of each player's upkeep, Karma deals damage to that
  player equal to the number of Swamps they control.

Script:
  T:Mode\$ Phase | Phase\$ Upkeep | ValidPlayer\$ Player | Execute\$ TrigDamage
  SVar:TrigDamage:DB\$ DealDamage | Defined\$ TriggeredPlayer | NumDmg\$ X
  SVar:X:Count\$Valid Swamp.ActivePlayerCtrl

Findings (2026-05-30_#2530(f7a005ca), compat-wave14-jeskai):

1. [x] Parses as {2}{W}{W} Enchantment.
2. [x] BeginningOfUpkeep trigger present, ValidPlayer\$ Player => fires on
   EACH player's upkeep (NOT controller_turn_only).
3. [BROKEN->FIXED] Before this wave the phase-trigger DealDamage handler in
   loader/card.rs only recognised `Defined\$ You` + a FIXED NumDmg. Karma's
   `Defined\$ TriggeredPlayer | NumDmg\$ X` produced NO effect => the trigger
   fired (log line) but dealt 0 damage. Karma was a do-nothing enchantment.
4. [x] FIX: added Effect::DealDamageToTriggeredPlayer { count, target_self }.
   - loader/card.rs phase-trigger DealDamage branch now emits it for
     Defined\$ TriggeredPlayer / ChosenPlayer (variable amount) and for
     Defined\$ You with a variable amount; fixed Defined\$ You stays plain
     DealDamage (unchanged Juzam Djinn path).
   - check_triggers_for_controller resolves it against the ACTIVE player
     (whose upkeep fired) and evaluates the CountExpression against that same
     player (CR 603 triggered ability).
   - count_permanents_matching extended to recognise basic-land subtypes
     (Swamp/Plains/Island/Mountain/Forest) and ActivePlayerCtrl.
