# Description

## Avoid Invalid game action irrespective of controller

**RESOLVED 2025-11-29**: The summoning sickness bug described in this issue has been fixed.

### Original Bug
Random controller was able to choose creatures with summoning sickness to attack, resulting in:
```
Error declaring attacker: Invalid game action: Creature has summoning sickness and can't attack this turn
```

### Fix Already Implemented
The fix is in `mtg-engine/src/game/game_loop/actions.rs:get_available_attacker_creatures()`:
- Lines 27-40 check for summoning sickness before adding creatures to available attackers list
- Creatures that entered this turn without haste are filtered out
- Creatures with defender are also filtered out

### Verification
Ran multiple games with random controllers (seeds 1-42, 100, 9999, 12345, etc.) - no summoning sickness errors occurred.

### Legacy Interface
The deprecated legacy interface in `legacy.rs` still has a `// TODO: Check for summoning sickness` comment, but this code is:
1. Marked `#[deprecated]`
2. Not used by the main game loop (combat.rs uses `get_available_attacker_creatures` instead)

The issue is resolved without needing additional code changes.
