CARD STATUS: not-tested (skeleton placeholder)

Findings (2026-05-30_#2487(30dd3c20), compat-oldschool-wave11):

1. [x] Parses as {G} World Enchantment.
2. [x] S:Mode$ Continuous | Affected$ Creature | AddKeyword$ Haste parses into
   a GrantKeyword(Haste) static ability with AffectedSelector::AllCreatures
   (bare 'Affected$ Creature' => all creatures on the battlefield, both
   players). Verified by unit test (parser shape).
3. [x] Granted haste defeats summoning sickness in real combat (CR 702.10b,
   CR 302.6). With Concordant Crossroads on the battlefield, a Grizzly Bears
   cast on Turn 1 immediately attacks that SAME turn. Verified via game log.
   - Engine already routed combat attack-eligibility through
     has_keyword_with_effects(Haste) (actions/combat.rs:87,
     game_loop/actions.rs:97), so granted keywords are honored. No engine
     change required.
   - Control (no Crossroads): identical bears enters Turn 1 but cannot
     attack until Turn 3 (P1's next turn) — confirms the haste grant is what
     enables the Turn-1 attack.

Known adjacent gap (NOT exercised by this card, no card in rogerbrand relies
on it): a few mana/tap-ability summoning-sickness checks still use the
inherent-only card.has_keyword(Haste) (mana_engine.rs:378/659/957,
mana_index.rs:402) rather than has_keyword_with_effects. This would matter
only for a creature with a {T}: ability that gains haste from a grant the
turn it enters; out of scope for Concordant Crossroads (haste's relevant
effect here is attacking, which is correct).
