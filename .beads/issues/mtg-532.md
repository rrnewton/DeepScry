# Description

Test all behavioral aspects of Power Sink in MTG Forge-rs.

Card: cardsfolder/p/power_sink.txt
Set: LEA (Alpha)
Deck: thedeck Peterschnidrig (mtg-413, sideboard)

Card text:
  Power Sink {X}{U} Instant
  Counter target spell unless its controller pays {X}. If that player
  doesn't, they tap all lands with mana abilities they control and
  lose all unspent mana.

Script:
  A:SP$ Counter | UnlessCost$ X | TargetType$ Spell | ValidTgts$ Card
   | SubAbility$ TapLands | UnlessResolveSubs$ WhenNotPaid
  SVar:TapLands:DB$ TapAll | ValidCards$ Land.hasManaAbility
   | Defined$ TargetedController | TapperController$ True
   | SubAbility$ ManaLose
  SVar:ManaLose:DB$ DrainMana | Defined$ TargetedController
  SVar:X:Count$xPaid

Findings (2026-05-29_#2473(1048d207), compat-oldschool-wave9):

1. [x] Parses as {X}{U} Instant.
2. [x] SP$ Counter with UnlessCost$ X resolves: caster chooses X,
       targets a spell, opponent gets the "pay {X}" prompt at
       resolution.
3. [x] When not paid, TapAll fires and taps all opponent's lands with
       mana abilities ("Power Sink taps all matching permanents").
4. [BROKEN] DB$ DrainMana ("lose all unspent mana") is unimplemented:
   the log emits "Power Sink (...) has unimplemented effect 'DrainMana'"
   — the opponent never actually loses unspent mana. Filed as the
   pattern-level engine gap (one Bug: issue covers all DrainMana
   cards).
5. [N/A] Mana / triggers / static / replacement abilities — none.
