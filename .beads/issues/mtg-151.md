# Description

## Problem

The undo test `test_aggressive_undo_snapshots` in `tests/undo_e2e.rs` only checks zone sizes and basic player state (lines 696-822), but does NOT verify deep card-level state like:
- Card fields: turn_entered_battlefield, attached_to, power_bonus, toughness_bonus, damage
- Mana pool contents
- Tapped state of individual cards
- Counters on cards
- Which specific cards are in each zone (only checks counts, not identities)

This is why our tests PASS even though several mutations are not logged to the undo system! The missing undo logging for turn_entered_battlefield, attached_to, and lands_played_this_turn should have been caught by these tests.

## Solution

The `state_hash.rs` module already provides `compute_state_hash()` which serializes the ENTIRE GameState to JSON and hashes it. We should:

1. Replace `verify_state_equivalence()` in `tests/undo_e2e.rs` to use state hashing
2. Compare `compute_state_hash(current)` vs `compute_state_hash(snapshot)`
3. When hashes differ, deserialize both to JSON and compare to show which fields diverged
4. This will catch ANY divergence in game state after undo

## Files to Modify

- `tests/undo_e2e.rs` - Update verify_state_equivalence() function (lines 696-822)

## Acceptance Criteria

- [ ] verify_state_equivalence uses compute_state_hash for comparison
- [ ] When state diverges, show diff of JSON representations
- [ ] After fixing this test, it should FAIL (exposing the missing undo logging bugs)
- [ ] Keep existing assertions as fallback for clearer error messages
