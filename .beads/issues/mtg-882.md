# Description

Structural refactor (no behavior change) to bring heuristic_controller.rs under the ~2000-line file guideline in CLAUDE.md.

## What
Split the 10110-line mtg-engine/src/game/heuristic_controller.rs into a module
directory mtg-engine/src/game/heuristic_controller/ organized by MTG-decision shape:
- mod.rs: HeuristicController struct, constructors, helper types (CombatFactors,
  CombatOutcome, ActivatedAbilityType), and the PlayerController trait impl
  (the decision entrypoints the engine calls, which dispatch into submodules).
- creature_eval.rs: creature/card valuation (evaluate_creature family).
- mana_lands.rs: mana counting, land selection, mana-source scoring.
- spell_selection.rs: master choose_best_spell dispatcher.
- combat.rs: attack evaluation, block assignment, combat math.
- spell_eval.rs: should_cast_* family, pump tricks, removal targeting.
- abilities.rs: activated-ability classification/scoring/activation timing.
- tests.rs: the unit-test module.

## Behavior preservation
Pure relocation: method bodies are byte-identical; only visibility was widened
to pub(crate) so sibling submodules can call across module boundaries, and the
three helper types were made pub(crate) for the same reason. Verified by diffing
seeded heuristic-vs-heuristic game logs (multiple seeds x decks) before and after.

## Follow-ups
- Break the 3 longest functions into helpers (evaluate_creature_impl ~552 lines,
  should_cast_pump ~395, should_cast_spell ~307) - separate increment.
- tests.rs is ~3558 lines; could be split by topic later.
- mtg-77 determinism hazard in choose_modes/choose_from_options flagged in-code,
  fix tracked separately (NOT a pure refactor).
