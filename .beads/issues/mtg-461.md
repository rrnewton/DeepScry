# Description

## CI clippy WASM target fails — FIXED

Closed 2026-06-01 gardening: FIXED in branch fix-mtg-461 (merged to integration).

Evidence: rust-toolchain.toml line 4: targets = ["wasm32-unknown-unknown"]

The fix adds the WASM target to the pinned toolchain so both CI and fresh local clones automatically install wasm32-unknown-unknown rust-std when rustup installs nightly-2025-11-28. The dtolnay/rust-toolchain action reads this file and installs the listed targets.

Root cause: ci.yml used dtolnay/rust-toolchain@nightly (latest nightly) for the WASM install but rust-toolchain.toml overrode to nightly-2025-11-28; wasm rust-std only existed on latest nightly.

Verified: cargo clippy -p mtg-forge-rs --target wasm32-unknown-unknown --no-default-features --features wasm-tui -- -D warnings runs clean.
