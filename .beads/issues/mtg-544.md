# Description

Test all behavioral aspects of Spirit Link in MTG Forge-rs for the 1994 Old School playtest goal.

Card file: cardsfolder/s/spirit_link.txt

Decks containing this card (1994 Old School playtest):
- 02_thedeck_peterschnidrig (mtg-413)
- 03_robots_jesseisbak (mtg-559)
- 06_troll_disk_daniellebrunazzo (mtg-562)

Card text:
  Spirit Link {W} Enchantment - Aura
  Enchant creature
  Whenever enchanted creature deals damage, you gain that much life.

Script:
  K:Enchant:Creature
  T:Mode$ DamageDealtOnce | ValidSource$ Card.AttachedBy | Execute$ TrigGain
  SVar:TrigGain:DB$ GainLife | Defined$ You | LifeAmount$ X
  SVar:X:TriggerCount$DamageAmount

Findings (2026-05-30_#2530(199b91e1), compat-wave13-thedeck):

1. [x] Parses as {W} Enchantment - Aura with Enchant:Creature.
2. [x] Castable from hand targeting a creature; resolves and ATTACHES
       correctly ('Spirit Link enchants Grizzly Bears').
3. [BROKEN] The lifegain trigger NEVER fires. T:Mode$ DamageDealtOnce
       + ValidSource$ Card.AttachedBy + LifeAmount$ X (TriggerCount$DamageAmount)
       is unsupported: the trigger mode isn't parsed, Card.AttachedBy source
       resolution for triggers doesn't exist, and the damage amount isn't
       plumbed into GainLife. When the enchanted creature deals 2 combat
       damage, the controller gains 0 life (stays at start).
       Filed as: mtg-r9po1 (3-part engine gap, see that issue).

Reproducer (lifegain does NOT fire — controller life unchanged):

```sh
cat > /tmp/sl.pzl <<'P'

./target/release/mtg tui --start-state /tmp/sl.pzl --p1=fixed --p2=zero \
  --p1-fixed-inputs='attack Grizzly Bears;pass;pass;pass;pass;pass' \
  --stop-on-choice=12 --seed 42 --verbosity 3
```

