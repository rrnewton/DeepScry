# Description

Bug: Animate Dead ETB-reanimate trigger not firing — sacrifice-on-leave FIXED

## Status: sacrifice-on-leave DBDelay now FIXED (2026-06-09, claude/compat-1994-enchant-triggers-v2)

The previously-deferred DBDelay drawback now works: "When Animate Dead leaves
the battlefield, that creature's controller sacrifices it."

## What was missing (deferred item 1 in this issue)
reanimate_aura_target reanimated the creature and attached the Aura but never
registered the leave-the-battlefield sacrifice trigger, so destroying/bouncing
Animate Dead left the reanimated creature on the battlefield permanently
(strictly better than printed — a free reanimation).

## Fix
- core/delayed_trigger.rs: new DelayedEffect::SacrificeOther { target } —
  sacrifice a DIFFERENT card than the one the trigger watches. (The existing
  Sacrifice variant sacrifices the tracked card itself; Animate Dead must watch
  the AURA but sacrifice the CREATURE.)
- state.rs fire_delayed_trigger: SacrificeOther moves `target` Battlefield->
  graveyard (no-op if the creature already left, e.g. it died first and dragged
  the Aura off as an SBA — no double-sacrifice / crash) and logs
  "<creature> is sacrificed (Animate Dead left)".
- actions/mod.rs reanimate_aura_target: after attaching, register a
  DelayedTrigger watching the AURA (tracked_card = aura_id) for a
  ZoneChange{ from: Battlefield, to: any }, with effect SacrificeOther{ target =
  reanimated creature }.
