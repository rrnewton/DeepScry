# Description

Animate Dead's ETB-reanimate trigger and the DBDelay leave-the-battlefield SACRIFICE trigger.

## Status: WORKING (2026-06-10, branch claude/compat-animate-dead-sacrifice)

Reanimation flow (resolve_spell_finalize -> reanimate_aura_target):
  * Aura whose chosen target is in a graveyard skips the immediate attach path.
  * Moves the graveyard creature to battlefield under the Aura's controller (GainControl$ True).
  * Applies etbCounter + ETB triggers; attaches the Aura post-reanimation; -1/-0 continuous effect applies.

### DBDelay sacrifice-on-leave: IMPLEMENTED
"When Animate Dead leaves the battlefield, that creature's controller sacrifices it."
  * New DelayedEffect::SacrificeOther { card } (delayed_trigger.rs).
  * reanimate_aura_target registers a ZoneChange{from:[Battlefield], to:[]} delayed trigger watching the Aura; effect sacrifices the reanimated creature.
  * fire_delayed_trigger executes SacrificeOther, no-op if the creature already left (no double-sacrifice).
Matches Java SVar:DBDelay (Mode$ ChangesZone | Origin$ Battlefield | Execute$ TrigSacrifice | RememberObjects$ RememberedLKI). CR 603.7 (delayed triggers), CR 701.21 (sacrifice).

### REWIND/REPLAY DESYNC BUG — FOUND AND FIXED (2026-06-10)
The first version of this change was WRONG: it had a 100%-deterministic rewind/replay desync that blocked landing (the network.multideck leg / rogerbrand seed-3, an All Hallow's Eve + Animate Dead reanimator deck). My commit FALSELY claimed the trigger "reconstructs identically on rewind"; the replay oracle proved otherwise:
  REWIND/REPLAY FATAL: turn-start state hash for turn 6 changed across rewinds.
  [VERIFIER FIELD DIFF] 'delayed_triggers': PRIOR next_id=1 triggers=[]  vs  CURRENT next_id=2 ...

Two missing undo-log entries (the registration AND the firing of the delayed trigger were not recorded in the undo log, so rewind-to-turn-start was not a faithful inverse):
  1. reanimate_aura_target called delayed_triggers.add() WITHOUT logging GameAction::RegisterDelayedTrigger — so rewind never removed the registered trigger. FIX: log RegisterDelayedTrigger after add (mirrors every other registration site, e.g. Mana Drain / dies-trigger in actions/mod.rs).
  2. check_delayed_triggers_on_zone_change fired (removed) a delayed trigger WITHOUT logging GameAction::FireDelayedTrigger — so when the SacrificeOther trigger fired mid-turn (Aura destroyed) it was removed un-loggably, and on rewind RegisterDelayedTrigger's undo found nothing to roll back, leaving delayed_triggers.next_id drifted. FIX: log FireDelayedTrigger before firing in the zone-change path (mirrors the already-correct Mode$ Phase firing path). This also hardens EVERY zone-change delayed trigger (Earthbend-style returns, dies-triggers), not just Animate Dead.

After both fixes: rogerbrand seed-3 PASSES 3/3 (was FATAL 100%); monored seed-13 PASSES; animate_dead_sacrifice_on_leave e2e still PASSES.
