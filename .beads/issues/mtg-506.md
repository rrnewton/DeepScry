# Description

Test all behavioral aspects of Flash Counter in MTG Forge-rs.

Card: cardsfolder/f/flash_counter.txt
Set: LEG (Legends)
Deck: thedeck Peterschnidrig (mtg-413, main deck)

Card text:
  Flash Counter {1}{U} Instant
  Counter target instant spell.

Script:
  A:SP$ Counter | TargetType$ Spell | ValidTgts$ Instant

Findings (2026-05-29_#2470(06c7caec), compat-oldschool-wave9):

1. [x] Parses as {1}{U} Instant (CardType::Instant, generic=1, blue=1).
2. [x] SP$ Counter with TargetType$ Spell + ValidTgts$ Instant is
       converted to the same Effect::Counter path that powers
       Counterspell (mtg-495). The ValidTgts$ Instant predicate
       restricts targeting to instant spells on the stack.
3. [x] Live counterspell behavior verified: P1 casts Lightning Bolt;
       P2 with {U}{U} available is offered Flash Counter as an
       in-priority action and successfully counters the Bolt.
4. [N/A] Mana abilities — instant with no mana ability.
5. [N/A] Triggered / static / replacement abilities — none printed.

Reproducer:

```sh
./target/release/mtg tui \
  --start-state test_puzzles/flash_counter_counters_instant.pzl \
  --p1=fixed --p2=fixed \
  --p1-fixed-inputs="cast Lightning Bolt;*;*" \
  --p2-fixed-inputs="cast Flash Counter;*;*" \
  --stop-on-choice=10 --seed 42 --verbosity 3
```

Expected log evidence:

```
Player 1 casts Lightning Bolt (...) (putting on stack)
Player 2 casts Flash Counter (...) (putting on stack)
  → targeting Lightning Bolt (...)
Flash Counter (...) resolves
Flash Counter (...) counters Lightning Bolt (...)
```
