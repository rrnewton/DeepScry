The .dck deck format

A .dck file is a plain-text list of the cards in a Magic: The Gathering deck. It is the format DeepScry (and the desktop Forge program it descends from) uses. Because it is plain text, you can write it by hand, store it anywhere, and share it as a simple link.

Minimal example

[metadata]
Name=My Burn Deck

[Main]
4 Lightning Bolt
20 Mountain

[Sideboard]
2 Pyroblast

The rules

ElementMeaning
[metadata]Optional header. The Name= line under it sets the deck's display name.
[Main]Starts the main deck. Lines after it are main-deck cards (this is also the default if no section header appears).
[Sideboard]Starts the sideboard. Lines after it are sideboard cards.
4 Lightning BoltA card line: a count, a space, then the exact card name.
# commentA line starting with # is a comment and is ignored.

Notes

  • Section headers are case-insensitive: [main], [MAIN], and [Main] are the same.
  • An optional set/collector suffix after the card name is ignored, e.g. 4 Lightning Bolt|LEA|[161] loads as Lightning Bolt.
  • Card names must match the spelling in DeepScry's card database exactly.
  • Blank lines are ignored, so you can space the file out for readability.

Loading a deck from a URL

On the launcher you can paste a Custom Deck URL that points directly at a plain-text .dck file. The browser fetches and parses it on your device.

The host must allow cross-origin reads (CORS). Raw file links from GitHub (raw.githubusercontent.com), GitHub Gists (the "Raw" button), and most raw paste services work. A normal web page (not a raw file) or a host that blocks cross-origin reads will not load — use a raw/plain-text link instead.
